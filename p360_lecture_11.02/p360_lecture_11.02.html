Introduction to Arrays

Same Idea, Different Contract

  • One name holds a sequence of values.
  • The capacity cannot grow or shrink.

Slides 3–12

Why Arrays?

  • Some collections have a natural, permanent capacity.
  • Fixed capacity makes storage requirements predictable.
  • Arrays are built into C++ and appear throughout existing code.

Array Declaration

int scores[5];
element type   name   capacity
     int      scores     5

Reading a Declaration

double temperatures[7];
char gradeLetters[5];
bool seatsTaken[30];
  • temperatures can hold seven double values.
  • gradeLetters can hold five char values.
  • seatsTaken can hold thirty bool values.

The Size Must Be Known

const int DAYS = 7;
double temperatures[DAYS];
  • A standard C++ array requires a compile-time constant capacity.
  • A value entered while the program runs cannot set this capacity.

One Name, Many Values

scores value value value value value
storage position first second third fourth fifth
  • Every position stores the same element type.
  • The positions occupy one contiguous block of memory.

[Graphic suggestion: redraw the table as five adjacent labeled storage boxes under one array name.]

Capacity and Data Count

  • Capacity: the total number of available positions.
  • Data count: the number of positions currently holding meaningful data.
  • A raw array does not track the data count for the program.

Arrays and Vectors

Shared idea Different contract
Ordered values Array capacity is fixed
One element type No push_back() or pop_back()
Contiguous storage No built-in size member
Can visit every value Program manages logical data count

Summary

  • An array is an ordered, fixed-capacity collection of one type.
  • Its compile-time capacity and meaningful data count are separate ideas.

Array Initialization

Starting with Known Values

  • What should every position contain when the array is created?

Slides 13–24

Full Initialization

int scores[5] = {91, 84, 78, 96, 88};
  • Five positions receive five values.
  • Values are stored in the order written.

Initialization Layout

int scores[5] = {91, 84, 78, 96, 88};
position first second third fourth fifth
value 91 84 78 96 88

Partial Initialization

int scores[5] = {91, 84};
position first second third fourth fifth
value 91 84 0 0 0
  • Missing trailing values are initialized to zero.

Zero Initialization

int scores[5] = {};
value 0 0 0 0 0
  • Empty braces initialize every element to zero.

Inferred Size

int scores[] = {91, 84, 78, 96, 88};
  • The compiler counts the initializer values.
  • The resulting capacity is five and remains fixed.

Uninitialized Arrays

int scores[5];
  • A local array declared this way has indeterminate element values.
  • Reading any element before assigning it a value is unsafe.

Initializer Count

int first[5] = {4, 8};             // allowed: remaining values are zero
int second[3] = {4, 8, 15, 16};   // error: too many initializers
  • The initializer list may contain at most the declared capacity.

Initialization Happens Once

int scores[3] = {70, 80, 90};

// scores = {75, 85, 95};  // array assignment is not allowed
  • Braces initialize the array as it is created.
  • Later changes target individual elements.

Choosing an Initialization Form

Need Form
Known starting values Full initializer list
Known prefix, zero remainder Partial initializer list
All zeros Empty braces
Capacity matches supplied values Inferred size

Summary

  • Initializer lists assign values from left to right.
  • Missing trailing values become zero, excess values are rejected, and uninitialized local elements must not be read.

Indexing Array Elements

Selecting One Element

  • How does the program identify one position inside an array?

Slides 25–39

Zero-Based Positions

human position first second third fourth fifth
C++ index 0 1 2 3 4
  • The first element uses index 0.
  • The index is always one less than its human position number.

Index and Value

int scores[5] = {91, 84, 78, 96, 88};
index 0 1 2 3 4
value 91 84 78 96 88
  • The index identifies a position; the value is stored at that position.

The Subscript Operator

scores[2]
array name   index expression
  scores          2
  • Brackets select one element from the array.

Reading an Element

cout << scores[0] << '\n';

double average = (scores[0] + scores[1]) / 2.0;
  • A subscripted element behaves like a value of the array’s element type.

Changing an Element

scores[2] = 82;
scores[4] += 5;
index 0 1 2 3 4
value 91 84 82 96 93

Indexes Are Expressions

int i = 2;

cout << scores[i] << '\n';
cout << scores[i + 1] << '\n';
cout << scores[4 - i] << '\n';
  • An index can be any whole-number expression that produces a valid position.

Valid Index Range

capacity:      SIZE
first index:   0
last index:    SIZE - 1
  • For capacity five, valid indexes are 0 through 4.
  • Negative indexes and index SIZE are outside the array.

Out-of-Bounds Access

const int SIZE = 5;
int scores[SIZE] = {};

scores[5] = 100;   // outside the array
  • Raw arrays do not perform automatic bounds checks.
  • Out-of-bounds access causes undefined behavior.

Named Capacity

const int SIZE = 5;
int scores[SIZE] = {};

cout << "Last index: " << SIZE - 1 << '\n';
  • One name keeps the declaration and bounds reasoning aligned.

Counting Elements with sizeof

int scores[] = {91, 84, 78, 96, 88};

int count = sizeof(scores) / sizeof(scores[0]);
  • Total array bytes divided by one element’s bytes gives the element count.

Where the sizeof Count Works

int scores[] = {91, 84, 78, 96, 88};
int count = sizeof(scores) / sizeof(scores[0]);
  • This works where scores is still an actual array variable.
  • An array parameter does not preserve this size information; functions come next lecture.

Tracing Index Access

int values[4] = {10, 20, 30, 40};
int i = 1;

values[i] += 5;
i++;
values[i] = values[0];
index 0 1 2 3
final value 10 25 10 40

Summary

  • Index 0 selects the first element; SIZE - 1 selects the last.
  • The subscript operator reads or changes one element, and its index must remain inside the valid range.

Traversing Arrays

Moving Through Every Element

  • How can one operation reach each position exactly once?

Slides 40–53

Index Loop Structure

for (int i = 0; i < SIZE; i++)
loop part array meaning
i = 0 start at the first index
i < SIZE remain inside the valid range
i++ advance to the next index

Index Loop Code

#include <iostream>
using namespace std;

int main()
{
    const int SIZE = 5;
    int scores[SIZE] = {91, 84, 78, 96, 88};

    for (int i = 0; i < SIZE; i++)
    {
        cout << scores[i] << '\n';
    }

    return 0;
}

Following the Index

iteration i scores[i]
1 0 91
2 1 84
3 2 78
4 3 96
5 4 88
  • When i becomes 5, i < SIZE is false.

Processing with an Index

int total = 0;

for (int i = 0; i < SIZE; i++)
{
    if (scores[i] >= 70)
    {
        total += scores[i];
    }
}

Changing Elements by Index

for (int i = 0; i < SIZE; i++)
{
    scores[i] += 5;
}
  • Each subscript selects the element to modify during that iteration.

Range-Based Traversal

for (int score : scores)
{
    cout << score << '\n';
}
  • The loop visits every declared element in order.
  • No index variable is exposed.

Copies and References

for (int score : scores)       // score is a copy
{
    cout << score << '\n';
}

for (int& score : scores)      // score refers to the element
{
    score += 5;
}

Range-Based Loop Code

#include <iostream>
using namespace std;

int main()
{
    int scores[] = {91, 84, 78, 96, 88};

    for (int& score : scores)
    {
        score += 2;
    }

    return 0;
}

Capacity or Data Count?

const int CAPACITY = 10;
int scores[CAPACITY] = {};
int dataCount = 4;

for (int i = 0; i < dataCount; i++)
{
    cout << scores[i] << '\n';
}
  • A range-based loop would visit all ten declared positions.
  • The index loop can stop after the four meaningful values.

Loop Bound Reasoning

condition final attempted index result
i < SIZE SIZE - 1 inside the array
i <= SIZE SIZE outside the array
  • The condition describes when another iteration is allowed.

Choosing the Loop

Need Loop choice
Read every declared value Range-based loop
Change every declared value Range-based loop with reference
Use the position Index loop
Stop at a logical data count Index loop
Compare neighboring elements Index loop

Summary

  • Index loops connect repetition directly to valid positions.
  • Range-based loops visit full capacity; index loops can stop at the logical data count.

Inputting and Displaying Array Data

From User Input to Stored Data

  • How can several inputs remain available after they are entered?

Slides 54–67

Inputting One Element

cin >> scores[0];
  • The subscripted element is the destination for the input value.
  • Later input can target a different index without replacing this value.

The Fill Loop

step selected destination
1 scores[0]
2 scores[1]
3 scores[2]
… …
final scores[SIZE - 1]
  • The loop index chooses where each new value is stored.

Fill Loop Code

for (int i = 0; i < SIZE; i++)
{
    cout << "Enter score " << i + 1 << ": ";
    cin >> scores[i];
}

Indexes and User Labels

meaning expression first value
array index i 0
displayed item number i + 1 1
  • Keep zero-based indexing inside the program.
  • Present one-based numbering when it is clearer for the user.

The Display Loop

  • Input stores values during one traversal.
  • A later traversal reads the stored values.
  • The second pass can use different formatting or calculations.

Display Loop Code

for (int i = 0; i < SIZE; i++)
{
    cout << "Score " << i + 1
         << ": " << scores[i] << '\n';
}

The Two-Pass Pattern

input pass                       display pass

user -> scores[0]                scores[0] -> output
user -> scores[1]                scores[1] -> output
user -> scores[2]       then     scores[2] -> output
user -> ...                      ...
  • Storage separates when data is collected from when it is used.

Partially Filled Arrays

const int CAPACITY = 10;
int scores[CAPACITY] = {};
int dataCount = 4;
  • Capacity remains ten.
  • Only indexes 0 through dataCount - 1 contain user data.

Choosing the Data Count

do
{
    cout << "How many scores (0-" << CAPACITY << ")? ";
    cin >> dataCount;
}
while (dataCount < 0 || dataCount > CAPACITY);
  • Validation makes the requested count safe to use as a loop bound.

Stopping at Capacity

for (int i = 0; i < dataCount; i++)
{
    cin >> scores[i];
}

for (int i = 0; i < dataCount; i++)
{
    cout << scores[i] << '\n';
}
  • Both loops use the same validated logical count.

Complete Program

#include <iostream>
using namespace std;

int main()
{
    const int CAPACITY = 10;
    int scores[CAPACITY] = {};
    int dataCount;

    do
    {
        cout << "How many scores (0-10)? ";
        cin >> dataCount;
    }
    while (dataCount < 0 || dataCount > CAPACITY);

    for (int i = 0; i < dataCount; i++)
    {
        cout << "Enter score " << i + 1 << ": ";
        cin >> scores[i];
    }

    for (int i = 0; i < dataCount; i++)
    {
        cout << scores[i] << '\n';
    }

    return 0;
}

Summary

  • A fill loop writes each input to a selected element.
  • A later display loop uses the same validated count to read only meaningful data.

Arrays vs. Vectors

Same Collection, Different Contract

  • Which container matches what the program needs to do?

Slides 68–82

Declarations Side by Side

#include <vector>
using namespace std;

const int SIZE = 5;

int scoresArray[SIZE] = {91, 84, 78, 96, 88};
vector<int> scoresVector = {91, 84, 78, 96, 88};

Fixed and Dynamic Capacity

Array Vector
Capacity is part of the declaration Size can change while running
Cannot grow or shrink Can grow or shrink
Capacity must be known at compile time Initial size may come from runtime data

Adding and Removing Values

vector<int> scores;
scores.push_back(91);
scores.pop_back();
  • A raw array has no push_back() or pop_back().
  • The program must manage which fixed positions contain meaningful values.

Knowing the Size

cout << scoresVector.size() << '\n';

cout << SIZE << '\n';

int count = sizeof(scoresArray) / sizeof(scoresArray[0]);
  • A vector reports its current size.
  • An array usually relies on a named capacity; the sizeof formula is local to the actual array.

Element Access

cout << scoresArray[2] << '\n';
cout << scoresVector[2] << '\n';
  • Both containers use zero-based bracket indexing.
  • Both bracket operators assume the index is valid.

Bounds Safety

cout << scoresVector.at(2) << '\n';
Access Bounds checked?
raw array [] no
vector [] no
vector .at() yes
  • A raw array has no .at() member function.

Whole-Collection Operations

vector<int> first = {1, 2, 3};
vector<int> second;

second = first;
bool same = (second == first);
  • Vectors support whole-collection assignment and comparison.
  • Raw arrays require element-by-element copying and comparison.

Passing Collections to Functions

Vector parameter Raw-array parameter
Retains vector operations Does not carry its capacity
Can report .size() Usually needs a separate size argument
Commonly passed by reference Has special parameter behavior
  • Array parameter syntax and decay are next lecture’s topic.

Partially Filled Storage

const int CAPACITY = 10;
int arrayValues[CAPACITY] = {};
int dataCount = 4;

vector<int> vectorValues = {8, 15, 16, 23};
  • The array needs a separate logical count.
  • The vector’s size already equals its current element count.

Choose an Array

  • Capacity is small, fixed, and known at compile time.
  • Predictable storage is a real design requirement.
  • The code must interact with an interface that expects a raw array.

Choose a Vector

  • The number of values can change.
  • The size comes from input or another runtime source.
  • Built-in size tracking, checked access, copying, or comparison improves the design.

Decision Framework

  1. Can the number of values change? Use a vector.
  2. Is the size known only while running? Use a vector.
  3. Do collection operations or size tracking matter? Use a vector.
  4. Is fixed capacity an intentional requirement? Consider an array.

Summary

  • Fixed capacity is the defining array contract.
  • Vectors provide dynamic size and a richer interface, so choose from program requirements rather than habit.

Performance — Arrays vs. Vectors

Are Arrays Faster?

  • The honest answer depends on which work is being measured.

Slides 83–96

Contiguous Storage

Array elements Vector elements
Adjacent in memory Adjacent in memory
Efficient sequential traversal Efficient sequential traversal
Good cache locality Good cache locality
  • Both containers provide contiguous element storage.

Element Access Cost

arrayValues[i]
vectorValues[i]
  • With valid indexes, both bracket expressions locate an element directly.
  • Equivalent optimized loops commonly produce equivalent machine work.

Where the Difference Begins

  • A typical local raw array stores its elements with the function’s automatic data.
  • A vector object tracks its element storage, size, and capacity.
  • The vector normally obtains its element storage through dynamic allocation.

“Stack” and “heap” are common implementation terms, not storage guarantees imposed by the C++ language standard.

Allocation and Growth

  • Creating vector storage may require a dynamic allocation.
  • Growing beyond capacity may allocate a larger block and move existing elements.
  • A fixed raw array never reallocates because its capacity never changes.

Fixed-Size Vectors

int dataCount;
cin >> dataCount;

vector<int> values(dataCount);
  • Constructing the vector at its final size performs one initial allocation.
  • It avoids repeated growth while retaining runtime sizing.

Large Collections

  • Automatic storage is usually limited.
  • A very large local raw array can exhaust that space.
  • A vector’s dynamically allocated element storage is often the safer design for large collections.

What Optimizers Remove

  • Release builds remove much loop and abstraction overhead.
  • Simple array and vector traversals may compile to nearly identical instructions.
  • Debug-build timing does not predict optimized performance.

Benchmarking the Right Work

  • Time construction separately from traversal.
  • Repeat enough work to reduce timer noise.
  • Use optimized builds and realistic data sizes.
  • Prevent unused results from being optimized away.

Equivalent Work

Unfair comparison Equivalent comparison
uninitialized array vs. zero-initialized vector initialize both the same way
array [] vs. vector .at() compare [] with []
fixed array vs. repeatedly growing vector give both final storage once

When Performance Matters

  • Embedded systems with strict memory limits.
  • Real-time systems with allocation restrictions.
  • Measured hot paths that dominate total runtime.
  • Interfaces that require fixed raw storage.

Choose for Design First

  • Begin with the container whose size and safety contract fits the problem.
  • Measure the real program before trading clarity or flexibility for assumed speed.
  • Optimize only the cost that measurements identify.

Summary

  • Arrays are not universally faster than vectors.
  • Allocation and growth create the main difference; equivalent access and traversal often cost the same after optimization.

Key Takeaways

  • Raw arrays store a fixed number of same-type elements.
  • Indexes begin at zero and must remain between 0 and SIZE - 1.
  • Index loops expose positions; range-based loops visit every declared element.
  • A separate count tracks meaningful data in a partially filled array.
  • Vectors remain the default when size changes or collection operations matter.
