Processing Array Contents

Same loop patterns — now with arrays

  • Sum, min, and max are accumulator and tracker patterns you already know
  • The only change is the data source: an array instead of a vector

Total slides: 8

Sum — the accumulator pattern

const int SIZE = 5;
int scores[SIZE] = {91, 78, 84, 95, 62};

int total = 0;                           //.......... initialize to zero

for (int i = 0; i < SIZE; i++) {
    total += scores[i];                  //.......... add each element
}

std::cout << "Total: " << total << "\n"; //.......... 410

The accumulator pattern is identical to what students used with vectors. Emphasize that the data structure changes but the reasoning does not.

Max

int max = scores[0];                     //.......... seed with first element

for (int i = 1; i < SIZE; i++) {         //.......... start at 1, already have 0
    if (scores[i] > max) {
        max = scores[i];
    }
}

std::cout << "Max: " << max << "\n";     //.......... 95

Seeding with the first element (not zero or INT_MIN) is correct for any value range, including all-negative inputs. Students saw this pattern in 09.02 state tracking patterns.

Min — same structure, flipped comparison

int min = scores[0];                     //.......... seed with first element

for (int i = 1; i < SIZE; i++) {
    if (scores[i] < min) {               //.......... flipped: < instead of >
        min = scores[i];
    }
}

std::cout << "Min: " << min << "\n";     //.......... 62

Average — sum divided by count

int total = 0;

for (int i = 0; i < SIZE; i++) {
    total += scores[i];
}

double average = static_cast<double>(total) / SIZE;  //.......... avoid int division
std::cout << "Average: " << average << "\n";         //.......... 82.0

static_cast here is essential — without it, total / SIZE is integer division and truncates. Students covered type casting in 09.01.

Combining into one pass

int total = 0;
int max   = scores[0];
int min   = scores[0];

for (int i = 0; i < SIZE; i++) {
    total += scores[i];                  //.......... accumulate
    if (scores[i] > max) max = scores[i]; //.......... track max
    if (scores[i] < min) min = scores[i]; //.......... track min
}

One pass is more efficient and is the natural form when you need multiple statistics at once. Worth showing after students have seen each pattern separately so they understand what’s being combined.

Summary

  • Sum uses an accumulator initialized to zero; min and max use a tracker seeded with the first element
  • These are the same patterns from loop work — the array index just replaces the range-for variable

Linear Search

Finding a value — without any shortcuts

  • Linear search checks each element in order until it finds a match or runs out of elements
  • No sorting required; works on any array

Total slides: 6

The basic search loop

const int SIZE = 5;
int scores[SIZE] = {91, 78, 84, 95, 62};
int target = 84;

for (int i = 0; i < SIZE; i++) {
    if (scores[i] == target) {           //.......... found it
        std::cout << "Found at index " << i << "\n";
    }
}

This naive version keeps looping after a match — it will report every occurrence. The next slide shows how to stop early once a match is found.

Stopping early with a flag

bool found = false;
int  foundAt = -1;                       //.......... -1 signals "not found"

for (int i = 0; i < SIZE; i++) {
    if (scores[i] == target) {
        found   = true;
        foundAt = i;
        break;                           //.......... stop at first match
    }
}

if (found) {
    std::cout << "Found at index " << foundAt << "\n";
} else {
    std::cout << "Not found\n";
}

Returning the index from a function

int linearSearch(int arr[], int size, int target) {
    for (int i = 0; i < size; i++) {
        if (arr[i] == target) {
            return i;                    //.......... return index on match
        }
    }
    return -1;                           //.......... -1 means not found
}

-1 as a sentinel return value is a C++ convention for “not found” with index-returning functions. Students will see this pattern again with string’s find() method later in the course.

Performance — how slow is slow?

  • Linear search inspects at most N elements for an array of size N
  • Best case: target is at index 0 — one comparison
  • Worst case: target is at the last index or not present — N comparisons
  • For small arrays this is fine; for large sorted data, binary search is faster

Keep this brief — the point is that students understand the tradeoff exists, not that they implement binary search today.

Summary

  • Linear search walks the array from index 0, checking each element; -1 as the return value signals “not found”
  • Use break or return to stop at the first match rather than scanning the rest of the array

Sorting — Bubble Sort

Putting an array in order

  • Sorting rearranges elements so each one is in the right position relative to its neighbors
  • Bubble sort is the simplest algorithm to read and trace

Total slides: 8

The core idea

[Graphic suggestion: a row of five boxes with values {5, 3, 8, 1, 4}. Draw arrows between adjacent pairs being compared, then show the larger value “bubbling” one step to the right after each swap.]

  • Compare two adjacent elements; if they are out of order, swap them
  • Repeat across the whole array — the largest unsorted value “bubbles” to its correct position each pass
  • After N−1 passes, the array is sorted

Swapping two elements

int a = scores[i];
int b = scores[i + 1];

if (a > b) {
    int temp   = scores[i];      //.......... hold a's value
    scores[i]  = scores[i + 1];  //.......... overwrite a with b
    scores[i + 1] = temp;        //.......... put a's original value into b's slot
}

The temp variable is essential. Without it, overwriting scores[i] destroys a’s value before it can be moved. Students commonly try to skip temp — worth demonstrating the failure.

One pass through the array

for (int i = 0; i < SIZE - 1; i++) {        //.......... SIZE-1 pairs
    if (scores[i] > scores[i + 1]) {
        int temp       = scores[i];
        scores[i]      = scores[i + 1];
        scores[i + 1]  = temp;
    }
}

After one pass, the largest element is guaranteed to be at the last index. This is the “one bubble rises” mental model.

Full bubble sort — nested loops

for (int pass = 0; pass < SIZE - 1; pass++) {       //.......... N-1 passes
    for (int i = 0; i < SIZE - 1 - pass; i++) {     //.......... shrink inner bound
        if (scores[i] > scores[i + 1]) {
            int temp       = scores[i];
            scores[i]      = scores[i + 1];
            scores[i + 1]  = temp;
        }
    }
}

SIZE - 1 - pass: after each pass, the last pass elements are already in their final position, so the inner loop doesn’t need to visit them. This is the standard optimization that avoids re-comparing sorted elements.

Tracing a sort — before and after

Before

Index Value
0 62
1 95
2 78
3 91
4 84

After

Index Value
0 62
1 78
2 84
3 91
4 95

Summary

  • Bubble sort repeatedly compares adjacent pairs and swaps them if out of order; after N−1 passes the array is sorted
  • Always use a temp variable when swapping — overwriting an element without saving its value first destroys data

typedef

Giving a type a shorter name

  • typedef creates an alias — a new name for an existing type
  • Same type, different label

Total slides: 6

Basic syntax

typedef existing_type alias_name;
typedef unsigned int uint;     //.......... uint now means unsigned int
typedef double       Dollars;  //.......... Dollars now means double

uint count = 0;
Dollars price = 9.99;

The alias is just a name — it does not create a new type. The compiler treats uint and unsigned int as identical.

typedef with arrays

typedef int ScoreArray[5];     //.......... ScoreArray is an array of 5 ints

ScoreArray scores = {91, 78, 84, 95, 62};
  • The alias captures both the element type and the size
  • Useful when the same array shape appears repeatedly in a program

typedef with vectors

#include <vector>
typedef std::vector<int> IntVec;   //.......... IntVec is a vector of ints

IntVec scores = {91, 78, 84, 95, 62};
scores.push_back(73);
  • Particularly useful for long template types that appear many times
  • The alias does not change the type’s behavior — all vector methods still work

using — the modern alternative

using uint     = unsigned int;
using Dollars  = double;
using IntVec   = std::vector<int>;
  • using does the same job as typedef and is preferred in modern C++
  • Easier to read: alias on the left, type on the right
  • You will see both in existing code — know how to read them

Summary

  • typedef creates an alias for a type; using is the modern equivalent with the same effect
  • Use aliases to shorten verbose types and to give domain meaning to generic types like double or vector<int>

Arrays as Function Arguments

Passing an array to a function — not what you expect

  • Arrays do not behave like int or double when passed to a function
  • Understanding why changes how you write every array function

Total slides: 7

Array decay

int scores[5] = {91, 78, 84, 95, 62};

void printFirst(int arr[]) {
    std::cout << arr[0];         //.......... works fine
}

printFirst(scores);              //.......... passes a pointer, not a copy
  • When you pass an array, C++ converts it to a pointer to the first element
  • The function receives an address, not a copy of the data
  • This is called array decay

[Graphic suggestion: two boxes — left shows scores[5] on the stack with five elements; right shows arr inside the function as a single pointer arrow pointing to scores[0]. Emphasize that no copy is made.]

You must pass the size separately

void printArray(int arr[], int size) {    //.......... size is a required parameter
    for (int i = 0; i < size; i++) {
        std::cout << arr[i] << " ";
    }
}

printArray(scores, 5);
  • Once the array decays to a pointer, sizeof no longer knows the original size
  • Always pass size as a second parameter — there is no other way to know it

Functions can modify the original array

void doubleAll(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        arr[i] *= 2;                      //.......... modifies the caller's array
    }
}

doubleAll(scores, 5);                     //.......... scores is changed
  • Because the function receives a pointer, any write to arr[i] changes the original
  • There is no copy to protect you — modifications are permanent

Use const to prevent modification

void printArray(const int arr[], int size) {   //.......... read-only
    for (int i = 0; i < size; i++) {
        std::cout << arr[i] << " ";
    }
    // arr[0] = 0;  // compile error — const prevents writes
}
  • const signals intent: this function will not modify the array
  • The compiler enforces it — any write attempt is a compile error
  • Use const on any parameter that should not change

Putting it together

int  sumArray   (const int arr[], int size);   //.......... read-only
void sortArray  (      int arr[], int size);   //.......... modifies in place
int  searchArray(const int arr[], int size, int target);
  • const for read-only operations; no const when the function must write
  • The pattern is the same for all three: array pointer + size

Summary

  • Arrays passed to functions decay to a pointer — the function receives an address, not a copy
  • Always pass size as a separate parameter; use const to prevent accidental modification

sizeof with Arrays

How many elements are in this array?

  • sizeof tells you the total bytes — not the element count
  • One formula bridges the gap

Total slides: 5

sizeof on a single element vs. the whole array

int scores[5] = {91, 78, 84, 95, 62};

sizeof(scores[0]);   //.......... 4 — bytes in one int
sizeof(scores);      //.......... 20 — bytes in the whole array (5 × 4)
  • sizeof(array) returns the total byte count of all elements combined
  • sizeof(element) returns the byte count of one element

sizeof is evaluated at compile time — it is not a function call. The parentheses are required but it is an operator.

Computing element count

int scores[5];

int count = sizeof(scores) / sizeof(scores[0]);  //.......... 20 / 4 = 5
  • Total bytes divided by bytes per element gives the number of elements
  • This works for any element type — no magic numbers needed

The catch — arrays passed to functions

void printSize(int arr[]) {
    int count = sizeof(arr) / sizeof(arr[0]);  //.......... WRONG — sizeof(arr) is 8
}
  • When an array is passed to a function it decays to a pointer
  • sizeof on a pointer gives the pointer size (8 bytes on 64-bit), not the array size
  • Always pass the size as a separate parameter — do not rely on sizeof inside a function

This is a preview of the next topic (arrays as function arguments). Plant the issue here so students are primed when the full explanation arrives.

Summary

  • sizeof(array) / sizeof(array[0]) gives the element count — but only where the array is declared
  • Inside a function, the array decays to a pointer and sizeof no longer reflects the original size

Key Takeaways

  • Accumulator and tracker patterns from loop work apply directly to arrays — same logic, indexed access
  • Linear search returns -1 for “not found”; bubble sort requires a temp variable to swap safely
  • sizeof(array) / sizeof(array[0]) gives element count only where the array is declared — not inside functions
  • Arrays decay to pointers when passed to functions; always pass size separately and use const for read-only parameters
