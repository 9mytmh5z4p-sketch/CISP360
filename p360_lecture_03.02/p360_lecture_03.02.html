Vectors

One Variable, Many Values

string employee1 = "Maya";
string employee2 = "Ari";
string employee3 = "Chen";
string employee4 = "Jordan";
string employee5 = "Sam";
  • What would we have to do to process all five employees?
  • What would we have to change to add a new employee?

Slides 3-11

The Problem

  • We can access the content of the employee variables, but we can’t access those variables specifically through the program.

  • What we need is a way to structure the data so we can access the variables as well as the associated values - a data structure.

Why This Is a Data Structure

  • A data structure organizes data so a program can work with it.

  • C++ has a frequently used data structure called a vector.

  • A vector organizes values in order, so the program can add, access, and process them as a group.

A Vector IPO Model

Input Processing Output
three scores store scores in a vector printed score list

The IPO model helps identify why the vector exists before we write the code.

Declaring a Vector

Vectors require the <vector> header.

#include <iostream>
#include <vector>
using namespace std;

vector<int> scores;

The type inside < > tells the vector what kind of values it stores.

Initializing a Vector

A vector can start with values already inside it.

vector<int> scores = {88, 92, 75};
vector<string> names = {"Maya", "Chen", "Ari"};

All values in one vector must have the same type.

Vector Order Picture

Position first second third
Value 88 92 75

The values stay in order, so the vector can process them as one group.

Summary

  • A vector stores many related values using one variable name.
  • Vectors are ordered, typed, and designed for grouped data.
  • This is our first data structure.

Vector Operations

Changing a Vector

  • How does a vector grow, shrink, and report what it contains?
  • Vector operations change or inspect the container.

Slides 12-27

push_back Adds a Value

push_back adds one value to the end of the vector.

vector<int> scores;

scores.push_back(88);
scores.push_back(92);

The vector now stores 88, then 92.

Execution Trace: push_back

Statement scores
vector<int> scores; { }
scores.push_back(88); {88}
scores.push_back(92); {88, 92}
scores.push_back(75); {88, 92, 75}

Each call adds one new value at the end.

pop_back Removes the Last Value

pop_back removes the final value from the vector.

vector<int> scores = {88, 92, 75};

scores.pop_back();

After this statement, the vector stores 88, then 92.

Interactive Vector Changes

vector<int> scores = { };

size Reports the Count

size reports how many values are currently stored.

vector<int> scores = {88, 92, 75};

cout << scores.size() << endl;

This prints 3.

Save Before Removing

If the last value matters, save it before removing it.

vector<int> scores = {88, 92, 75};

int droppedScore = scores.back();
scores.pop_back();

cout << "Removed: " << droppedScore << endl;

After pop_back, the vector no longer stores 75.

Example 1: Daily Temperatures Code

#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<double> temperatures;

    temperatures.push_back(68.5);
    temperatures.push_back(72.0);
    temperatures.push_back(70.5);

    cout << "Days recorded: " << temperatures.size() << endl;
    return 0;
}

Example 2: Loading Scalar Variables

#include <iostream>
#include <vector>
#include <string>
using namespace std;

int main() {
    string firstItem = "eggs";
    string secondItem = "rice";
    string thirdItem = "apples";
    vector<string> items;

    items.push_back(firstItem);
    items.push_back(secondItem);
    items.push_back(thirdItem);

    cout << "Items loaded: " << items.size() << endl;
    return 0;
}

Example 3: Storing Formula Results

#include <iostream>
#include <vector>
using namespace std;

int main() {
    double hourlyRate = 18.50;
    double mondayHours = 6.0;
    double tuesdayHours = 7.5;
    vector<double> dailyPay;

    dailyPay.push_back(hourlyRate * mondayHours);
    dailyPay.push_back(hourlyRate * tuesdayHours);

    cout << "Paychecks stored: " << dailyPay.size() << endl;
    return 0;
}

Operation Summary Table

Operation What it does
push_back(value) adds a value to the end
pop_back() removes the last value
size() reports the current count
back() reads the last value

Common Practices

  • Use plural names for vectors, such as scores or items.
  • Check that a vector has values before reading from it.
  • Use push_back when the program discovers values one at a time.

Summary

  • Vector operations can add, remove, count, or read values.
  • push_back and pop_back change the vector.
  • size and back help inspect what the vector contains.

Range-Based For Loops

When Every Value Matters

  • How can a program use every value in a vector?
  • A range-based for loop visits each value without writing position numbers.

Slides 28-39

The Range-Based For Loop

The range-based for loop has this shape:

for (int score : scores) {
    cout << score << endl;
}

The loop body runs once for each value in scores.

Read It Like English

for (int score : scores)

Read this as:

for each score in scores

The vector name is plural; the loop variable name is singular.

Display Every Value

#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> scores = {88, 92, 75};

    for (int score : scores) {
        cout << score << endl;
    }

    return 0;
}

Execution Trace: For Each

Trip score Output
1 88 88
2 92 92
3 75 75

The loop variable receives one value at a time.

The Loop Variable Is a Copy

In this form, the loop variable stores a copy of each vector value.

for (int score : scores) {
    score = 100;
}

Changing score does not change the original vector.

One Value Per Trip

The loop body runs once for each value.

If the vector has three values, the body runs three times.

If the vector has five values, the body runs five times.

Example: Print a Shopping List

#include <iostream>
#include <vector>
#include <string>
using namespace std;

int main() {
    vector<string> items = {"eggs", "rice", "apples"};

    for (string item : items) {
        cout << "- " << item << endl;
    }

    return 0;
}

Loop Body Scope

int count = 0;
vector<int> scores = {88, 92};

for (int score : scores) {
    count = count + 1;
    cout << score << endl;
}

cout << count << endl;
// cout << score << endl;  // score is not available here

The loop body creates a new level of scope.

Common Practices

  • Use a plural name for the vector and a singular name for the loop variable.
  • Use range-based for when every value should be read.
  • Keep the loop body focused on one repeated action.

Summary

  • A range-based for loop visits every value in a vector.
  • The loop variable receives one value per trip.
  • The loop body has its own scope.

Key Takeaways

  • A vector stores many related values of the same type under one name.
  • push_back, pop_back, and size change or inspect the vector.
  • Range-based for is the beginner-friendly way to read every value.
  • IPO models and execution traces help explain why the vector exists and how it changes.

  • Choose the vector element type before choosing the variable name.
  • Use vectors when values belong together as one ordered group.

  • Mixing different value types in one vector.
  • Treating the vector as a single value instead of a container.

Range-Based For Loops: Good Practices

  • Use range-based for when reading every value.
  • Give the vector a plural name and the loop variable a singular name.
  • Keep the repeated action inside the loop body.

Range-Based For Loops: Common Mistakes

  • Forgetting the colon between the loop variable and the vector.
  • Expecting the loop variable to exist after the loop ends.
  • Expecting changes to the copied loop variable to change the vector.

Range-Based For Loops: More Examples

    cout << "$" << price << endl;
}
