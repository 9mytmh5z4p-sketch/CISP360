=== Code Exited With Errors ===

C++ did not recognize the variable, but we declared it!

  • myInt was declared but it was a different level of scope than the cout statement.
  • Scope can possess a hierarchy.
  • The level of scope inside the for each loop is a lower level than the scope inside of main().
  • To access a variable, the level of scope must be the same or lower than the level upon which the variable was declared.

We are using the variable at a higher level of scope than what it was declared at. The compiler does not recognize the variable.

    int index = 0;

        index++;

    }

    return 0;

The Fix

  • The rule is the scope can be at the same level or lower.
  • To implement that here, we would simply move the declaration int myInt =125; out of the for-each loop and into main().
  • That would make it at the same level where we are using it in the cout statement and that would eliminate our problems.

Why Do We Have Scope?

Scope serves several purposes:

  1. Prevents Name Conflicts
  2. Memory Management.
  3. Fosters Encapsulation and Information Hiding
  4. Reduce Bugs
  5. Makes code more readable

Prevents Name Conflicts

Without scope, you could only use each variable name once in your entire program:

int count = 0;  // counting users
// ...1000 lines later...
int count = 0;  // ERROR! want to count something else, but name is taken

With scope, you can reuse names in different contexts without collision.

Memory Management

Variables are automatically destroyed when they go out of scope:

for (int i = 0; i < 1000; i++) {
    string tempData = loadLargeFile();  // memory allocated
    processData(tempData);
}  // tempData destroyed here, memory freed automatically

Without scope, you’d have to manually track and delete every variable, leading to memory leaks.

Fosters Encapsulation and Information Hiding

Scope lets you hide implementation details:

void calculateTotal() {
    int subtotal = 100;  // internal calculation
    int tax = 10;        // nobody outside needs to see this
    return subtotal + tax;
}
// subtotal and tax don't clutter the outer namespace

This means there is less we need to remember. It lightens our cognitive load.

Reduce Bugs

Scope limits where variables can be accidentally modified:

int userID = 5;

for (int i = 0; i < 10; i++) {
    int result = userID * i;
}

cout << result << endl;  // ERROR: result doesn't exist here

We can’t accidentally use result outside of the loop.

Makes code more readable

Scope tells you where to look for variable definitions. If you see a variable used in a function, you know it’s either declared in that function, passed as a parameter, or is a global—you don’t have to search the entire codebase.

In Summary

Scope helps organize code, manage memory efficiently, prevent bugs, and make programs easier to understand and maintain.

Dates & Random Numbers in C++

Working with Dates and Times

Sometimes we need to get the current date and/or time.

C++ provides functionality to work with dates and times through the <ctime> library.

Getting the Current Date and Time

// dateTime.cpp -- date & time example
#include <iostream>
using namespace std;

int main() {
    time_t now = time(nullptr);
    cout << asctime(localtime(&now)) << endl;
    
    return 0;
}

Output:

Wed Feb  1 18:19:01 2023

Understanding the Date/Time Components

time_t is a data type that represents time as a number (typically seconds since January 1, 1970).

time(nullptr) gets the current time and stores it in a time_t variable.

localtime(&now) converts the time_t value to a structure containing the broken-down local time (year, month, day, hour, minute, second).

asctime() converts the time structure into a human-readable string format.

Practical Uses

Date and Time

  • Logging and timestamps
  • Scheduling and timing events
  • Calculating elapsed time
  • Displaying current date/time to users

Random Numbers

Sometimes we need to generate random numbers in our programs. C++ provides several ways to accomplish this, ranging from simple to more sophisticated approaches.

Basic Random Number Generation

The simplest way to generate random numbers in C++ is using the rand() function from the <cstdlib> library.

A Simple Example

Type this in.

//3-30.cpp -- random numbers
// generates the same numbers
#include <iostream>
#include <cstdlib>     // for rand
using namespace std;

int main() {
    cout << rand() << "\t";
    cout << rand() << "\t";
    cout << rand() << endl;
    
    return 0;
}

Rerun and examine the output. What do you notice about the numbers?

Problem: This program will generate the same sequence of “random” numbers every time you run it!

Why? Because the starting value of the random number generator or seed, does not change. This leads to the same result every time it’s run.

To get different random numbers each time the program runs, we need to “seed” the random number generator with a different number using srand().

Using Time as a Seed

//3-32.cpp -- useful random numbers
// generates the same numbers
#include <iostream>
#include <cstdlib>     // for rand
#include <ctime>       // for time
using namespace std;

int main() {
    unsigned seed;
    // seed = 100;              // for debugging
    seed = time(0);             // different random numbers
    srand(seed);                // randomize
    
    cout << rand() << "\t";
    
    // 6 sided die
    cout << rand() % 6 + 1 << "\t";
    
    // random number between 8 and 12
    cout << (rand() % (12 - 8 + 1)) + 8 << endl;
    
    return 0;
}

Type this in and run. We use the seed for debugging.

Understanding the Seeding Process

time(0) returns the current time in seconds since January 1, 1970 (Unix epoch). Since this value changes every second, it provides a different seed value each time you run the program.

srand(seed) initializes the random number generator with the seed value, ensuring different sequences of random numbers.

Generating Random Numbers in Specific Ranges

Simulating a Six-Sided Die

To generate a random number between 1 and 6:

rand() % 6 + 1

How it works: - rand() % 6 produces values from 0 to 5 - Adding 1 shifts the range to 1 to 6

Random Numbers in Any Range

To generate a random number between a minimum and maximum value:

(rand() % (max - min + 1)) + min

Example: For a random number between 8 and 12:

(rand() % (12 - 8 + 1)) + 8

This formula works because: - 12 - 8 + 1 = 5 (there are 5 possible values: 8, 9, 10, 11, 12) - rand() % 5 produces 0, 1, 2, 3, or 4 - Adding 8 shifts these to 8, 9, 10, 11, or 12

Practical Uses

Random Numbers

  • Game development (dice rolls, random events)
  • Simulations and modeling
  • Generating test data
  • Cryptographic applications (though more sophisticated methods are needed for security)

Important Notes

Pseudo-random vs. True Random: The rand() function generates pseudo-random numbers. They appear random but follow a deterministic sequence based on the seed. For most applications, this is sufficient.

Better Alternatives: Modern C++ (C++11 and later) provides more sophisticated random number generation through the <random> library, which offers better statistical properties and more control over distributions.

Terminal Color Codes in C++

Introduction to Terminal Colors

Making your console programs more visually appealing is easy with terminal color codes! By using special escape sequences, you can change the color of text displayed in the terminal, making your output more readable and engaging.

What Are ANSI Escape Codes?

ANSI escape codes are special character sequences that tell the terminal to change text attributes like color, background, and style. These codes work on most modern terminals, including Linux, macOS, and Windows (Windows 10 and later).

Basic Color Implementation

A Simple Color Example

Here’s a complete program that demonstrates how to use color codes:

// simCol.cpp -- simple terminal color example
#include <iostream>
#include <string>
using namespace std;

int main() {
    const string RED = "\033[031m;";
    const string YELLOW = "\033[033m;";
    const string NORM = "\033[0m";
    
    cout << RED << "Hello" << NORM << endl;
    cout << YELLOW << "there" << NORM << endl;
    
    return 0;
}

Output:

./main
;Hello
;there

The text “Hello” appears in red, and “there” appears in yellow!

Understanding the Color Code Format

Each color code follows this pattern:

\033[COLORm

Breaking it down: - \033 is the escape character (ESC in ASCII) - [ starts the color sequence - COLOR is the color code number - m ends the sequence

Common Color Codes

Here are the most frequently used foreground (text) color codes:

  • \033[030m - Black
  • \033[031m - Red
  • \033[032m - Green
  • \033[033m - Yellow
  • \033[034m - Blue
  • \033[035m - Magenta
  • \033[036m - Cyan
  • \033[037m - White

The Importance of NORM (Reset)

The NORM constant is crucial:

const string NORM = "\033[0m";

\033[0m resets all text attributes back to the terminal’s default. Always use this after colored text to prevent all subsequent output from being colored!

Practical Usage Tips

Storing Colors as Constants

It’s best practice to define color codes as named constants at the beginning of your program:

const string RED = "\033[031m";
const string GREEN = "\033[032m";
const string BLUE = "\033[034m";
const string NORM = "\033[0m";

This approach offers several benefits: - Makes your code more readable - Easy to modify colors throughout your program - Prevents typos in color codes

Using Colors in Output

To color your text, simply insert the color code before the text and reset after:

cout << RED << "Error: File not found!" << NORM << endl;
cout << GREEN << "Success! Operation completed." << NORM << endl;
cout << YELLOW << "Warning: Low disk space." << NORM << endl;

Best Practices

Always reset colors: Remember to use NORM after colored text to avoid coloring unintended output.

Use colors meaningfully: - Red for errors or important warnings - Green for success messages - Yellow for caution or warnings - Blue for informational messages

Test on your target terminal: Not all terminals support the same color codes, so test your program on the systems where it will run.

Finding More Information

If you want to explore more color options and text effects, you can search online for:

“How do I output colored text to a Linux terminal?”

This will lead you to resources about: - Background colors - Bold and italic text - Underlined text - Bright/bold color variants - 256-color and true color support

Additional Color Codes to Explore

Bright/Bold Colors: Add 60 to the color code or use the bold attribute: - \033[091m - Bright Red - \033[092m - Bright Green - \033[093m - Bright Yellow

Background Colors: Use codes 40-47 for background colors: - \033[041m - Red background - \033[042m - Green background - \033[044m - Blue background

Text Styles: - \033[1m - Bold - \033[4m - Underline - \033[7m - Reverse (swap foreground and background)

#include <iostream>
#include <string>
#include <vector>
using namespace std;

int main() {
    // Define color codes as const strings
    const string NORM = "\033[0m";
    const string RED = "\033[031m";
    const string GREEN = "\033[032m";
    const string YELLOW = "\033[033m";
    const string BLUE = "\033[034m";
    const string MAGENTA = "\033[035m";
    const string CYAN = "\033[036m";
    const string WHITE = "\033[037m";
    
    // Store them in a vector
    vector<string> colors = {
        NORM,    // 0
        RED,     // 1
        GREEN,   // 2
        YELLOW,  // 3
        BLUE,    // 4
        MAGENTA, // 5
        CYAN,    // 6
        WHITE    // 7
    };
    
    // Now you can use either the named constants OR the vector
    // This gives you the best of both worlds!
    cout << RED << "Using the constant directly" << NORM << endl;
    cout << colors[1] << "Using the vector" << colors[0] << endl;
    
    int i = 0;
    for (auto COLOR:colors)
        cout << COLOR << i++ << NORM << endl;
    
    return 0;
}
