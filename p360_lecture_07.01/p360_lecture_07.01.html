Checking if a Vector is Empty

How can we determine whether a vector contains any elements before trying to access or process its data? Today we’ll learn what empty vectors are and explore two methods for checking if a vector is empty.

Total slides: 6

Consider including a visual showing an empty vector container versus one with elements.

What is an Empty Vector?

An empty vector is a vector that contains zero elements. It is a valid, initialized container that exists in memory but currently holds no data.

Empty vectors commonly occur when you create a new vector without initial values, after removing all elements, or when no data has been added yet. Checking if a vector is empty prevents errors that occur when trying to access elements that don’t exist.

Understanding empty vectors is crucial for input validation, loop termination conditions, and ensuring your program handles edge cases correctly. Attempting to access elements in an empty vector using bracket notation causes undefined behavior.

Consider a diagram showing an empty vector box with “size: 0” label versus a vector with elements.

Using the empty Method

The .empty() method returns a boolean value indicating whether the vector contains zero elements. This is the most direct and readable way to check for an empty vector.

vector<int> numbers;

if (numbers.empty()) {
    cout << "Vector is empty" << endl;
} else {
    cout << "Vector has elements" << endl;
}

vector<string> names = {"Alice", "Bob"};
if (!names.empty()) {
    cout << "First name: " << names[0] << endl;
}

The method returns true when the vector has no elements and false when it contains one or more elements. Use .empty() before accessing elements to avoid runtime errors.

Using size to Check for Empty Vectors

Checking if .size() equals zero is another way to determine if a vector is empty. This method explicitly compares the number of elements to zero.

vector<double> prices;

if (prices.size() == 0) {
    cout << "No prices available" << endl;
}

vector<char> letters = {'a', 'b', 'c'};
if (letters.size() != 0) {
    cout << "Vector contains " << letters.size() << " elements" << endl;
}

The .size() method returns the number of elements in the vector. Comparing this value to zero achieves the same result as .empty() but is less direct.

Comparing the Two Approaches

Both .empty() and .size() == 0 correctly determine if a vector is empty, but they differ in clarity and intent. The .empty() method explicitly states the question being asked and is more readable.

Use .empty() when you only need to know whether the vector has elements or not. Use .size() when you need the actual count of elements for other purposes, such as displaying how many items exist.

vector<int> data = {5, 10, 15};

// Preferred when only checking emptiness
if (data.empty()) { /* ... */ }

// Better when you need the size value
if (data.size() > 0) {
    cout << "Processing " << data.size() << " items" << endl;
}

Consider a comparison table showing .empty() vs .size() == 0 with their use cases.

Summary

A vector is empty when it contains zero elements, and checking for emptiness prevents errors when accessing vector data. Use .empty() for clarity when checking emptiness, or .size() == 0 when you also need the element count.

Appendix

Common Practices

Always check if a vector is empty before accessing its first or last element using bracket notation. This prevents undefined behavior that can crash your program or produce incorrect results.

Use .empty() rather than .size() == 0 when you only care about whether the vector has elements. This makes your code’s intent clearer to other programmers reading your code.

Check for empty vectors at the beginning of functions that process vector data. This allows you to handle the empty case explicitly or return early if there’s nothing to process.

Combine emptiness checks with meaningful error messages or default behaviors. Don’t silently ignore empty vectors when they might indicate a problem in your program logic.

Common Mistakes

Forgetting to check if a vector is empty before accessing elements leads to undefined behavior. Always verify the vector has elements before using bracket notation or calling methods like .front() or .back().

Assuming that an uninitialized vector pointer is the same as an empty vector is incorrect. A null pointer is different from a valid vector object that happens to be empty.

Using .size() > 0 instead of !.empty() is less clear when you only need a true/false answer. The .empty() method better communicates the intent of checking for no elements.

Checking only once at the start of a program and not rechecking after operations that might empty the vector creates bugs. Vectors can become empty during execution through operations like .clear() or .pop_back().

More Examples

// Safe element access
vector<int> scores;
if (!scores.empty()) {
    cout << "First score: " << scores[0] << endl;
    cout << "Last score: " << scores[scores.size() - 1] << endl;
} else {
    cout << "No scores to display" << endl;
}

// Processing only if data exists
vector<string> tasks = getUserTasks();
if (tasks.empty()) {
    cout << "No tasks today!" << endl;
} else {
    for (string task : tasks) {
        cout << "- " << task << endl;
    }
}

// Validating input
vector<double> measurements;
readMeasurements(measurements);

if (measurements.empty()) {
    cout << "Error: No measurements recorded" << endl;

double average = calculateAverage(measurements);

Comparing Vector Elements

How can we determine if values stored in vectors are equal, greater than, or less than each other? Today we’ll explore comparing individual vector elements and comparing entire vectors using both manual and built-in comparison methods.

Total slides: 6

Consider including a visual showing two vectors with arrows pointing to elements being compared.

Comparing Individual Vector Elements

Individual vector elements can be compared using relational operators just like regular variables. Access elements using bracket notation and apply operators such as ==, !=, <, >, <=, and >=.

vector<int> scores = {85, 92, 78, 95};

if (scores[0] == scores[2]) {
    cout << "First and third scores are equal" << endl;
}

if (scores[1] > scores[0]) {
    cout << "Second score is higher than first" << endl;
}

bool isLower = (scores[2] < scores[3]);

Each comparison evaluates to a boolean value that can be used in conditionals or stored in variables. The elements being compared must be of compatible types that support the comparison operator being used.

Manual Element-by-Element Comparison

Comparing vectors element-by-element manually is possible but requires accessing elements by position, which is challenging with for-each loops alone. For-each loops iterate through one vector at a time without providing corresponding positions.

// For-each can only compare if vectors contain comparable objects
vector<int> values = {10, 20, 30};

// Can check properties of individual elements
for (int val : values) {
    if (val > 25) {
        cout << val << " is greater than 25" << endl;
    }
}

Fortunately, C++ provides built-in operators that handle element-by-element comparison automatically, eliminating the need for manual iteration. These built-in operators are more convenient and less error-prone for comparing entire vectors.

Consider a diagram showing the limitation of for-each loops when comparing two vectors simultaneously.

Comparing Entire Vectors with Built-in Operators

C++ provides built-in comparison operators that work directly on entire vectors. These operators (==, !=, <, >, <=, >=) compare vectors without requiring manual loops.

vector<int> vec1 = {10, 20, 30};
vector<int> vec2 = {10, 20, 30};
vector<int> vec3 = {10, 25, 30};

if (vec1 == vec2) {
    cout << "vec1 and vec2 are equal" << endl;
}

if (vec1 != vec3) {
    cout << "vec1 and vec3 are different" << endl;
}

if (vec1 < vec3) {
    cout << "vec1 is less than vec3" << endl;
}

The == operator returns true only if both vectors have the same size and all corresponding elements are equal. The != operator returns true if vectors differ in size or any elements differ.

Understanding Lexicographic Comparison

The relational operators <, >, <=, and >= perform lexicographic comparison, similar to dictionary ordering. Vectors are compared element-by-element from left to right until a difference is found.

vector<int> vec1 = {10, 20, 30};
vector<int> vec2 = {10, 25, 30};
vector<int> vec3 = {10, 20};

// vec1 < vec2 is true (20 < 25 at index 1)
// vec3 < vec1 is true (shorter vector is less if all elements match)

When elements differ at a position, that difference determines the result regardless of later elements. If all elements match but one vector is shorter, the shorter vector is considered less than the longer one.

Consider a visual showing step-by-step lexicographic comparison with arrows indicating where the comparison stops.

Summary

Vector elements can be compared individually using relational operators, or entire vectors can be compared using built-in operators. Built-in comparison operators use lexicographic ordering to determine relationships between vectors.

Appendix

Common Practices

Always verify that vectors have compatible types before comparing them. Comparing vectors of different element types will result in compilation errors.

Check vector sizes before manual element-by-element comparison to avoid accessing invalid indices. Attempting to access elements beyond the vector size causes runtime errors.

Use built-in comparison operators (==, !=, <, >, <=, >=) when comparing entire vectors for simplicity and clarity. Reserve manual element-by-element comparison for cases requiring custom comparison logic.

Store comparison results in boolean variables with descriptive names when the logic is complex. This improves code readability and makes the purpose of the comparison clear.

Common Mistakes

Comparing vectors of different sizes without checking lengths first leads to accessing invalid indices. Always verify both vectors have the same size before manual element-by-element iteration.

Assuming that vector comparison works like array comparison is incorrect. Unlike arrays, vectors support direct comparison operators that handle the comparison logic automatically.

Forgetting that lexicographic comparison stops at the first difference can lead to unexpected results. Elements after the first difference do not affect the comparison outcome.

Using == to compare vectors of floating-point numbers may produce unexpected results due to precision issues. Consider using a tolerance-based comparison for floating-point vectors instead.

More Examples

// Finding maximum element by comparing with each element
vector<int> numbers = {45, 12, 67, 23, 89};
int maxValue = numbers[0];

for (int num : numbers) {
    if (num > maxValue) {
        maxValue = num;
    }
}
cout << "Maximum: " << maxValue << endl;

// Checking if all elements are positive
vector<int> values = {5, 10, 15, 20};
bool allPositive = true;

for (int val : values) {
    if (val <= 0) {
        allPositive = false;
        break;
    }
}

// Comparing vectors with different sizes
vector<string> names1 = {"Alice", "Bob"};
vector<string> names2 = {"Alice", "Bob", "Charlie"};

if (names1 != names2) {
    cout << "Name lists differ" << endl;
}
if (names1 < names2) {
    cout << "names1 is less (shorter)" << endl;
}
