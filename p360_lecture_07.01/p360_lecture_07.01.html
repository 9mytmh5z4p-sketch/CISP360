Conditional Statements

Choosing the Shape of a Decision

  • A conditional statement chooses whether a block of code should run.
  • The structure should match the output paths the program needs.

Slides 3-9

One-Way, Two-Way, Multi-Way

Structure Best Use
if One optional action
if/else Two mutually exclusive actions
if/else if More than two mutually exclusive categories

Focus on the Output

  • Start with what the program should do.
  • Then choose the conditional structure that matches those possible outcomes.
  • Conditions are tools for reaching the intended output, not the starting point of the design.

Blocks Decide What Runs

if (score >= 70) {
    cout << "Pass";
} else {
    cout << "Try again";
}

Each branch controls a block. Exactly one of these two blocks runs.

When Basic Selection Is Not Enough

  • Some decisions compare one value against fixed cases.
  • Some decisions require a second question inside the first.
  • Some decisions need more than one condition at the same time.

Summary

  • Conditional statements are design choices.
  • Pick the structure that matches the number and relationship of the possible outcomes.

The Conditional Operator

A Shorter Two-Way Choice

  • The conditional operator is a compact way to choose between two values.
  • It is useful when a full if/else statement would be longer than the idea itself.

Slides 10-17

The Shape of ?:

condition ? value_if_true : value_if_false
  • The condition is tested first.
  • One of the two values is produced.

An Expression, Not a Statement

string status = score >= 70 ? "passing" : "not passing";

The conditional operator produces a value, so it can be used in an assignment, an output statement, or another expression.

From if/else to ?:

if (age >= 18) {
    label = "adult";
} else {
    label = "minor";
}
label = age >= 18 ? "adult" : "minor";

When It Helps

  • Choose one of two short labels.
  • Choose one of two numeric values.
  • Keep a simple output statement readable.

When It Hurts

grade = score >= 90 ? "A" : score >= 80 ? "B" : score >= 70 ? "C" : "D";

Nested conditional operators are legal, but they are often harder to read than an if/else if chain.

Summary

  • Use ?: when the choice is simple, local, and easier to read as one expression.
  • Use a normal conditional statement when the decision needs more room.

The switch Statement

Choosing From Fixed Cases

  • A switch statement chooses among fixed exact values.
  • It is useful when one expression is compared against several known cases.

Slides 18-26

The Shape of a switch Statement

switch (category) {
case 'A':
    cout << "Excellent";
    break;
default:
    cout << "Unrecognized category";
}

switch vs. if/else if

Use switch when… Use if/else if when…
One expression is checked Different expressions are checked
Cases are exact values Conditions use ranges
The choices are discrete The choices need relational tests

case Labels Match Exact Values

case 1:
case 'A':
case true:

case labels are exact matches. They do not test ranges like score >= 90.

break Controls Fall-Through

case 'A':
    cout << "Excellent";
    break;
case 'B':
    cout << "Good";
    break;

Without break, execution continues into the next case.

default Handles Everything Else

default:
    cout << "Unknown category";

The default branch handles values that do not match any listed case.

Category Code Example

char shippingCode;
cin >> shippingCode;

switch (shippingCode) {
case 'S':
    cout << "Standard shipping";
    break;
case 'E':
    cout << "Express shipping";
    break;
case 'P':
    cout << "Pickup";
    break;
default:
    cout << "Unknown shipping code";
}

Summary

  • switch is best when one expression is compared against a fixed set of exact values.
  • Use if/else if when the decision depends on ranges or different questions.

Nested if Statements

A Decision Inside a Decision

  • A nested if statement places one conditional inside another conditional block.
  • The inner decision only matters after the outer decision has selected a path.

Slides 27-36

When the Second Test Depends on the First

  • Test another condition only if the first condition is true.
  • Test another condition only if the first condition is false.
  • That dependency is the reason to nest.

Nested if vs. if/else if

Structure Question Shape
if/else if Which one category applies?
Nested if After this branch, what else must be checked?

Nested decisions are dependent. if/else if chains are usually categories at the same level.

Reading the Indentation

if (isMember) {
    if (orderTotal >= 50) {
        cout << "Free shipping";
    } else {
        cout << "Member shipping rate";
    }
}

Indentation shows that the order-total decision belongs inside the member decision.

Shipping Fee Example

if (isMember) {
    if (orderTotal >= 50) {
        shipping = 0.00;
    } else {
        shipping = 4.99;
    }
} else {
    shipping = 7.99;
}

The order total matters only after the program knows the customer is a member.

Discount Eligibility Example

if (accountActive) {
    if (purchaseAmount >= 100) {
        discountRate = 0.10;
    } else {
        discountRate = 0.00;
    }
} else {
    cout << "Account inactive";
}

The purchase amount matters only if the account is active.

Access Level Example

if (loggedIn) {
    if (role == "admin") {
        cout << "Full access";
    } else {
        cout << "Standard access";
    }
} else {
    cout << "Please log in";
}

The role check only makes sense after the program knows the user is logged in.

Keep Nesting Shallow

  • One level of nesting is usually easy to read.
  • Multiple levels can hide the main path through the program.
  • If the code becomes hard to follow, reconsider the design.

Summary

  • A nested if is useful when the next question depends on the answer to the first question.
  • Use indentation and small blocks to make the dependency visible.

Compound Conditions

Combining Questions

  • A compound condition joins two or more smaller conditions.
  • Logical operators let one decision ask a larger question.

Slides 37-46

The AND Operator &&

if (age >= 18 && hasTicket) {
    cout << "Enter";
}

The whole condition is true only when both smaller conditions are true.

The OR Operator ||

if (payment == "cash" || payment == "card") {
    cout << "Payment accepted";
}

The whole condition is true when at least one smaller condition is true.

Short-Circuit Evaluation

if (denominator != 0 && numerator / denominator > 2) {
    cout << "Safe comparison";
}

C++ can stop checking once the final answer is already known. Here, the division only happens if the denominator is not zero.

The NOT Operator !

if (!isOpen) {
    cout << "Closed";
}

The NOT operator reverses the truth value of a condition.

Truth Tables

A B A && B A || B
true true true true
true false false true
false true false true
false false false false

Reading Compound Conditions Out Loud

if (score >= 0 && score <= 100) {
    cout << "Valid score";
}

Read it as: “if score is at least 0 and score is at most 100.”

Reducing Extra Branches

if (isWeekend || isHoliday) {
    rate = 1.5;
}

Logical operators can combine related questions instead of forcing separate branches for every combination.

Summary

  • && requires both conditions to be true.
  • || requires at least one condition to be true.
  • ! reverses a condition.

Range Checking

Inside or Outside the Range

  • Range checking asks whether a value belongs between two boundaries.
  • The boundaries may be included or excluded depending on the problem.

Slides 47-55

Checking Inside a Range

if (score >= 0 && score <= 100) {
    cout << "Valid score";
}

Inside-range checks usually need both boundary comparisons to be true.

Why Inside Uses AND

For a score to be valid:

  • It must be at least 0.
  • It must be at most 100.
  • Both facts must be true at the same time.

Checking Outside a Range

if (score < 0 || score > 100) {
    cout << "Invalid score";
}

Outside-range checks usually need only one boundary problem to be true.

Why Outside Uses OR

For a score to be invalid:

  • It can be below 0.
  • It can be above 100.
  • Either problem is enough to reject the score.

Boundary Values Matter

Condition Are 0 and 100 valid?
score >= 0 && score <= 100 yes
score > 0 && score < 100 no

Inclusive operators include the endpoints. Exclusive operators leave them out.

Score Evaluation Example

if (score < 0 || score > 100) {
    cout << "Invalid score";
} else if (score >= 90) {
    cout << "A";
} else if (score >= 80) {
    cout << "B";
} else {
    cout << "Keep working";
}

The range check protects the category logic that follows.

Summary

  • Inside-range checks use &&.
  • Outside-range checks use ||.
  • Decide whether boundary values should count before writing the operators.

Key Takeaways

  • Conditional structures should match the output paths the program needs.
  • The conditional operator is useful for simple value choices.
  • switch works best for exact fixed cases.
  • Nested if statements are for dependent decisions.
  • Compound conditions and range checks let one decision ask a larger question.

Compound Conditions: Good Practices

  • Combine conditions that belong to the same decision.
  • Use parentheses when grouping improves readability.
  • Put safety checks before expressions that depend on them.

Compound Conditions: Common Mistakes

  • Using && when either condition should be enough.
  • Using || when both conditions are required.
  • Forgetting that ! reverses the condition that follows it.

Compound Conditions: More Examples

if (temperature >= 60 && temperature <= 80) {
    cout << "Comfortable";
}

if (choice == 'Y' || choice == 'y') {
    cout << "Continuing";
}

Range Checking: Good Practices

  • Decide whether endpoints should count before writing the condition.
  • Use && for inside-range checks.
  • Use || for outside-range checks.

Range Checking: Common Mistakes

  • Writing 0 <= score <= 100 as if C++ understood chained comparisons.
  • Using || for inside-range checks.
  • Using && for outside-range checks.
