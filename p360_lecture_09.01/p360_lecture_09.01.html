Menu-Driven Programs

What keeps a menu program running between choices?

  • Combine a loop with a decision structure
    • the loop keeps the menu alive,
    • the decision routes each choice.

Total slides: 4

Menu-Driven Program Structure

  • The loop
    • The menu display
    • The Prompt
    • Input Validation
    • The dispatcher

The do-while is a natural fit: display first, then check whether to continue.

Menu-driven program structure

Input Validation on the Choice

int choice;
do {
    displayMenu();
    cin >> choice;

    if (choice < 1 || choice > 4) {
        cout << "Invalid choice. Enter 1-4.\n";
        continue;
    }

    dispatch(choice);

} while (choice != 4);

Validate the choice before dispatching. continue skips the dispatch and re-displays the menu.

Summary

A menu-driven program wraps a loop around the menu structure you are already familiar with.

Core Loop Patterns

What are loops actually for?

Loop patterns are the recurring solutions that show up across every program you will write.

Total slides: 4

Three Pattern Categories

Loops fall into three conceptual groups based on what they do:

  • Control Patterns
  • State Tracking Patterns
  • Structural Patterns

Control Patterns

  • Control Patterns — decide when to stop:
    • sentinel loops,
    • flag-controlled loops,
    • input validation loops

State Tracking Patterns

  • State Tracking Patterns — accumulate information across iterations:
    • counters,
    • running totals,
    • min/max tracking

Structural Patterns

  • Structural Patterns — move through a collection: -iteration over arrays and vectors

What Loops Are Used For

Every loop you write does one of these four things:

  1. Repeats an action a controlled number of times
  2. Processes every element in a collection
  3. Tracks or accumulate information over time
  4. Keeps running until a condition occurs

When you are stuck on a loop problem, identify which of these applies first.

Summary

  • Loops are patterns.
  • Classifying a loop by what it does (control, state tracking, structural) tells you which variables to initialize, what the condition should test, and what belongs in the body.

Control Patterns

Stopping a loop that doesn’t know in advance when to stop?

  • How do you stop a loop when you don’t know how much data you need to process?

Total slides: 8

Sentinel Loop

A sentinel is a special value that signals the end of input — not real data, just a stop signal.

  • The loop reads values until it sees the sentinel
  • The sentinel must be outside the valid data range
  • Common sentinels: -1, 0, "quit", EOF

Use when the number of inputs is unknown and a special value can mark the end.

Sentinel Loop — Code

int value;
cout << "Enter a number (-1 to quit): ";
cin >> value;

while (value != -1) {
    process(value);
    cout << "Enter a number (-1 to quit): ";
    cin >> value;
}

The priming read runs before the loop. The same read repeats at the bottom of the body.

Flag-Controlled Loop

A flag is a boolean variable that starts as true (or false) and flips when a condition is met.

  • More flexible than a sentinel — the stopping condition can be complex
  • The flag can be set from anywhere inside the loop body
  • Clear naming matters: done, found, isValid

Use when the exit condition involves logic that’s hard to express in a single comparison.

Flag-Controlled Loop — Code

bool done = false;

while (!done) {
    int input = getInput();

    if (input == QUIT_CODE || attemptsExceeded()) {
        done = true;
    } else {
        process(input);
    }
}

The flag decouples the exit decision from the loop header. Either condition can trigger it.

Input Validation Loop

An input validation loop rejects bad input and reprompts until the user provides something acceptable.

  • Prompt, read, test — repeat until the input is valid
  • Give a specific error message on each rejection
  • The do-while matches this pattern well: prompt once, then loop while invalid

This is a control pattern: the loop exists entirely to enforce a stopping condition on input quality.

Input Validation Loop — Code

int age;
do {
    cout << "Enter age (1-120): ";
    cin >> age;
    if (age < 1 || age > 120)
        cout << "Invalid. Age must be between 1 and 120.\n";
} while (age < 1 || age > 120);

The condition is the same in both places — once to check, once to decide whether to loop.

Summary

Control patterns share one purpose: managing when the loop ends. Sentinels use a special value, flags use a boolean, and validation loops enforce input quality. Choose based on what kind of stopping condition your problem has.

State Tracking Patterns

How does a loop remember what happened on previous iterations?

  • uses variables updated on every pass.
  • When the loop ends, those variables hold a result computed across all iterations.

Total slides: 8

Counters

A counter increments (or decrements) each time a condition is met.

  • Initialize to zero before the loop
  • Increment inside the body when the condition holds
  • The final value is the count of qualifying iterations

Use when you need to know how many times something happened.

Counter — Code

int count = 0;

for (int val : scores) {
    if (val >= 90)
        count++;            // tells us how many times the loop executed
}

cout << "High scores: " << count << "\n";

count is initialized outside, updated inside, and read after. This is the counter pattern.

Running Totals (Accumulators)

An accumulator adds each value to a running sum.

  • Initialize to zero before the loop
  • Add each element to the accumulator inside the body
  • Divide after the loop to compute an average

Use when you need a sum or average over a collection or stream of values.

Accumulator — Code

double total = 0.0;

for (int val : scores) {
    total += val;
}

double average = total / scores.size();
cout << "Average: " << average << "\n";

The accumulator pattern requires total initialized to zero — any other starting value corrupts the result.

Maximum / Minimum Tracking

A max/min tracker keeps the largest (or smallest) value seen so far.

  • Initialize to the first element — not to zero
  • Compare each element; update if it beats the current record
  • Common mistake: initializing to zero when all values are negative

Use when you need the extreme value in a dataset.

Max / Min Tracking — Code

int maxVal = scores[0];

for (int i = 1; i < scores.size(); i++) {
    if (scores[i] > maxVal)
        maxVal = scores[i]; //.... replace maxVal when we find a higher one
}

cout << "High score: " << maxVal << "\n";

Starting at index 1 and initializing to scores[0] avoids the empty-comparison problem.

Summary

State tracking patterns turn a loop into a computation. Counters count, accumulators sum, and max/min trackers find extremes. In every case: initialize before the loop, update inside, read after.

Structural Patterns

Moving a loop through every element in a collection?

  • Structural patterns are about traversal
    • visiting each element in a vector or array exactly once.

Total slides: 4

Two Ways to Traverse a Collection

Range-for (value only)

for (int val : scores) {
    cout << val << "\n";
}

Clean and safe — no index, no bounds error. Use when you only need the value.

Index loop

for (int i = 0; i < scores.size(); i++) {
    cout << i << ": " << scores[i] << "\n";
}

Use when you need the index — i.e. the position matters.

Choosing the Right Traversal

Need Use
Read each value for-each
Modify elements in place index loop
Compare adjacent elements index loop
Build a parallel result index loop
Print with position numbers index loop

Summary

  • Structural patterns are traversals.
  • For-each is the right default — it reads clearly and eliminates bounds errors.
  • Index loop only when the position itself is part of the computation.

Nested loops

How Nested Loops Execute

  • The outer loop controls the number of full passes
  • The inner loop runs completely on each outer pass
  • Total iterations = outer count × inner count

A 5-iteration outer loop containing a 10-iteration inner loop executes 50 total iterations.

[Graphic suggestion: a simple grid showing outer rows × inner columns, each cell = one inner iteration]

Execution Trace

for (int i = 0; i < 3; i++) {   //............ outer loop
    for (int j = 0; j < 3; j++) { //.......... inner loop
        cout << i << "," << j << "  ";
    }
    cout << "\n";  //..... moves to new ROW
}

Execution Trace — Output

0,0  0,1  0,2
1,0  1,1  1,2
2,0  2,1  2,2

i changes once per row. j cycles through all values for every value of i.

When to Use Nested Loops

  • Processing a 2D structure (grid, table, matrix)
  • Comparing every element to every other element
  • Generating all combinations of two sets

Keep nesting to two levels when possible. Three or more levels are hard to reason about and usually signal a design problem.

Summary

  • A nested loop multiplies iteration counts — outer × inner.
  • Use them for 2D problems and combination generation.

Coding Exercise

Let’s write it

Apply nested loops to a concrete problem. Work through it before looking at the solution.

Total slides: 3

The Task

Write a program that prints a multiplication table for values 1 through 5.

Expected output:

 1  2  3  4  5
 2  4  6  8 10
 3  6  9 12 15
 4  8 12 16 20
 5 10 15 20 25
  • Outer loop controls the row (i)
  • Inner loop controls the column (j)
  • Each cell prints i * j, formatted to width 3

Solution

#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    for (int i = 1; i <= 5; i++) {
        for (int j = 1; j <= 5; j++) {
            cout << setw(3) << i * j;
        }
        cout << "\n";
    }
    return 0;
}

setw(3) right-aligns each value in a field of width 3. The newline after the inner loop ends each row.

Key Takeaways

  • Every loop belongs to a category: control (when to stop), state tracking (what to remember), or structural (how to traverse). Identify the category first, then choose the syntax.
  • State tracking variables follow one rule: initialize before the loop, update inside, read after.
  • Nested loops multiply iteration counts. Keep nesting to two levels; flatten deeper structures with functions.

  • Forgetting to update the sentinel variable inside the loop body
  • Using a flag but never setting it to true — infinite loop

State Tracking: Good Practices

  • Always initialize accumulators to zero and max/min trackers to the first element
  • Do all state updates inside the loop; read the result after
  • Use meaningful variable names: total, count, maxScore — not x, temp

State Tracking: Common Mistakes

  • Initializing a max tracker to 0 — fails when all values are negative
  • Reading the accumulator inside the loop before it is complete
  • Forgetting to initialize the counter, causing undefined behavior

State Tracking: More Examples

  • Find the highest and lowest temperatures in a dataset simultaneously

Structural Patterns: Good Practices

  • Default to range-for when you only need values — it eliminates off-by-one errors
  • Use size_t or int consistently for index variables; mixing signed and unsigned invites warnings
  • If you modify elements inside a range-for, use a reference: for (int& val : scores)

Structural Patterns: Common Mistakes

  • Using an index loop when range-for would be cleaner
  • Off-by-one: i <= scores.size() instead of i < scores.size()
  • Forgetting & in range-for when you intend to modify elements

Structural Patterns: More Examples

  • Print every other element using an index loop
  • Double every value in a vector using range-for with reference
  • Find the index of the maximum value (requires an index loop)

Structural Patterns: Practice Problems

  3. Find both the value and the index of the maximum element in a vector.

Nested Loops: Good Practices

  • Keep nesting to two levels — deeper nesting should trigger a refactor
  • Use distinct, meaningful loop variables: row/col, i/j — never reuse the same variable name
  • Calculate expected iteration count (outer × inner) before writing — surprises here are expensive

Nested Loops: Common Mistakes

  • Using i for both the outer and inner loop variable — shadows the outer, causes wrong results
  • Forgetting that the inner loop resets on every outer iteration
  • Placing a statement that should be outside the inner loop inside it (e.g., printing a newline per cell instead of per row)

Nested Loops: More Examples

  • Print a right triangle of asterisks (outer = row, inner = number of stars per row)
  • Check whether any two elements in a vector are equal (all-pairs comparison)
  • Generate all pairs (i, j) where i < j from a vector

Nested Loops: Practice Problems

  1. Print a 5×5 grid of asterisks.
  2. Print a lower-left triangle: row 1 has 1 star, row 2 has 2 stars, etc.
  3. Given a vector, print all pairs of elements where the first is less than the second.
