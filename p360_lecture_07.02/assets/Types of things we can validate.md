Types of things we can validate
1. Presence (is there any data at all?)
2. Data type (is it an int, string, etc.?)
3. Range / bounds (is a number within min/max?)
4. Format (does it match a pattern like a zip code or phone number?)
5. Set membership (is it one of a specific list of allowed values?)
6. Business logic (does it make sense in context — e.g., end date after start date?)
7. Consistency (does it agree with other fields already entered?)
