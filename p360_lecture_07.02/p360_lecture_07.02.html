Why Iteration Matters

Why Repeat Work?

  • How do we tell a program to do the same kind of work more than once?
  • Iteration is controlled repetition.

Slides 3-8

Programming vs. Coding

Design Question Syntax Question
What work repeats? Which loop construct expresses it?
What changes each time? Where does the update happen?
When should repetition stop? Where is the condition checked?

Iteration in IPO

IPO Part Loop Connection
Input New data may arrive each pass
Process The repeated work usually lives here
Output Results may appear once per pass or after the loop

Same Work, New Data

  • Each pass through a loop should have a purpose.
  • The repeated work may use a new input value, a new counter value, or a new program state.
  • A loop is useful when the shape of the work stays the same.

Summary

  • Iteration repeats useful work under control of a condition.
  • The design question comes first: what repeats, what changes, and when should it stop?

Loop Anatomy and Program Control

A Loop You Already Know

  • A range-based for loop says: do this once for each item in a collection.
  • You have already used loops when reading or displaying every value in a vector.

Slides 9-19

What for-each Hides

for (int score : scores) {
    cout << score << endl;
}

The loop visits each element, but the stopping condition and movement through the collection are handled for you.

That Sounds Like an if Statement

  • Both if statements and loops use a condition.
  • The condition decides whether a block should run.
  • The difference is what happens after the block finishes.

The Difference Is Control Flow

Structure After the Block
if Continue to the next statement
Loop Return to the loop condition or end the loop

if vs. Loop Control Flow

loop

The Three Pieces of a Loop

  • Condition: the true/false test that controls repetition.
  • Control variable: the value that helps decide whether to continue.
  • Body: the block of work that repeats.

Entry vs. Exit Conditions

Entry-condition loops may run zero times. Exit-condition loops run at least once.

The Condition Must Change

  • A loop must move toward stopping.
  • That usually means input changes, a counter changes, or a flag changes.
  • If the condition never becomes false, the loop does not end.

Tracing the Control Variable

Pass count Condition count <= 3 Output
1 1 true 1
2 2 true 2
3 3 true 3
4 4 false stop

Summary

  • Explicit loops expose the control machinery that range-based for keeps out of sight.
  • A loop is a decision plus a path back to the decision.

The while Loop

When Might the Work Happen Zero Times?

  • A while loop checks the condition before the body runs.
  • Use it when the program may need to skip the repeated work entirely.

Slides 20-29

The Shape of a while Loop

while (condition) {
    statement;
    statement;
}

The condition is tested first. If it is true, the body runs.

while Loop Control Flow

while in IPO

IPO Part Example
Input Get or already have a value
Process Repeat while the condition is true
Output Show each result or the final result

The loop belongs to the processing design.

A First while Example

#include <iostream>
using namespace std;

int main() {
    int count = 1;

    while (count <= 3) {
        cout << count << endl;
        count++;
    }

    return 0;
}

Trace the Control Variable

Pass count Before Test Test count After Update
1 1 true 2
2 2 true 3
3 3 true 4
4 4 false 4

The Stop Condition

  • The loop stops when the condition becomes false.
  • In the example, count <= 3 becomes false when count reaches 4.
  • The update is what moves the loop toward stopping.

Unknown Repetition

  • while is useful when the number of passes is not known ahead of time.
  • The program keeps checking whether more work is needed.
  • The loop might run many times, once, or not at all.

Summary

  • A while loop tests first.
  • Use it when repetition depends on a condition that may already be false.

The do-while Loop

When Must the Work Happen Once?

  • A do-while loop runs the body before testing the condition.
  • Use it when the program must act once before it can decide whether to repeat.

Slides 30-39

The Shape of a do-while Loop

do {
    statement;
    statement;
} while (condition);

The semicolon after the condition is part of the syntax.

do-while Loop Control Flow

do-while in IPO

IPO Part Example
Input Ask at least once
Process Decide whether the value is acceptable
Output Prompt again or continue

The first pass happens before the loop can stop.

A Prompt-Then-Check Example

#include <iostream>
using namespace std;

int main() {
    int choice;

    do {
        cout << "Enter 1 to continue or 0 to quit: ";
        cin >> choice;
    } while (choice != 0);

    return 0;
}

Trace the First Pass

Moment choice Condition choice != 0
Before body unknown not tested yet
After input 1 true
After input 0 false

Menus and Validation

  • Menus usually need to display before the user can choose.
  • Prompts usually need to ask before the program can judge the answer.
  • That makes do-while a common fit for “ask, then check” designs.

do-while vs. while

Use while when… Use do-while when…
The test should happen first The body must happen first
Zero passes are possible One pass is required
Existing state controls entry New input controls the next pass

Summary

  • A do-while loop tests after the body.
  • Use it when the first pass is part of getting the information needed to decide.

The for Loop

When Do We Know the Count?

  • A for loop is a natural fit for counted repetition.
  • Use it when the program knows the number of passes or the numeric range to visit.

Slides 40-49

The Shape of a for Loop

for (initialization; condition; update) {
    statement;
    statement;
}

The header gathers the counter setup, the test, and the update in one place.

for Loop Control Flow

for in IPO

IPO Part Example
Input Read speed and totalHours
Process Calculate distance once per hour
Output Display the hour and distance

The count is known before the repeated processing begins.

The Header Controls the Counter

for (int hour = 1; hour <= totalHours; hour++) {
    int distance = speed * hour;
    cout << hour << " " << distance << endl;
}

The counter starts at 1, continues through totalHours, and changes by 1 each pass.

Distance by Hour

#include <iostream>
using namespace std;

int main() {
    int speed = 40;
    int totalHours = 3;

    for (int hour = 1; hour <= totalHours; hour++) {
        int distance = speed * hour;
        cout << hour << " " << distance << endl;
    }

    return 0;
}

Trace the Counter

hour speed distance = speed * hour
1 40 40
2 40 80
3 40 120
4 40 stop

for vs. Range-Based for

Use counted for when… Use range-based for when…
The position matters Each item is enough
A numeric range matters The collection controls the passes
You need a counter value You do not need an index

Summary

  • A for loop packages counter setup, test, and update in the header.
  • Use it when the repetition is naturally counted.

Choosing a Loop

Start With the Repetition Need

  • Do not start by picking syntax.
  • Start by asking what controls the repetition.

Slides 50-57

Use Range-Based for When You Have a Collection

for (int score : scores) {
    total += score;
}

Use this when every item in the collection should be processed and the item value is enough.

Use for When You Know the Count

for (int hour = 1; hour <= totalHours; hour++) {
    cout << hour << endl;
}

Use this when the number of passes or the numeric range is known.

Use while When You Must Test First

while (moreData) {
    processData();
    moreData = checkForMoreData();
}

Use this when the repeated work should happen only if the condition is already true.

Use do-while When One Pass Is Required

do {
    displayMenu();
    cin >> choice;
} while (choice != 0);

Use this when the program must act once before it can decide whether to repeat.

Decision Table

Repetition Need Best Fit
Process every item in a collection Range-based for
Repeat a known number of times for
Repeat while a condition is true while
Run once, then decide whether to repeat do-while

Summary

  • Choose the loop whose control structure matches the problem.
  • The goal is not to memorize loops; the goal is to match the design need.

Key Takeaways

  • Iteration is controlled repetition.
  • Loops extend the IPO model by repeating part of the process.
  • while tests before the body; do-while tests after the body.
  • for is best for counted repetition.
  • Range-based for remains the clearest choice when each collection item is enough.

The for Loop: Good Practices

  • Use for for counted repetition.
  • Keep initialization, condition, and update easy to read.
  • Use a counter name that describes the count, such as hour or day.

The for Loop: Common Mistakes

  • Using <= when the intended range requires <.
  • Changing the counter unexpectedly inside the body.
  • Using for when the number of passes is not known.

The for Loop: More Examples

    cout << "Day " << day << endl;
}

Choosing a Loop: Good Practices

  • Choose based on the repetition need.
  • Prefer range-based for when the collection controls the loop.
  • Prefer the simplest loop that states the control clearly.

Choosing a Loop: Common Mistakes

  • Choosing a loop because it is familiar instead of because it fits.
  • Using a counted for loop when the condition is not count-based.
  • Forgetting that range-based for is still available.
