Relational Operators

Comparing Values

  • How can a program ask whether two values have a specific relationship?
  • Relational operators let code ask questions with yes-or-no answers.

Slides 3-12

The Relational Operators

Operator Meaning
== equal to
!= not equal to
< less than
> greater than
<= less than or equal to
>= greater than or equal to

A Comparison Produces a Value

int score = 87;

score >= 70
score == 100
score < 60

Each expression evaluates to either true or false.

Equality Is Not Assignment

score = 70;   // Stores 70 in score
score == 70;  // Asks whether score is equal to 70

The single equals sign changes a variable. The double equals sign compares two values.

Common Comparison Shapes

Shape Example
Threshold age >= 21
Exact choice choice == 1
Boundary temperature < 32
Different value menuChoice != 0

Chained Comparisons Need Care

0 < score < 100

C++ does not read this like a math expression. Split the idea into separate comparisons when you need a range.

[Graphic suggestion: show the math notation on one side and the two separate C++ comparisons on the other.]

Comparing Floating-Point Values

Integers can usually be compared directly.

Floating-point values may be slightly different because of rounding. Decide how close is close enough before calling them equal.

Conditions Use Relational Expressions

if (score >= 70) {
    cout << "Passing";
}

The condition inside the parentheses is a relational expression. It asks the question the if statement will use.

Summary

  • Relational operators compare values.
  • A comparison evaluates to true or false.
  • Decision statements use those true-or-false results to choose what code runs.

The if Statement

Doing Something Only Sometimes

  • What should a program do when an action depends on a condition?
  • An if statement gives the program one conditional action.

Slides 13-21

The Shape of an if Statement

if (condition) {
    statement;
}

The condition is placed in parentheses. The controlled code is placed inside braces.

Condition First, Action Second

  1. Evaluate the condition.
  2. If the condition is true, run the block.
  3. If the condition is false, skip the block.

[Graphic suggestion: small flowchart with condition diamond, true path into the block, false path around it.]

One Statement or a Block

if (temperature < 32) {
    cout << "Freezing";
    cout << "Watch for ice";
}

Braces make the controlled region visible. Use them even when the block starts with one statement.

Protecting Code from Bad Inputs

if (denominator != 0) {
    quotient = numerator / denominator;
}

The condition protects the division. The calculation runs only when the denominator is safe to use.

Comparing Two Values

if (first > second) {
    cout << "First is larger";
}

if (first == second) {
    cout << "The values match";
}

Separate if statements ask separate questions.

Reading if Statements Out Loud

if (temperature < 32) {
    cout << "Freezing";
}

Read this as: if temperature is less than 32, print "Freezing".

Summary

  • An if statement runs a block only when its condition is true.
  • The condition must evaluate to true or false.
  • Code outside the block is not controlled by the if statement.

The cin Fail Problem

When Input Breaks the Program

  • What happens when the user enters the wrong type of data?
  • The input operation can fail, and the program needs a way to notice.

Slides 22-30

What cin Was Expecting

int age;
cin >> age;

This statement expects input that can become an int. Text such as hello cannot be stored in age.

Testing cin.fail()

int age;
cin >> age;

if (cin.fail()) {
    cout << "Age must be a whole number.";
}

cin.fail() is a condition. It is true when the previous input operation failed.

Detecting Is Not Fixing

cin.fail() tells the program that input failed.

It does not repair the stream, remove the bad input, or ask the user again. Those actions must be written separately.

The Stream Stays Failed

After a failed extraction, later input attempts may not work.

The stream is still carrying the failed state. The bad input may also still be waiting to be read.

A First Recovery Pattern

if (cin.fail()) {
    cin.clear();
    cin.ignore(1000, '\n');
    cout << "Please enter a whole number.";
}

The condition detects the problem. The recovery code clears the error state and removes the bad input.

Why This Is Still an if Statement

The recovery block should run only when input failed.

That makes cin.fail() a practical example of one-way selection: if the problem happened, respond to it.

Summary

  • Input can fail when the user enters a value that does not match the expected type.
  • cin.fail() detects that failed state.
  • Detection and recovery are separate parts of the program.

The if/else Statement

Two Possible Paths

  • What should a program do when one action happens if a condition is true, and a different action happens if it is false?
  • An if/else statement gives the program two mutually exclusive paths.

Slides 31-39

One Decision, Two Outcomes

Condition Result
true run the if block
false run the else block

Exactly one branch runs.

The Shape of if/else

if (condition) {
    trueBranch;
} else {
    falseBranch;
}

The else branch belongs to the condition above it.

Exactly One Branch Runs

if (isOpen) {
    cout << "Come in";
} else {
    cout << "Closed";
}

The program does not run both branches. It chooses based on the condition.

Night Club Example

if (age >= 21) {
    cout << "Admit";
} else {
    cout << "Do not admit";
}

The condition asks whether the person meets the age requirement. The else handles everyone who does not.

if vs. if/else

Use separate if statements when both conditions could be true.

Use if/else when one path must be true and the other must be false.

Avoid Opposite Conditions

if (age >= 21) {
    cout << "Admit";
} else {
    cout << "Do not admit";
}

The else already means the condition was false. A second opposite test is not needed.

Summary

  • if/else creates a two-way decision.
  • The if branch runs when the condition is true.
  • The else branch runs when the condition is false.

The if/else if Statement

More Than Two Outcomes

  • What should a program do when a decision has several possible categories?
  • An if/else if chain checks multiple conditions in order.

Slides 40-50

The Shape of if/else if

if (firstCondition) {
    firstAction;
} else if (secondCondition) {
    secondAction;
} else {
    defaultAction;
}

The chain begins with if and can include one or more else if branches.

Checked From Top to Bottom

The program checks the first condition.

If that condition is false, the program moves to the next condition. The first true branch runs, and the rest are skipped.

Starter if

if (score >= 90) {
    grade = 'A';
}

The first if starts the chain. It should represent the first category the program needs to test.

else if Conditions

else if (score >= 80) {
    grade = 'B';
} else if (score >= 70) {
    grade = 'C';
}

Each else if adds another possible category.

Default else

else {
    grade = 'F';
}

The final else handles none of the above. It gives the chain a default outcome.

Grade Category Example

if (score >= 90) {
    grade = 'A';
} else if (score >= 80) {
    grade = 'B';
} else if (score >= 70) {
    grade = 'C';
} else {
    grade = 'F';
}

Each score belongs to one category.

Order Matters

if (score >= 70) {
    grade = 'C';
} else if (score >= 90) {
    grade = 'A';
}

A score of 95 never reaches the second branch because the first condition is already true.

Choosing the Right Conditional Shape

Need Use
Do something only when true if
Choose between two paths if/else
Choose among several categories if/else if

Summary

  • An if/else if chain handles several mutually exclusive categories.
  • Conditions are checked from top to bottom.
  • The first true branch runs, and the rest are skipped.

Key Takeaways

  • Relational expressions ask yes/no questions that evaluate to boolean values.
  • if statements run code only when a condition is true.
  • if/else and if/else if structures choose among mutually exclusive paths.
  • Input failure can be detected with cin.fail(), but detection and recovery are separate steps.

Relational Operators: Good Practices

  • Use relational expressions that read like clear questions.
  • Prefer explicit comparisons such as score >= 70 over clever expressions that hide the condition.
  • When comparing floating-point values, decide what difference is small enough to count as equal.

Relational Operators: Common Mistakes

  • Using = when the intent is comparison.
  • Writing chained comparisons such as 0 < score < 100 as if C++ handled them like algebra.
  • Comparing floating-point values for exact equality when the calculation may produce rounding differences.

The if Statement: Good Practices

  • Put the condition in parentheses and the controlled code in braces.
  • Keep the condition focused on one question when possible.
  • Use an if statement when the action is optional.

  • Forgetting that code outside the braces always runs.
  • Making the block do too many unrelated things.

The cin Fail Problem: Good Practices

  • Check input before using the value in later calculations.
  • Treat failure detection and failure recovery as separate steps.
  • Give the user a specific message about what kind of input was expected.

The cin Fail Problem: Common Mistakes

  • Assuming cin.fail() fixes the stream.
  • Trying to keep reading after the stream has entered a failed state.
  • Repeating a prompt without clearing the bad input first.

The cin Fail Problem: More Examples

int quantity;
cin >> quantity;

if (cin.fail()) {
    cout << "Quantity must be a whole number.";
}

This detects the problem. A complete recovery also clears the stream and removes the bad input.

The if/else Statement: Good Practices

  • Use if/else when the two outcomes exclude each other.
  • Let else mean “everything that did not satisfy the if condition.”
  • Keep both branches parallel when they represent matching outcomes.

The if/else Statement: Common Mistakes

  • Writing a second opposite condition when else already covers it.
  • Using two separate if statements for a choice that must select only one path.
  • Putting shared code in both branches instead of after the decision.

The if/else Statement: More Examples

} else {
    cout << "Do not admit";
}

Only one of these messages can print.

The if/else if Statement: Good Practices

  • Put the most specific conditions before broader conditions when order matters.
  • Use a final else when the program needs a default path.
  • Keep each branch focused on one category.

The if/else if Statement: Common Mistakes

  • Placing a broad condition first and blocking later branches.
  • Repeating conditions that are already excluded by earlier branches.
  • Using an if/else if chain when independent checks would be clearer.

The first true branch wins.
