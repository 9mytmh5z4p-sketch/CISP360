Stringstream

Text into Typed Values

  • How can we extract 42 as an int and 3.5 as a double?

Stream Connected to Read with Write with

Reading Tokens from a String

#include <sstream>
#include <string>
using namespace std;

string text = "42 3.5";
istringstream input(text);

int count;
double price;

input >> count >> price;

Extraction converts each token to the type of its destination variable.

Parsing a Complete Line

getline() captures the entire line. An istringstream then extracts individual values from that line.

string line;
getline(cin, line);

istringstream input(line);
string name;
int attempts;
double average;

input >> name >> attempts >> average;

This separates collecting input from interpreting it.

Parsing Code

#include <iostream>
#include <sstream>
#include <string>
using namespace std;

int main() {
    string line = "Ava 4 91.5";
    istringstream input(line);

    string name;
    int attempts;
    double average;

    if (input >> name >> attempts >> average) {
        cout << name << " completed " << attempts
             << " attempts with an average of " << average << "\n";
    }

    return 0;
}

The condition succeeds only if all three extractions succeed. The values are converted according to the destination variable types.

Checking Extraction

int quantity;
double price;

if (input >> quantity >> price) {
    cout << "Valid values\n";
} else {
    cout << "Expected an integer and a decimal value\n";
}

Successful extraction leaves the stream usable. Missing or incompatible input places the stream in a failed state.

Formatting a String

ostringstream output;

output << "Order " << orderNumber
       << ": " << quantity << " items";

string message = output.str();

An ostringstream collects output in memory instead of displaying it immediately. The str member function returns the formatted contents as a string.

Formatting Code

#include <iomanip>
#include <iostream>
#include <sstream>
#include <string>
using namespace std;

int main() {
    int orderNumber = 27;
    double total = 84.5;
    ostringstream output;

    output << "Order " << orderNumber << ": $"
           << fixed << setprecision(2) << total;

    string message = output.str();
    cout << message << "\n";

    return 0;
}

Formatting manipulators work because an output string stream follows the same rules as cout and an ofstream.

Delimited Fields

string record = "Ava,91.5,A";
istringstream input(record);

string name;
string scoreText;
string grade;

getline(input, name, ',');
getline(input, scoreText, ',');
getline(input, grade, ',');

The three-argument form of getline stops at a chosen delimiter. Each call consumes its delimiter before the next field is read.

Delimited Parsing Code

#include <iostream>
#include <sstream>
#include <string>
using namespace std;

int main() {
    string record = "Ava,91.5,A";
    istringstream fields(record);
    string name, scoreText, grade;

    getline(fields, name, ',');
    getline(fields, scoreText, ',');
    getline(fields, grade, ',');

    istringstream scoreInput(scoreText);
    double score;

    if (scoreInput >> score) {
        cout << name << " earned " << grade
             << " with " << score << "\n";
    }

    return 0;
}

This handles a simple comma-delimited record. Full CSV permits quoted commas and requires a dedicated parser.

Choosing a String Stream

Need Choose
Convert text into typed values istringstream
Combine values into formatted text ostringstream
Read and write one string buffer stringstream

Choose the narrowest stream type that matches the direction of data movement.

Summary

  • String streams apply familiar stream operators to an in-memory string
  • Use extraction and stream state to parse typed values
  • Use insertion and .str() to build formatted text
  • Use delimited getline() when fields are separated by a known character

Random Access Files

Updating One Stored Value

  • A file contains 10,000 scores
  • One score changes
  • Must the program read and rewrite all 10,000 values?

Slides 17–36

Sequential and Random Access

Sequential access Random access
Processes values in order Jumps to a calculated position
Position advances automatically Program moves the position explicitly
Natural for text and variable-length data Natural for fixed-size binary data

Random access avoids processing unrelated values before reaching the target.

File Position Markers

A file stream tracks where its next operation will occur.

  • The get position identifies where the next read begins
  • The put position identifies where the next write begins
  • Each successful operation advances its position by the number of bytes processed

[Graphic suggestion: a row of equal-width file blocks with separate get and put arrows positioned above two different blocks]

Opening a Binary File

ifstream input("scores.bin", ios::binary);

ofstream output("scores.bin", ios::binary);

fstream file("scores.bin",
    ios::in | ios::out | ios::binary);
  • ios::binary preserves bytes exactly as written
  • ios::in | ios::out permits an existing file to be read and updated
  • Always verify that the stream opened successfully

Seek Origins

Every seek uses an offset and an origin.

Origin Offset begins at
ios::beg Beginning of the file
ios::cur Current position
ios::end End of the file

Positive offsets move forward. Negative offsets move backward when the origin permits it.

Moving the Read Position

ifstream input("scores.bin", ios::binary);

input.seekg(0, ios::beg);   // beginning
input.seekg(16, ios::beg);  // byte 16
input.seekg(-8, ios::end);  // eight bytes before the end

seekg moves the get position used by the next read. The offset is measured in bytes, not values or lines.

Finding the Read Position

ifstream input("scores.bin", ios::binary);

input.seekg(0, ios::end);
streamoff fileSize = input.tellg();

cout << "File size: " << fileSize << " bytes\n";

input.seekg(0, ios::beg);

tellg reports the current get position. Seeking to the end before calling tellg determines the file size.

Read and Write Positions

Operation Position Purpose
seekg(offset, origin) Get Move the next read
tellg() Get Report the read position
seekp(offset, origin) Put Move the next write
tellp() Put Report the write position

The letters suggest the direction: get data and put data.

Why Text Records Vary

125
42
  • The values occupy different numbers of characters
  • Line endings also consume bytes
  • The location of one value depends on the lengths of all earlier values

Text is easy to inspect, but its variable widths prevent a simple index-to-offset formula.

Fixed-Size Binary Values

Suppose every stored score is a double.

score 0     score 1     score 2     score 3
8 bytes     8 bytes     8 bytes     8 bytes

Every value occupies sizeof(double) bytes, so the location of any index is predictable.

[Graphic suggestion: four equal-width blocks labeled by index, with byte offsets 0, 8, 16, and 24 beneath them]

Writing Binary Values

double score = 91.5;

output.write(
    reinterpret_cast<const char*>(&score),
    sizeof(score)
);
  • write() copies a specified number of bytes
  • &score supplies the address of the value
  • The cast exposes those bytes through the pointer type required by write()

Treat reinterpret_cast here as the standard adapter required by the binary stream interface. Pointer mechanics are covered later in the course.

The Offset Formula

For equal-size values:

byte offset = index × sizeof(value type)

For double values:

Index Calculation Byte offset
0 0 × 8 0
1 1 × 8 8
4 4 × 8 32

Always use sizeof; do not hardcode the byte count.

Reading One Value

int index = 4;
double score;

input.seekg(index * sizeof(double), ios::beg);
input.read(
    reinterpret_cast<char*>(&score),
    sizeof(score)
);

Move the get position, then read the bytes into a variable. Index 4 selects the fifth stored value because indexing begins at zero.

Reading One Value: Complete Code

#include <fstream>
#include <iostream>
using namespace std;

int main() {
    ifstream input("scores.bin", ios::binary);
    if (!input.is_open()) {
        cerr << "Could not open scores.bin\n";
        return 1;
    }

    int index = 4;
    double score;
    input.seekg(index * sizeof(double), ios::beg);
    input.read(reinterpret_cast<char*>(&score), sizeof(score));

    if (input) {
        cout << "Score: " << score << "\n";
    }

    input.close();
    return 0;
}

The stream check after read detects an invalid offset, a shortened file, or another read failure.

Replacing One Value

int index = 4;
double replacement = 96.0;

file.seekp(index * sizeof(double), ios::beg);
file.write(
    reinterpret_cast<const char*>(&replacement),
    sizeof(replacement)
);

Move the put position, then overwrite exactly one value. The bytes before and after that value remain unchanged.

Replacing One Value: Complete Code

#include <fstream>
#include <iostream>
using namespace std;

int main() {
    fstream file("scores.bin",
        ios::in | ios::out | ios::binary);

    if (!file.is_open()) {
        cerr << "Could not open scores.bin\n";
        return 1;
    }

    int index = 4;
    double replacement = 96.0;
    file.seekp(index * sizeof(double), ios::beg);
    file.write(reinterpret_cast<const char*>(&replacement),
               sizeof(replacement));

    if (!file) {
        cerr << "The update failed\n";
    }

    file.close();
    return 0;
}

Opening with both ios::in and ios::out permits an existing file to be updated without truncating it.

Validating the Position

input.seekg(0, ios::end);
streamoff fileSize = input.tellg();
int valueCount = fileSize / sizeof(double);

if (index < 0 || index >= valueCount) {
    cerr << "Index is outside the file\n";
    return 1;
}

Derive the number of values from the file size and reject invalid indices before seeking. Check the stream again after every binary read or write.

Choosing an Access Pattern

Need Access pattern
Human-readable records Sequential text
Variable-length records Sequential text
Process every stored value Sequential access
Jump directly to a fixed-size value Random-access binary
Replace one fixed-size value in place Random-access binary

Choose the file format and access pattern together; each constrains the other.

Summary

  • seekg() and seekp() move the next read or write position
  • tellg() and tellp() report the current positions
  • Fixed-size binary values make index × sizeof(type) predictable
  • Validate the index and stream state before using file data

Key Takeaways

  • String streams parse and format text using familiar stream operators
  • Stream state reports whether typed extraction succeeded
  • Random access moves directly to a calculated byte position
  • Fixed-size binary values make index-based offsets predictable
