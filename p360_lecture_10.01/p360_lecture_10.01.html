File I/O

Writing data that outlasts the program?

  • Every variable disappears when the program ends.
  • Files let you persist data between runs and share it with other programs.

Kinds of Files

Two access patterns:

  • Sequential — read or write from start to finish, in order
  • Random — jump directly to any byte offset in the file

Total slides: 13

The File I/O Process

  1. Include <fstream>
  2. Declare a file stream object
  3. Open the file
  4. Read or write data
  5. Close the file

Every file program follows this sequence. Steps 3–5 repeat for each file.

The fstream library

Include <fstream> to get three stream types:

  • ofstream — output (write) stream
  • ifstream — input (read) stream
  • fstream — bidirectional (read and write)

These work like cin and cout but are connected to a file instead of the console.

Opening and closing a file

#include <fstream>
#include <iostream>
using namespace std;

int main() {
    ofstream outFile;
    outFile.open("scores.txt");     // opens (creates if missing)

    outFile << "Alice" << " " << 95 << "\n";
    outFile << "Bob"   << " " << 87 << "\n";

    outFile.close();
    return 0;
}

If the file does not exist, ofstream creates it. If it exists, the default mode overwrites it from the beginning.

Always check if the file opened

ifstream inFile("scores.txt");

if (!inFile.is_open()) {
    cerr << "Error: could not open scores.txt\n";
    return 1;
}
  • is_open() returns false if the file does not exist, the path is wrong, or permissions block access
  • Reading from a failed stream silently produces garbage values
  • Always check — never assume the file is there

Overwrite vs. Append mode

By default, ofstream truncates the file every time it opens it:

ofstream out("log.txt");              // default: overwrite
out << "Run at 09:00\n";
out.close();

ofstream out2("log.txt");             // overwrites — previous content gone
out2 << "Run at 10:00\n";
out2.close();

To append instead, pass ios::app:

ofstream out("log.txt", ios::app);    // append mode
out << "Run at 10:00\n";              // adds to existing content
out.close();

When to use each mode

Mode Flag Behavior
Overwrite (default) Truncates file to zero bytes, writes fresh
Append ios::app Preserves existing content, adds at end

Use overwrite when regenerating data from scratch each run. Use append when building a log or accumulating data across multiple program runs.

Sequential write

#include <fstream>
#include <vector>
using namespace std;

int main() {
    vector<int> scores = {91, 78, 84, 95, 62};

    ofstream outFile("scores.txt");

    for (int score : scores) {
        outFile << score << "\n";     // one value per line
    }

    outFile.close();
    return 0;
}

The stream position advances automatically with each write — no manual tracking needed.

Sequential read

ifstream inFile("scores.txt");
vector<int> scores;
int value;

while (inFile >> value) {             // reads until EOF
    scores.push_back(value);
}

inFile.close();

inFile >> value returns a false stream state when extraction fails — including at end of file. That makes it a natural loop condition.

Writing a CSV file

A CSV file uses commas to separate fields. Writing one is just writing plain text:

#include <fstream>
using namespace std;

int main() {
    ofstream out("students.csv");

    out << "name,score,grade\n";      // header row
    out << "Alice,95,A\n";
    out << "Bob,78,C\n";
    out << "Carol,84,B\n";

    out.close();
    return 0;
}

Any program that reads CSV — spreadsheets, Python, databases — can open this file.

Reading a CSV file

#include <fstream>
#include <sstream>
#include <string>
#include <iostream>
using namespace std;

int main() {
    ifstream in("students.csv");
    string line;

    getline(in, line);                // skip header row

    while (getline(in, line)) {       // one line per iteration
        stringstream ss(line);
        string name, scoreStr, grade;

        getline(ss, name,     ',');   // read up to first comma
        getline(ss, scoreStr, ',');   // read up to second comma
        getline(ss, grade,    ',');   // read remainder

        cout << name << " scored " << scoreStr << "\n";
    }

    in.close();
    return 0;
}

getline(stream, string, delimiter) reads up to the delimiter instead of newline.

End-of-file detection

Two patterns you will see in the wild:

Preferred

while (inFile >> value) {
    // process value
}

Extraction fails cleanly at EOF. No off-by-one reads.

Avoid

while (!inFile.eof()) {
    inFile >> value;
    // may process stale value
}

eof() is set after a failed read — loop body runs once too many.

Summary

Files persist data beyond the program’s lifetime. Use ofstream to write and ifstream to read. Default mode overwrites; ios::app appends to existing content.

CSV files are plain text — write commas between fields, use getline with a delimiter to parse them back.

Sequential File I/O

Build it from the ground up

Three tasks, each building on the last. Get the mechanism right on a single value before adding a container.

  • Task 1: Write one integer to a file, then read it back
  • Task 2: Write multiple values to a file, then read them all back into a vector

Total slides: 10

Task 1 — write one value

Write the integer 42 to a file called data.txt.

#include <fstream>
using namespace std;

int main() {
    // your code here
    return 0;
}

Work with your partner. Three minutes.

Solution — write one value

#include <fstream>
using namespace std;

int main() {
    ofstream outFile("data.txt");
    if (!outFile.is_open()) {
        return 1;
    }

    outFile << 42 << "\n";
    outFile.close();
    return 0;
}

Task 1 continued — read it back

Open data.txt and read the integer back into a variable. Print it.

#include <fstream>
#include <iostream>
using namespace std;

int main() {
    // your code here
    return 0;
}

Expected output: 42

Solution — read one value

#include <fstream>
#include <iostream>
using namespace std;

int main() {
    ifstream inFile("data.txt");
    if (!inFile.is_open()) {
        return 1;
    }

    int value;
    inFile >> value;
    inFile.close();

    cout << value << "\n";
    return 0;
}

Task 2 — write a vector to a file

Now with a vector. Write each score to scores.txt, one per line.

vector<int> scores = {91, 78, 84, 95, 62};

Expected file contents:

91
78
84
95
62

Work with your partner. Five minutes.

Task 2 continued — read the vector back

Open scores.txt and read every integer into a new vector. Print each value.

vector<int> loaded;
// ... your code here

for (int s : loaded) {
    cout << s << "\n";
}

Expected output: the same five values, same order.

Solution — write the vector

#include <fstream>
#include <vector>
using namespace std;

int main() {
    vector<int> scores = {91, 78, 84, 95, 62};

    ofstream outFile("scores.txt");
    if (!outFile.is_open()) { return 1; }

    for (int score : scores) {
        outFile << score << "\n";   // one per line
    }

    outFile.close();
    return 0;
}

Solution — read the vector back

#include <fstream>
#include <iostream>
#include <vector>
using namespace std;

int main() {
    ifstream inFile("scores.txt");
    if (!inFile.is_open()) { return 1; }

    vector<int> loaded;
    int value;

    while (inFile >> value) {       // stops at EOF
        loaded.push_back(value);
    }

    inFile.close();

    for (int s : loaded) {
        cout << s << "\n";
    }
    return 0;
}

Summary

The pattern is the same whether you are writing one value or a hundred: open a stream, transfer data, close. Starting with a single value isolates the mechanism before adding container complexity.

The read loop condition — while (inFile >> value) — handles any file length without a separate counter or size variable.

Random Access

Reading the 100th record without reading the first 99?

Sequential access reads from the start — every record in order. Random access jumps directly to any byte offset in the file using a position marker.

Total slides: 10

Binary mode

Text mode interprets certain byte sequences as control characters. Binary mode disables that — bytes are read and written as-is.

#include <fstream>
using namespace std;

ifstream inFile("data.bin", ios::binary);

Random access on structured data almost always uses binary mode. Text files work for sequential I/O; binary files are needed for reliable byte-offset arithmetic.

seekg() — moving the read position

seekg moves the internal read position to any byte in the file.

inFile.seekg(byte_count, origin);
Origin Meaning
ios::beg Count from the beginning
ios::end Count from the end
ios::cur Count from the current position

seekg() examples

ifstream inFile("data.bin", ios::binary);

inFile.seekg(0,   ios::beg);   // go to byte 0 (start)
inFile.seekg(0,   ios::end);   // go to last byte (end)
inFile.seekg(40,  ios::beg);   // go to byte 40
inFile.seekg(-8,  ios::cur);   // back 8 bytes from current

Negative offsets move backward. ios::end with a negative offset is a common way to step back from the end of a file.

tellg() — where is the read position?

ifstream inFile("data.bin", ios::binary);

inFile.seekg(0, ios::end);
streampos fileSize = inFile.tellg();      // size in bytes

cout << "File is " << fileSize << " bytes\n";

inFile.seekg(0, ios::beg);               // rewind before reading

tellg() returns the current read position as a streampos value. Common use: seek to end, call tellg() to get file size, then seek back to the beginning.

seekp() and tellp() — the write position

fstream file("data.bin",
    ios::in | ios::out | ios::binary);

file.seekp(40, ios::beg);                 // move write position to byte 40
file.write(reinterpret_cast<char*>(&value), sizeof(value));

streampos writePos = file.tellp();        // where did we land?
  • seekp / tellp operate on the write position
  • seekg / tellg operate on the read position
  • On a bidirectional fstream, both positions exist independently

Fixed-size records

Random access only works predictably when every record occupies the same number of bytes.

struct Record {
    int   id;       // 4 bytes
    float score;    // 4 bytes
    char  grade;    // 1 byte
    char  pad[3];   // 3 bytes padding
};                  // total: 12 bytes per record
  • Text files have variable-length lines — byte offsets are not predictable
  • Binary files with fixed-size structs make offset arithmetic exact
  • Always use sizeof(Record) — never hardcode the byte count

[Graphic suggestion: diagram showing a binary file as a row of equal-width blocks, each labeled “Record 0”, “Record 1”, “Record 2”, with byte offsets marked below]

Reading the Nth record

To reach record N in a file of fixed-size records:

byte offset = N × sizeof(Record)
struct Record { int id; float score; char grade; char pad[3]; };

ifstream inFile("records.bin", ios::binary);

int n = 4;                                         // 0-indexed: 5th record
inFile.seekg(n * sizeof(Record), ios::beg);

Record r;
inFile.read(reinterpret_cast<char*>(&r), sizeof(Record));

cout << "id: "    << r.id
     << "  score: " << r.score
     << "  grade: " << r.grade << "\n";

Writing the Nth record

The same formula applies to writing. Seek to the offset, then write.

fstream file("records.bin",
    ios::in | ios::out | ios::binary);

Record updated = {42, 99.5f, 'A', {0, 0, 0}};

int n = 4;
file.seekp(n * sizeof(Record), ios::beg);
file.write(reinterpret_cast<char*>(&updated), sizeof(Record));

file.close();

reinterpret_cast<char*> tells write to treat the struct as raw bytes. The existing record at that offset is overwritten in place.

Summary

Random access jumps the position marker with seekg/seekp instead of reading every record in sequence. Fixed-size records make the byte offset predictable: N × sizeof(Record).

Use tellg/tellp to query the current position — useful for computing file size and for debugging offset arithmetic.

Key Takeaways

  • Files persist data beyond the program’s lifetime — use ofstream to write, ifstream to read
  • Always check is_open() before reading or writing
  • Sequential access reads from start to finish; random access jumps directly to any byte offset
  • Fixed-size records make random access predictable: offset = N × sizeof(Record)
