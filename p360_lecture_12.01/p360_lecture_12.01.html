Arrays with Functions

What Crosses the Function Boundary?

  • Pass one element when a function needs one value
  • Pass the array when a function needs the collection

Slides 3–17

Array Parameter Syntax

void displayScores(const int scores[], int size);
void addBonus(int scores[], int size, int bonus);
  • The brackets mark an array parameter
  • The separate size parameter controls traversal
  • const marks a read-only array contract

The Array Is Not Copied

  • A scalar argument gives the function its own value
  • An array argument gives the function access to the caller’s elements
  • Writing through the parameter changes the original array

[Graphic suggestion: show a caller array with one arrow from the function parameter to the first element. Contrast it with a separate copied scalar value.]

Array Decay

When a built-in array crosses a function boundary, it decays into a pointer to its first element.

At the call site Inside the function
Complete array Address of element 0
Element count known from declaration Element count unavailable
sizeof reports all element bytes sizeof reports pointer bytes

Where Did the Size Go?

void displayScores(const int scores[], int size) {
    for (int i = 0; i < size; i++) {
        cout << scores[i] << "\n";
    }
}

The function can access elements through the address, but the address carries no element count. The caller must provide the logical size.

sizeof Inside and Outside a Function

int scores[5] = {91, 78, 84, 95, 62};

int count = sizeof(scores) / sizeof(scores[0]);
displayScores(scores, count);
  • The count expression works where scores is declared
  • After decay, the parameter is no longer a complete array

Does the Declared Size Matter?

void process(int values[]);
void process(int values[5]);
void process(int* values);

The 5 does not enforce the argument’s size. Pass the logical size separately.

Read-Only Array Parameters

int findMaximum(const int values[], int size);
  • const prevents assignments to array elements inside the function
  • It communicates that the function observes rather than modifies
  • Use it for display, aggregation, and search functions

Displaying an Array

#include <iostream>
using namespace std;

void displayScores(const int scores[], int size);

int main() {
    int scores[4] = {91, 78, 84, 95};
    displayScores(scores, 4);
    return 0;
}

void displayScores(const int scores[], int size) {
    for (int i = 0; i < size; i++) {
        cout << scores[i] << "\n";
    }
}

Modifying an Array

void addBonus(int scores[], int size, int bonus);
  • No const means the function may write to elements
  • The caller sees every change because the elements were not copied
  • The function name should make the mutation clear

Updating Every Element

#include <iostream>
using namespace std;

void addBonus(int scores[], int size, int bonus);

int main() {
    int scores[4] = {70, 75, 80, 85};
    addBonus(scores, 4, 5);
    cout << scores[0] << " " << scores[3] << "\n";
    return 0;
}

void addBonus(int scores[], int size, int bonus) {
    for (int i = 0; i < size; i++) {
        scores[i] += bonus;
    }
}

The output is 75 90. Trace one assignment back to the same indexed element in the caller’s array.

One Element or the Whole Array?

Call Function receives Typical purpose
classify(scores[i]) One copied int Analyze one score
displayScores(scores, size) Access to all elements Read the collection
addBonus(scores, size, 5) Access to all elements Modify the collection

Choosing the Parameter Contract

Function needs to… Parameter style
Use one value int value
Read an array const int values[], int size
Modify an array int values[], int size

The prototype should reveal the function’s data movement before anyone reads its body.

Summary

  • Built-in arrays decay at the function boundary, so pass the size separately
  • Use const for read-only access; omit it only when caller-visible mutation is intentional

Array Algorithms

What Can One Traversal Discover?

  • A total or average
  • A minimum, maximum, or matching position
  • A new arrangement of the elements

Slides 18–37

From Loop Pattern to Algorithm

An algorithm is a repeatable sequence of steps that solves a category of problems.

Pattern State carried through the loop
Aggregate Accumulator
Track Best value or index so far
Search Match position or sentinel
Sort Boundary between sorted and unsorted data

The Traversal Shape

for (int i = 0; i < size; i++) {
    // Process values[i]
}
  • Start at index 0
  • Stop before the logical size
  • Update algorithm state once per visited element

Computing a Sum

int sumArray(const int values[], int size) {
    int total = 0;

    for (int i = 0; i < size; i++) {
        total += values[i];
    }

    return total;
}

Computing an Average

double averageArray(const int values[], int size) {
    int total = sumArray(values, size);
    return static_cast<double>(total) / size;
}
  • Reuse the sum algorithm instead of duplicating its loop
  • Convert before division to preserve the fractional result
  • Require size > 0 as part of the function contract

What If the Array Is Empty?

Algorithm Empty-array behavior
Sum Return 0
Search Return -1
Sort Do nothing
Average Requires at least one element
Minimum or maximum Requires at least one element

Preconditions belong in the function contract.

Finding a Minimum or Maximum

  • Seed the tracker with values[0]
  • Begin comparisons at index 1
  • Replace the tracker whenever a better value appears

Avoid arbitrary seeds such as 0; an all-negative array would break a maximum tracker initialized that way.

Maximum Function

int findMaximum(const int values[], int size) {
    int maximum = values[0];

    for (int i = 1; i < size; i++) {
        if (values[i] > maximum) {
            maximum = values[i];
        }
    }

    return maximum;
}

Searching for a Value

values[i] == target
  • A match ends the search
  • Reaching size means no match exists
  • The matching index preserves the value’s location

Representing “Not Found”

Valid indexes begin at 0, so -1 can represent no matching position.

int position = linearSearch(scores, size, target);

if (position == -1) {
    cout << "Not found\n";
}

Linear Search Function

int linearSearch(const int values[], int size, int target) {
    for (int i = 0; i < size; i++) {
        if (values[i] == target) {
            return i;
        }
    }

    return -1;
}

How Much Work Does Search Do?

Situation Comparisons for N elements
Target is first 1
Target is near the middle About N / 2
Target is last or absent N

Linear search grows in direct proportion to the array size: O(N).

Sorting by Neighbor Comparisons

Bubble sort repeats two actions:

  • Compare adjacent elements
  • Swap them when the left value is larger

After one complete pass, the largest unsorted value has moved to the end of the unsorted region.

Why Swapping Needs Temporary Storage

int temp = values[i];
values[i] = values[i + 1];
values[i + 1] = temp;

The temporary variable preserves the first value before its array position is overwritten.

One Bubble Sort Pass

Comparison Array state
Start 5 3 8 1
Compare 5 and 3 3 5 8 1
Compare 5 and 8 3 5 8 1
Compare 8 and 1 3 5 1 8

The value 8 is now in its final position.

[Graphic suggestion: animate the active adjacent pair and move the larger value one cell to the right after each comparison.]

Repeating the Pass

for (int pass = 0; pass < size - 1; pass++) {
    for (int i = 0; i < size - 1 - pass; i++) {
        // Compare values[i] and values[i + 1]
    }
}
  • The outer loop counts completed passes
  • The inner bound shrinks because the largest remaining values are already placed

Bubble Sort Function

void bubbleSort(int values[], int size) {
    for (int pass = 0; pass < size - 1; pass++) {
        for (int i = 0; i < size - 1 - pass; i++) {
            if (values[i] > values[i + 1]) {
                int temp = values[i];
                values[i] = values[i + 1];
                values[i + 1] = temp;
            }
        }
    }
}

What Does Each Algorithm Promise?

Algorithm Result Mutates array? Precondition
Sum Total No None
Average Mean No Nonempty
Maximum Largest value No Nonempty
Linear search Index or -1 No None
Bubble sort Sorted elements Yes None

Summary

  • Accumulators, trackers, and sentinels turn traversal into reusable algorithms
  • Search reports a position; in-place sorting changes the caller’s array

Parallel Arrays

Can One Index Describe a Whole Record?

  • Separate arrays can store separate attributes
  • A shared index connects the attributes for one entity

Slides 38–56

A Parallel-Array Dataset

Index names scores present
0 Ada 91 true
1 Linus 78 false
2 Grace 95 true

At index 2, every value describes Grace.

[Graphic suggestion: draw three horizontal arrays with vertical alignment bands connecting elements that share each index.]

Declaring Parallel Arrays

const int CAPACITY = 5;

string names[CAPACITY];
int scores[CAPACITY];
bool present[CAPACITY];
  • Every array uses the same capacity
  • Each position has the same meaning across all arrays

Capacity and Logical Size

Name Meaning
CAPACITY Maximum number of records the arrays can hold
size Number of records currently stored

All loops use the same logical size, not the full capacity.

The Shared-Index Contract

For every active index i:

  • names[i] identifies the entity
  • scores[i] stores that entity’s score
  • present[i] stores that entity’s attendance

If one array moves without the others, the dataset becomes incorrect.

Filling the Arrays Together

for (int i = 0; i < size; i++) {
    cout << "Name: ";
    cin >> names[i];
    cout << "Score: ";
    cin >> scores[i];
    cout << "Present (1 or 0): ";
    cin >> present[i];
}

Traversing Related Data

for (int i = 0; i < size; i++) {
    cout << names[i] << " "
         << scores[i] << " "
         << present[i] << "\n";
}
  • One loop controls every array
  • The index selects one complete logical record

Passing Parallel Arrays to a Function

void displayRecords(const string names[],
                    const int scores[],
                    const bool present[],
                    int size);
  • Pass every required field array
  • Pass one shared logical size
  • Use const when the function only reads records

Displaying Records

void displayRecords(const string names[],
                    const int scores[],
                    const bool present[],
                    int size) {
    for (int i = 0; i < size; i++) {
        cout << names[i] << " "
             << scores[i] << " "
             << present[i] << "\n";
    }
}

Searching One Array

int index = findName(names, size, targetName);
  • Search names for the requested student
  • Keep the returned index
  • Use that index only when it is not -1

Using the Matching Index

if (index != -1) {
    cout << names[index] << "\n";
    cout << scores[index] << "\n";
    cout << present[index] << "\n";
}

The search result locates the entire logical record, not only the matched name.

Finding the Best Record

  • Initialize bestIndex to 0
  • Compare each scores[i] with scores[bestIndex]
  • Return the index of the highest score

An index preserves access to every related field.

Best-Record Function

int findHighestIndex(const int scores[], int size) {
    int bestIndex = 0;

    for (int i = 1; i < size; i++) {
        if (scores[i] > scores[bestIndex]) {
            bestIndex = i;
        }
    }

    return bestIndex;
}

Moving One Record

swap(scores[i], scores[i + 1]);
swap(names[i], names[i + 1]);
swap(present[i], present[i + 1]);

Moving only the scores would attach them to the wrong names.

Sorting Parallel Arrays

for (int pass = 0; pass < size - 1; pass++) {
    for (int i = 0; i < size - 1 - pass; i++) {
        if (scores[i] > scores[i + 1]) {
            swap(scores[i], scores[i + 1]);
            swap(names[i], names[i + 1]);
            swap(present[i], present[i + 1]);
        }
    }
}

When Parallel Arrays Work Well

  • The dataset is small and fixed-capacity
  • Every operation uses the same logical size
  • Only a few attributes must stay aligned
  • The shared-index contract remains easy to inspect

Where the Design Strains

  • Every new field adds another parameter and coordinated operation
  • Sorting or moving records requires updating every array
  • A structure can package all fields for one entity into a single value

Summary

  • Parallel arrays use one shared index to connect related values
  • Capacity, logical size, traversal, and movement must remain synchronized

Key Takeaways

  • Built-in arrays decay at function boundaries, so functions receive an address and need a separate logical size
  • Array algorithms combine traversal with accumulators, trackers, sentinels, or coordinated swaps
  • Parallel arrays represent one record through a shared index that must remain synchronized
