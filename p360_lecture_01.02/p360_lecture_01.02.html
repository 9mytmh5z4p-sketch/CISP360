Variables and Assignment

A Named Spot In Memory

  • A variable is a named memory location that holds a value.
  • The value can change while the program runs.
  • A literal is fixed in the source code; a variable is under program control.

Slides 3-14 Frame variables as the first tool students get for making programs respond to changing data.

What Declaration Does

  • Declaring a variable asks the computer to reserve a named spot in memory.
  • The type tells the computer how much memory to reserve.
  • The name is how the program refers to that spot later.
Variable Address Value
age 0xFA2 unknown
name 0xFA6 unknown

The addresses are illustrative. Students only need the idea that a variable lives somewhere specific in memory. [Graphic suggestion: redraw the declaration/init memory model as labeled memory boxes.]

Declaration Syntax

int age;
double price;
char grade;
string name;
bool isOver;

The pattern is type, then name, then semicolon. These declarations create variables, but they do not give most of them useful values yet.

Initialization

  • Declaration reserves the memory slot.
  • Initialization puts the first value into it.
  • Always initialize a variable before reading from it.
int age;      // declared; value is unknown
age = 25;     // initialized; value is 25

Keep the two-step version first so students can see that declaration and initialization are separate actions.

Assignment Is Not Equality

  • In C++, = is the assignment operator.
  • It means: evaluate the right side, then store the result in the variable on the left.
  • The old value is replaced.
int age = 25;
age = age + 1;    // age becomes 26

This is the key math-to-programming correction. The second line is impossible as algebra but normal as a programming instruction.

Declaration And Initialization Together

  • Most variables should be declared and initialized in one statement.
  • This gives the variable a name, a type, and a useful first value immediately.
int age = 25;
double price = 19.95;
char grade = 'A';
string name = "Pat";
bool isPassing = true;

This is the form students should write most often at the beginning of the course.

Constants

  • const makes a variable read-only after initialization.
  • Use it for values that should not change while the program runs.
  • A constant must be initialized when it is declared.
const double TAX_RATE = 0.0825;
const int MAX_SCORE = 100;

TAX_RATE = 0.09;    // error

Use constants for values that represent a rule, limit, or fixed setting. The compiler enforces the promise.

Other Initialization Forms

Form Example What to Know
Default int age; no useful value yet
Direct int age = 10; course default
Value int age{}; zero-initializes
List int age{10}; modern C++ form

The goal is recognition, not memorization. Students will see brace forms in references and examples outside class.

Which Form Should I Use?

  • Start with direct initialization: int age = 10;.
  • Use const when the value should not change.
  • Recognize brace initialization when reading modern C++ examples.

Direct initialization keeps the type and starting value visible, which helps beginners read their own code.

Variable Naming Rules

  • Must start with a letter or underscore, not a digit.
  • Can contain letters, digits, and underscores.
  • Cannot be a reserved word.
  • Names are case-sensitive.
int age;
double taxRate;
string first_name;

int 2fast;      // error
int return;     // error

Variable names should usually be nouns because variables hold things: age, score, total, name, taxRate.

Summary

  • A variable is a named, typed memory location.
  • Declaration reserves the space; initialization fills it.
  • Assignment updates the stored value.

Data Types

Why So Many Types?

  • Type controls memory: each type reserves a specific amount of space.
  • Type controls meaning: the same bits can be interpreted different ways.
  • Type controls behavior: operations depend on the kinds of values involved.

Slides 15-31 Frame type as a contract between the programmer and the computer, not as arbitrary vocabulary.

The Type Families

Type Keyword Holds Typical Use
Integer int whole numbers counts, indexes, IDs
Floating point double decimal numbers measurements, averages
Character char one character grades, initials
String string text names, messages
Boolean bool true or false flags, conditions
Void void no value function return type
Auto auto compiler-deduced modern convenience

This slide is the map. Students will not master every row today, but they should know what each family is for.

Integers

  • Integers store whole numbers only.
  • Use int for counts, indexes, ages, scores, and IDs.
  • Integers are exact, but they cannot store decimal parts.
int age = 25;
int count = -3;
int students = 0;

Connect this back to arithmetic: when integer values divide, the result can still be integer behavior.

Integer Sizes

  • C++ has several integer sizes because different programs need different ranges.
  • More bytes means a larger range.
  • Unsigned types trade negative values for a larger positive range.
Type Typical Size Use
short 2 bytes small whole numbers
int 4 bytes default whole-number choice
long long 8 bytes very large whole numbers
unsigned int 4 bytes positive-only values

Avoid range memorization. The pattern matters: size affects range, and signedness affects whether negative values are allowed.

Unsigned Integers

  • An unsigned integer cannot represent negative values.
  • Assigning a negative value can wrap around to a large positive number.
  • Use unsigned types only when the domain truly cannot go below zero.
unsigned int speed = 100;
unsigned int error = -1;    // compiles; stores a large value

This is a preview of a bug class that compiles cleanly but produces the wrong answer.

Which Integer Should I Use?

  • In this course, start with plain int.
  • Use long long only when the range might exceed about two billion.
  • Avoid unsigned types unless the positive-only rule is part of the problem.
Stored Value Good Type
age int
score int
population long long
menu choice int

The point is not to optimize memory in week one. The point is to choose a type that matches the meaning of the value.

Floating Point Numbers

  • Floating point types store numbers with decimal parts.
  • Most decimal values are approximations in memory.
  • Use floating point values for measurements and averages, not counts.
double price = 99.95;
double pi = 3.14159265358979;
double tiny = 1.5e-10;

Students do not need the binary encoding yet. They do need the habit that floating point math is approximate.

Float Vs. Double

  • float stores about 6 to 7 significant digits.
  • double stores about 15 significant digits.
  • In this course, use double.
float small = 10.0 / 3.0;
double precise = 10.0 / 3.0;

The cost of double is rarely important in this course. The extra precision is useful immediately.

Characters

  • char stores one character.
  • Character literals use single quotes.
  • A character digit is not the same as a numeric digit.
char grade = 'A';
char letter = 'z';
char digit = '5';    // character, not number

Single quotes for char, double quotes for string. This is a small syntax distinction that prevents many early compile errors.

Char And ASCII

  • A char is stored as a small integer.
  • ASCII maps those integer values to visible characters.
  • Character arithmetic works because characters have numeric codes.
char ch = 'M';    // stored as 77
int code = ch;    // code is 77
ch = ch + 1;      // now displays as 'N'

[Graphic suggestion: small ASCII table showing A, M, N, a, and 0 with their decimal codes.]

Strings

  • string stores text: a sequence of characters.
  • String literals use double quotes.
  • Use #include <string> when working with strings.
#include <string>

string name = "Pat Smith";
string message = "Hello";
string empty = "";

For now, students only need declaration and initialization. Member functions can wait.

Booleans

  • bool stores one of two values: true or false.
  • Boolean variables are used for flags and conditions.
  • By default, cout prints true as 1 and false as 0.
bool isOpen = true;
bool isEmpty = false;

cout << isOpen << endl;     // 1
cout << isEmpty << endl;    // 0

Booleans will become central when selection and loops arrive. For now, treat them as yes/no storage.

Void

  • void means no value.
  • You cannot create a normal variable with type void.
  • Later, void will describe functions that do not return a value.
void someValue;     // error
void result = 99;   // error

Keep this separate from auto. void is the absence of a value, which is conceptually different from deducing a value’s type.

Auto

  • auto tells the compiler to deduce the type from the initializer.
  • The variable must be initialized immediately.
  • Explicit types are usually clearer while learning.
auto age = 78;        // int
auto cost = 99.95;    // double
auto grade = 'A';     // char

Students should recognize auto in code they read, but they should usually write explicit types while building the type model.

Which Type Should I Use?

I Need To Store… Use
a whole number int
a decimal number double
one character char
text string
a yes/no flag bool
no return value void

This is the practical decision table students can use while writing first programs.

Summary

  • Type controls memory, interpretation, and allowed operations.
  • Use int for whole numbers and double for decimals.
  • Use string for text and bool for true/false flags.

Arithmetic Operators

Tools For Transforming Data

  • Structured programs store data, transform it, and produce output.
  • Arithmetic operators are the first tools for computing new values.
  • Operator behavior depends on the types of the operands.

Slides 32-45 Connect this section back to the larger course framing: data plus transformations.

The Five Arithmetic Operators

Operator Meaning Example Result
+ addition 4 + 2 6
- subtraction 4 - 2 2
* multiplication 4 * 2 8
/ division 4 / 2 2
% modulus 5 % 2 1

Flag the symbols that differ from handwritten math: *, /, and %.

Arithmetic In Code

#include <iostream>
using namespace std;

int main()
{
    double hats = 3.14;
    double heads = 7.89;

    cout << "sum: " << hats + heads << endl;
    cout << "product: " << hats * heads << endl;

    return 0;
}

Expressions can appear directly inside output statements. They are evaluated first, then the result is printed.

Integer Division

  • When both operands are integers, / performs integer division.
  • The remainder is dropped.
  • This is truncation, not rounding.
int a = 9 / 5;    // 1
int b = 1 / 2;    // 0
int c = 7 / 2;    // 3

This is the first place where type visibly changes arithmetic behavior.

Floating Point Division

  • If either operand is floating point, the result is floating point.
  • The operands determine the division behavior before assignment happens.
double a = 9.0 / 5;    // 1.8
double b = 9 / 5.0;    // 1.8
double c = 9 / 5;      // 1.0

int d = 9.0 / 5;       // 1

The third line is the trap: storing into a double is too late if the expression already used integer division.

Division By Zero

  • Integer division by zero is undefined behavior and often crashes.
  • Floating point division by zero may produce infinity or NaN.
  • The compiler may accept both because the syntax is legal.

Make the distinction between compile-time syntax checks and runtime behavior.

The Compiler Is Not Your Safety Net

#include <iostream>
using namespace std;

int main()
{
    int a = 9;
    double b = 9.0;
    int z = 0;

    cout << a / z << endl;
    cout << b / z << endl;

    return 0;
}

The program can compile because the grammar is valid. The bug appears when the program runs.

The Modulus Operator

  • % gives the remainder after integer division.
  • It works with integers only.
  • It is useful for divisibility, even/odd checks, and extracting digits.
cout << 10 % 3 << endl;    // 1
cout << 15 % 5 << endl;    // 0
cout << 7 % 2 << endl;     // 1
cout << 42 % 10 << endl;   // 2

Modulus appears later in loops, games, menus, and algorithms that wrap around a range.

Mathematical Expressions

  • An expression evaluates to a single value.
  • That value can be stored, printed, or used inside a larger expression.
int result1 = 5 + 3 * 2;      // 11
int result2 = (5 + 3) * 2;    // 16
int result3 = 10 - 6 / 2;     // 7
double avg = (a + b + c) / 3.0;

Use the average line to reinforce floating point division from integer inputs.

Order Of Precedence

Priority Operator Meaning
1 () parentheses
2 - unary negation
3 * / % multiply, divide, modulus
4 + - add, subtract
  • When intent is not obvious, add parentheses.

Students do not need the full C++ precedence table yet. This is the arithmetic subset they need today.

Associativity

  • When operators have equal precedence, arithmetic evaluates left to right.
int a = 10 - 3 - 2;    // (10 - 3) - 2 = 5
int b = 12 / 6 / 2;    // (12 / 6) / 2 = 1

Associativity explains cases where precedence alone does not decide the grouping.

The cmath Header

  • C++ has five built-in arithmetic operators.
  • Powers, square roots, trigonometry, and logarithms are functions.
  • Those functions require #include <cmath>.
#include <cmath>

double root = sqrt(16.0);
double power = pow(2.0, 10.0);
double distance = abs(-7.5);

Brief recognition slide only. The main point is that ^ is not exponentiation in C++.

Summary

  • Arithmetic operators transform numeric data.
  • Integer division drops the remainder.
  • Precedence and associativity determine evaluation order; parentheses make intent clear.

Key Takeaways

  • Variables are named, typed memory locations.
  • Declaration reserves space; initialization gives a first value; assignment updates the value.
  • Type controls memory, interpretation, and operator behavior.
  • Use int for whole numbers, double for decimals, string for text, and bool for flags.
  • Integer division drops the remainder unless at least one operand is floating point.

Appendix

Variables and Assignment: Good Practices

  • Initialize variables where they are declared.
  • Name variables after the value they represent.
  • Use const for values that should not change.

Variables and Assignment: Common Mistakes

  • Reading a variable before assigning it a useful value.
  • Treating = like mathematical equality.
  • Using vague names such as x, num, or temp when the value has a clear meaning.

Variables and Assignment: More Examples

int total = 0;
int firstScore = 85;
int secondScore = 92;

total = firstScore + secondScore;

cout << "Total: " << total << endl;

Data Types: Good Practices

  • Use int for whole-number counts and indexes.
  • Use double for decimal calculations.
  • Use bool for yes/no states instead of storing flags as integers.

Data Types: Common Mistakes

  • Storing decimal values in int variables.
  • Confusing 'A' with "A".
  • Using auto before you can confidently predict the deduced type.

Arithmetic Operators: Good Practices

  • Use a floating point operand when a division should produce a decimal result.
  • Add parentheses when operator order is not obvious.
  • Check a denominator before dividing by a variable.

Arithmetic Operators: Common Mistakes

  • Expecting integer division to round.
  • Expecting ^ to mean exponentiation.
  • Dividing by zero and assuming the compiler will catch it.

Arithmetic Operators: More Examples

int onesDigit = total % 10;
