File I/O

Writing data that outlasts the program?

Every variable disappears when the program ends. Files let you persist data between runs and share it with other programs.

This section covers how C++ reads from and writes to files — and the difference between sequential and random access.

Total slides: 10

The fstream library

Include <fstream> to get three stream types:

  • ofstream — output (write) stream
  • ifstream — input (read) stream
  • fstream — bidirectional (read and write)

These work like cin and cout but are connected to a file instead of the console.

Opening and closing a file

#include <fstream>
#include <string>

int main() {
    std::ofstream outFile;
    outFile.open("scores.txt");   // open for writing

    outFile << "Alice" << " " << 95 << "\n";
    outFile << "Bob"   << " " << 87 << "\n";

    outFile.close();              // flush and release

    std::ifstream inFile("scores.txt"); // open in constructor
    // ...
    inFile.close();
}

You can open in the constructor or call open() separately. Always call close() when done.

Always check if the file opened

std::ifstream inFile("scores.txt");

if (!inFile.is_open()) {            // check before using
    std::cerr << "Error: could not open scores.txt\n";
    return 1;
}
  • is_open() returns false if the file doesn’t exist, the path is wrong, or permissions block access
  • Reading from a failed stream silently produces garbage values
  • Always check — never assume the file is there

Sequential access — writing

std::vector<int> scores = {91, 78, 84, 95, 62};

std::ofstream outFile("scores.txt");

for (int score : scores) {
    outFile << score << "\n";     //.......... one value per line
}

outFile.close();

Sequential write: values are appended one after another from the beginning of the file. The stream position advances automatically with each write.

Sequential access — reading

std::ifstream inFile("scores.txt");
std::vector<int> scores;
int value;

while (inFile >> value) {         //.......... reads until EOF
    scores.push_back(value);
}

inFile.close();

inFile >> value returns a falsy stream state when extraction fails — including at end of file. That makes it a natural loop condition.

End-of-file detection

Two patterns you will see in the wild:

Preferred

while (inFile >> value) {
    // process value
}

Extraction fails cleanly at EOF. No off-by-one reads.

Avoid

while (!inFile.eof()) {
    inFile >> value;
    // may process stale value
}

eof() is set after a failed read — loop body runs once too many.

Random access — the file position

Every open file has a position marker — an integer byte offset from the start of the file.

  • Sequential reads/writes move the marker forward automatically
  • Random access lets you jump the marker to any byte offset before reading or writing
Function Stream Purpose
seekg(offset, origin) ifstream Move read position
seekp(offset, origin) ofstream Move write position
tellg() ifstream Read current position
tellp() ofstream Write current position

Seek origins

inFile.seekg(0, std::ios::beg);   //.......... jump to byte 0
inFile.seekg(0, std::ios::end);   //.......... jump to end of file
inFile.seekg(-10, std::ios::cur); //.......... back 10 bytes from here

Three origin constants:

  • std::ios::beg — from the start of the file
  • std::ios::end — from the end of the file
  • std::ios::cur — from the current position

Positive offsets move forward; negative offsets move backward (from cur or end).

Summary

Files persist data beyond the program’s lifetime. Use ofstream to write, ifstream to read, and always check is_open() before proceeding.

Sequential access reads or writes from start to finish. Random access jumps the position marker with seekg/seekp to reach any byte directly.

Sequential File I/O

Let’s write it

Two tasks. Work through them in order — the second builds on the first.

  • Task 1: Write a vector of integers to a file, one value per line
  • Task 2: Read that file back into a new vector and print it

Total slides: 6

Task — write scores to a file

Given this vector, write each score to scores.txt, one per line.

std::vector<int> scores = {91, 78, 84, 95, 62};

Expected file contents:

91
78
84
95
62

Work with your partner. Five minutes.

Task — read scores back into a vector

Now open scores.txt and read every integer back into a new vector.

std::vector<int> loaded;
// ... your code here

// after reading:
for (int s : loaded) {
    std::cout << s << "\n";
}

Expected output: the same five values in the same order.

Solution — write

std::vector<int> scores = {91, 78, 84, 95, 62};

std::ofstream outFile("scores.txt");
if (!outFile.is_open()) {
    std::cerr << "Could not open file\n";
    return 1;
}

for (int score : scores) {
    outFile << score << "\n";   //.......... one per line
}

outFile.close();

Solution — read

std::ifstream inFile("scores.txt");
if (!inFile.is_open()) {
    std::cerr << "Could not open file\n";
    return 1;
}

std::vector<int> loaded;
int value;

while (inFile >> value) {       //.......... stops at EOF
    loaded.push_back(value);
}

inFile.close();

for (int s : loaded) {
    std::cout << s << "\n";
}

Summary

Writing a vector to a file: open an ofstream, loop with <<, close. Reading it back: open an ifstream, loop with >> as the condition, close.

The round-trip — write then read — is the foundation of every file-based program.

Random Access

Reading the 100th record without reading the first 99?

Sequential access reads from the start — every record before the target is processed first. Random access jumps directly to any byte offset.

This section covers the position marker functions and the fixed-size record pattern that makes random access practical.

Total slides: 8

seekg() — moving the read position

std::ifstream inFile("data.bin", std::ios::binary);

inFile.seekg(0, std::ios::beg);   //.......... go to byte 0
inFile.seekg(0, std::ios::end);   //.......... go to last byte
inFile.seekg(40, std::ios::beg);  //.......... go to byte 40
inFile.seekg(-8, std::ios::cur);  //.......... back 8 bytes from current

seekg takes two arguments: a byte count and an origin. The origin tells seekg where to count from — beginning, end, or current position.

tellg() — where is the read position?

std::ifstream inFile("data.bin", std::ios::binary);

inFile.seekg(0, std::ios::end);
std::streampos fileSize = inFile.tellg();   // size in bytes

std::cout << "File is " << fileSize << " bytes\n";

inFile.seekg(0, std::ios::beg);   // rewind before reading

tellg() returns the current read position as a std::streampos value. Common use: seek to end, call tellg() to get the file size, then seek back to the beginning.

seekp() and tellp() — the write position

std::fstream file("data.bin",
    std::ios::in | std::ios::out | std::ios::binary);

file.seekp(40, std::ios::beg);    //.......... move write position to byte 40
file.write(reinterpret_cast<char*>(&value), sizeof(value));

std::streampos writePos = file.tellp();   // where did we land?
  • seekp / tellp operate on the write position
  • seekg / tellg operate on the read position
  • On a bidirectional fstream, both positions exist independently

Fixed-size records

Random access only works predictably when every record occupies the same number of bytes.

struct Record {
    int   id;       // 4 bytes
    float score;    // 4 bytes
    char  grade;    // 1 byte
    char  pad[3];   // 3 bytes padding (compiler may add anyway)
};                  // total: 12 bytes per record
  • Text files have variable-length lines — you can’t calculate byte offsets
  • Binary files with fixed-size structs make offset arithmetic exact
  • Always use sizeof(Record) — never hardcode the byte count

[Graphic suggestion: diagram showing a binary file as a row of equal-width blocks, each labeled “Record 0”, “Record 1”, “Record 2”, with byte offsets marked below]

Reading the Nth record — the formula

To reach record N in a file of fixed-size records:

byte offset = N × sizeof(Record)
struct Record {
    int   id;
    float score;
    char  grade;
    char  pad[3];
};

std::ifstream inFile("records.bin", std::ios::binary);

int n = 4;   // read the 5th record (0-indexed)
inFile.seekg(n * sizeof(Record), std::ios::beg);

Record r;
inFile.read(reinterpret_cast<char*>(&r), sizeof(Record));

std::cout << "id: " << r.id
          << "  score: " << r.score
          << "  grade: " << r.grade << "\n";

Writing the Nth record

The same formula applies to writing. Seek to the offset, then write.

std::fstream file("records.bin",
    std::ios::in | std::ios::out | std::ios::binary);

Record updated = {42, 99.5f, 'A', {0, 0, 0}};

int n = 4;   // overwrite record 4
file.seekp(n * sizeof(Record), std::ios::beg);
file.write(reinterpret_cast<char*>(&updated), sizeof(Record));

file.close();
  • reinterpret_cast<char*> tells write to treat the struct as raw bytes
  • sizeof(Record) is the byte count to write
  • The existing record at that offset is overwritten in place

Summary

Random access jumps the position marker with seekg/seekp instead of reading every record in sequence. Fixed-size records make the byte offset predictable: N × sizeof(Record).

Use tellg/tellp to query the current position — useful for computing file size and for debugging offset arithmetic.

Key Takeaways

  • Files persist data beyond the program’s lifetime — use ofstream to write, ifstream to read
  • Always check is_open() before reading or writing
  • Sequential access reads from start to finish; random access jumps directly to any byte offset
  • Fixed-size records make random access predictable: offset = N × sizeof(Record)
