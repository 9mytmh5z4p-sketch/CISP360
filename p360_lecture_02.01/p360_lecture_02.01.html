Displaying Output with cout

Introduction

When you write a program, you need a way to communicate with the user. How does your program display results? How does it show messages? How does it tell the user what’s happening?

That’s where cout comes in!

What is cout?

cout (pronounced “see-out”) stands for “character output” or “console output”

  • It sends data to the screen (the console/terminal)
  • It’s part of the <iostream> library
  • It’s in the std namespace

Think of cout as your program’s voice - it’s how your program “talks” to the user.

Basic Syntax

The basic syntax for cout uses the stream insertion operator <<

cout << "something to display";

Read it as: “send to cout” or “insert into cout”

The << operator is like an arrow pointing toward cout, showing that data flows TO the output stream.

Classic First Program

#include <iostream>
using namespace std;

int main()
{
    cout << "Hello, World!";
    
    return 0;
}

Output:

Hello, World!

What’s happening:

  1. #include <iostream> - Brings in the iostream library (where cout lives)

  2. using namespace std; - Lets us write cout instead of std::cout

  3. cout << "Hello, World!"; - Sends the text to the screen

  4. The semicolon ; ends the statement

Displaying Different Types of Data

  • String Literals
  • Numbers
  • Variables

All can go into the stream.

Displaying Text (String Literals)

Text must be enclosed in double quotes " "

cout << "Welcome to C++ Programming!";

Output:

Welcome to C++ Programming!

Displaying Numbers

Numbers don’t need quotes:

cout << 42;
cout << 3.14159;
cout << -17;

Output:

423.14159-17

Notice: Everything runs together! We’ll fix this soon.

Displaying Variables

You can display the value stored in a variable:

int age = 25;
string name = "Alice";

cout << age;    // Displays: 25
cout << name;   // Displays: Alice

Chaining Multiple Items

You can display multiple things in one statement by chaining them with <<

cout << "Hello" << " " << "World";

Output:

Hello World

Mixing Text and Variables

This is where cout becomes powerful:

string name = "Bob";
int age = 30;

cout << "My name is " << name << " and I am " << age << " years old.";

Output:

My name is Bob and I am 30 years old.

Notice: - Text goes in quotes: "My name is " - Variables don’t use quotes: name - Spaces in quotes: " and I am " (notice the spaces!)

Adding New Lines

By default, cout doesn’t move to the next line. Everything prints on the same line.

Method 1: Using endl

endl stands for “end line” - it moves to the next line:

cout << "First line" << endl;
cout << "Second line" << endl;
cout << "Third line" << endl;

Output:

First line
Second line
Third line

Method 2: Using \n

\n is the newline character:

cout << "First line\n";
cout << "Second line\n";
cout << "Third line\n";

Output:

First line
Second line
Third line

endl vs \n - What’s the Difference?

Both create a new line, but:

  • endl - Moves to next line AND flushes the output buffer (forces immediate display)

  • \n - Just moves to next line (slightly faster)

For beginners: Use whichever you prefer. endl is more common and safer.

Creating Space in Your Output

  • Blank Lines
  • Multiple Newlines
  • Spaces in Text

Blank Lines

cout << "Paragraph 1" << endl;
cout << endl;                   // Blank line
cout << "Paragraph 2" << endl;

Output:

Paragraph 1

Paragraph 2

Multiple Newlines

cout << "Top" << endl << endl << endl;
cout << "Bottom" << endl;

Output:

Top


Bottom

Spaces in Text

cout << "Hello    World";  // Multiple spaces in quotes

Output:

Hello    World

Hint: I use . instead spaces to make it easier to count.

cout << "Hello....World";  // Multiple spaces in quotes

Output:

Hello....World

When you get everything lined up you type over the . with a space.

Complete Example Program

Let’s put it all together:

// display_info.cpp - Demonstrating cout
#include <iostream>
using namespace std;

int main()
{
    // Variables
    string firstName = "Sarah";
    string lastName = "Johnson";
    int age = 22;
    double gpa = 3.85;
    
    // Display information
    cout << "=== Student Information ===" << endl;
    cout << endl;
    
    cout << "Name: " << firstName << " " << lastName << endl;
    cout << "Age: " << age << " years old" << endl;
    cout << "GPA: " << gpa << endl;
    cout << endl;
    
    cout << "Thank you for using this program!" << endl;
    
    return 0;
}

Output:

=== Student Information ===

Name: Sarah Johnson
Age: 22 years old
GPA: 3.85

Thank you for using this program!

Special Characters and Escape Sequences

Sometimes you need to display special characters. Use the backslash \ (called an escape character):

Escape Sequence Meaning Example
\n Newline (new line) "Line 1\nLine 2"
\t Tab (indent) "Name:\tValue"
\" Double quote "She said \"Hi\""
\\ Backslash "C:\\folder\\file"

Examples:

cout << "Column1\tColumn2\tColumn3" << endl;
cout << "Value1\tValue2\tValue3" << endl;

Output:

Column1    Column2    Column3
Value1     Value2     Value3
cout << "He said, \"Hello!\"" << endl;

Output:

He said, "Hello!"
cout << "First line\nSecond line\nThird line" << endl;

Output:

First line
Second line
Third line

Displaying Calculations

cout can display the result of calculations:

int a = 10;
int b = 20;

cout << "The sum is: " << a + b << endl;
cout << "The product is: " << a * b << endl;
cout << "The average is: " << (a + b) / 2 << endl;

Output:

The sum is: 30
The product is: 200
The average is: 15

Important: The calculation happens first, then the result is displayed.

Common Patterns

Pattern 1: Header and Content

cout << "=== Title ===" << endl;
cout << "Content here" << endl;
cout << "=============" << endl;

Output:

=== Title ===
Content here
=============

Pattern 2: Labels and Values

string item = "Laptop";
double price = 899.99;

cout << "Item:  " << item << endl;
cout << "Price: $" << price << endl;

Output:

Item:  Laptop
Price: $899.99

Pattern 3: Messages with Data

int score = 95;
cout << "Your score of " << score << " is excellent!" << endl;

Output:

Your score of 95 is excellent!

Pattern 4: Building Sentences

string subject = "C++";
int hours = 3;

cout << "I studied " << subject;
cout << " for " << hours << " hours";
cout << " today." << endl;

Output:

I studied C++ for 3 hours today.

Common Mistakes and How to Avoid Them

Mistake 1: Forgetting Quotes Around Text

WRONG:

cout << Hello;  // Error! C++ thinks Hello is a variable

RIGHT:

cout << "Hello";  // Correct! Text needs quotes

Mistake 2: Using Single Quotes for Text

WRONG:

cout << 'Hello';  // Error! Single quotes are for single characters only

RIGHT:

cout << "Hello";  // Use double quotes for text
cout << 'H';      // Single quotes for ONE character

Mistake 3: Forgetting #include <iostream>

WRONG:

// Missing #include <iostream>

int main()
{
    cout << "Hello";  // Error! cout doesn't exist
}

RIGHT:

#include <iostream>
using namespace std;

int main()
{
    cout << "Hello";  // Works!
}

Mistake 4: Missing Semicolon

WRONG:

cout << "Hello"  // Error! Missing semicolon
cout << "World";

RIGHT:

cout << "Hello";  // Each statement needs semicolon
cout << "World";

Mistake 5: Confusing Variables and Text

WRONG:

int age = 25;
cout << "age";  // Displays: age (the word, not the value!)

RIGHT:

int age = 25;
cout << age;   // Displays: 25 (the value)
cout << "age"; // Displays: age (the word)

Remember: - Quotes = literal text - No quotes = variable or number

Good Practices

1. Make Output Clear and Readable

Poor:

cout << name << age << city;

Output: Alice25Boston (confusing!)

Better:

cout << "Name: " << name << endl;
cout << "Age: " << age << endl;
cout << "City: " << city << endl;

Output:

Name: Alice
Age: 25
City: Boston

2. Use Blank Lines to Organize

cout << "=== Section 1 ===" << endl;
cout << "Content..." << endl;
cout << endl;  // Blank line

cout << "=== Section 2 ===" << endl;
cout << "More content..." << endl;

Output

=== Section 1 ===
Content...

=== Section 2 ===
More content...

3. Label Your Output

Poor:

cout << 42 << endl;
cout << 3.14 << endl;

Better:

cout << "Answer: " << 42 << endl;
cout << "Pi: " << 3.14 << endl;

4. Use Consistent Formatting

Pick a style and stick with it:

// Style 1: Everything on separate lines
cout << "Hello" << endl;
cout << "World" << endl;

// Style 2: Chain when related
cout << "Hello " << "World" << endl;

// Both are fine - just be consistent!

Getting User Input with cin

Introduction

So far, we’ve been using cout to output information to the screen. But what if we want our programs to be interactive? What if we want to ask the user questions and get their answers?

That’s where cin comes in!

What is cin?

cin (pronounced “see-in”) stands for “character input” or “console input”

  • It reads data from the keyboard
  • It’s part of the <iostream> library (same as cout)
  • It’s in the std namespace

Think of it this way:

  • cout = output (computer talks to user) -> screen
  • cin = input (user talks to computer) <- keyboard

Basic Syntax

The basic syntax for cin uses the stream extraction operator >>

cin >> variableName;
  • cout uses << (insertion operator - puts data OUT)
  • cin uses >> (extraction operator - pulls data IN)

Think of the arrows:

  • cout << - arrows point toward cout (data flows TO output)
  • cin >> - arrows point away from cin (data flows FROM input)

Simple Example

Let’s write a program that asks for the user’s age:

#include <iostream>
using namespace std;

int main()
{
    int age;  // Declare a variable to store the age
    
    cout << "How old are you? ";  // Ask the question
    cin >> age;                    // Wait for user input
    
    cout << "You are " << age << " years old." << endl;
    
    return 0;
}

What happens when this runs:

  1. Program displays: “How old are you?”

  2. Program waits for the user to type a number and press Enter

  3. User types 25 and presses Enter

  4. The value 25 is stored in the variable age

  5. Program displays: “You are 25 years old.”

Key Concepts

1. The Variable Must Exist First

WRONG:

cin >> age;  // Error! What is 'age'?

RIGHT:

int age;     // Declare the variable first
cin >> age;  // Now we can read into it

2. The Variable Type Matters

cin automatically converts the input to match your variable type:

int number;
cin >> number;     // Expects an integer (whole number)

double price;
cin >> price;      // Expects a decimal number

string name;
cin >> name;       // Expects text

We will have entire lectures about what happens if it doesn’t match.

3. The Program Waits

When your program reaches cin, it stops and waits for the user to:

  1. Type something

  2. Press Enter

This is called blocking - the program is blocked until input is received.

Reading Different Data Types

Reading Integers

int numberOfPets;

cout << "How many pets do you have? ";
cin >> numberOfPets;

cout << "You have " << numberOfPets << " pets." << endl;

Reading Decimals (Floating-Point)

double temperature;

cout << "What is the temperature? ";
cin >> temperature;

cout << "It is " << temperature << " degrees." << endl;

Reading Single Words (Strings)

string firstName;

cout << "What is your first name? ";
cin >> firstName;

cout << "Hello, " << firstName << "!" << endl;

Important: cin >> only reads one word at a time. It stops at whitespace (spaces, tabs, Enter).

Reading Multiple Values

You can read multiple values in one statement:

int month, day, year;

cout << "Enter your birth month, day, and year: ";
cin >> month >> day >> year;

cout << "You were born on " << month << "/" << day << "/" << year << endl;

User types: 3 15 2005 (separated by spaces) and presses Enter

Output: You were born on 3/15/2005

Reading Multiple Values Separately

You can also prompt for each value individually:

string firstName, lastName;
int age;

cout << "Enter your first name: ";
cin >> firstName;

cout << "Enter your last name: ";
cin >> lastName;

cout << "Enter your age: ";
cin >> age;

cout << "Hello, " << firstName << " " << lastName << "!" << endl;
cout << "You are " << age << " years old." << endl;

The Whitespace Problem

Important limitation: cin >> stops reading at the first whitespace (space, tab, or newline).

Example of the Problem:

string fullName;

cout << "What is your full name? ";
cin >> fullName;  // User types: John Smith

cout << "Your name is " << fullName << endl;

Output: Your name is John

What happened? cin >> only captured “John” and stopped at the space!

Solution: getline()

For reading entire lines (including spaces), use getline():

string fullName;

cout << "What is your full name? ";
getline(cin, fullName);  // Reads the entire line

cout << "Your name is " << fullName << endl;

Output: Your name is John Smith

Note: We’ll learn more about getline() in detail later.

Here’s a complete program that demonstrates cin:

Common Mistakes and How to Avoid Them

Mistake 1: Forgetting to Declare the Variable

WRONG:

cout << "Enter your age: ";
cin >> age;  // Error! 'age' doesn't exist

RIGHT:

int age;     // Declare first
cout << "Enter your age: ";
cin >> age;  // Now it works

Mistake 2: Type Mismatch

Problem:

int number;
cout << "Enter a number: ";
cin >> number;  // User types: "hello"

If the user enters something that doesn’t match the type (like text when expecting a number), cin will fail and your program may behave unexpectedly.

We’ll learn how to handle this problem later in the course.

Mistake 3: Using >> for Multi-word Input

WRONG for full names:

string fullName;
cin >> fullName;  // Only gets first word

RIGHT for full names:

string fullName;
getline(cin, fullName);  // Gets entire line

Mistake 4: No Prompt Message

Confusing for users:

int age;
cin >> age;  // User sees cursor blinking - what do they enter?

Better - always prompt:

int age;
cout << "Please enter your age: ";  // Clear prompt
cin >> age;

Consider using this prompt pattern whenever you have to use cin.

Good Practices

1. Always Prompt Before Input

Bad:

cin >> value;  // User doesn't know what to enter

Good:

cout << "Enter the temperature in Fahrenheit: ";
cin >> value;

2. Use Descriptive Prompts

Vague:

cout << "Enter number: ";

Clear:

cout << "Enter the number of students in your class: ";

3. Echo the Input Back

Confirm what the user entered:

int quantity;
cout << "How many items? ";
cin >> quantity;
cout << "You entered: " << quantity << endl;  // Echo back

4. Use Meaningful Variable Names

Bad:

int x;
cin >> x;

Good:

int numberOfStudents;
cin >> numberOfStudents;

Formatting Output with Manipulators

Introduction

You’ve learned how to display output with cout. But what if you want to:

  • Display money with exactly 2 decimal places?
  • Line up columns of numbers neatly?
  • Show true and false instead of 1 and 0?
  • Control how your output looks?

That’s where manipulators come in!

What Are Manipulators?

Manipulators are special functions that modify how cout displays data.

  • They change the formatting of output
  • They’re used with the << operator
  • Most require the <iomanip> header
  • Some are in <iostream> (like endl)

Think of manipulators as formatting instructions - they tell cout HOW to display something, not WHAT to display.

Two Important Headers

  • Basic Manipulators
  • Advanced Manipulators

<iostream> - Basic Manipulators

#include <iostream>

Contains:

  • endl (end line)
  • boolalpha / noboolalpha
  • fixed / scientific
  • showpoint / noshowpoint

<iomanip> - Advanced Manipulators

#include <iomanip>

Contains:

  • setprecision(n) (set decimal precision)
  • setw(n) (set field width)
  • setfill(c) (set fill character)
  • left / right (alignment)

Important: Always include <iomanip> when using advanced formatting!

Basic Program Setup

#include <iostream>
#include <iomanip>  // Don't forget this!
using namespace std;

int main()
{
    // Your code here
    
    return 0;
}

Precision Manipulators

  • setprecision
  • setprecision and fixed
  • fixed

setprecision(n) - Control Decimal Places

Controls how many digits are displayed.

Without fixed: Controls total significant digits

double pi = 3.14159265;

cout << setprecision(3) << pi << endl;  // 3.14
cout << setprecision(5) << pi << endl;  // 3.1416
cout << setprecision(8) << pi << endl;  // 3.1415927

With fixed: Controls decimal places (digits after the decimal point)

double price = 19.5;

cout << fixed << setprecision(2) << price << endl;  // 19.50

The fixed Manipulator

fixed tells C++ to use fixed-point notation (regular decimal format, not scientific notation).

Without fixed:

double num = 123.456789;
cout << setprecision(2) << num << endl;  // 1.2e+02 (scientific notation)

With fixed:

double num = 123.456789;
cout << fixed << setprecision(2) << num << endl;  // 123.46

Common Pattern: Money Formatting

Always use fixed with setprecision(2) for money!

double price = 29.5;
double tax = 2.3625;
double total = price + tax;

cout << fixed << setprecision(2);  // Set once for all following output

cout << "Price: $" << price << endl;    // Price: $29.50
cout << "Tax:   $" << tax << endl;      // Tax:   $2.36
cout << "Total: $" << total << endl;    // Total: $31.86

The boolalpha Manipulator

By default, C++ displays boolean values as numbers:

  • true displays as 1
  • false displays as 0

boolalpha makes booleans display as words!

Without boolalpha

bool isStudent = true;
bool hasLicense = false;

cout << "Student: " << isStudent << endl;      // Student: 1
cout << "License: " << hasLicense << endl;     // License: 0

With boolalpha

bool isStudent = true;
bool hasLicense = false;

cout << boolalpha;  // Turn on word display

cout << "Student: " << isStudent << endl;      // Student: true
cout << "License: " << hasLicense << endl;     // License: false

Turning It Off: noboolalpha

bool value = true;

cout << boolalpha << value << endl;      // true
cout << noboolalpha << value << endl;    // 1

Tip: Use boolalpha when displaying boolean values to users - it’s much clearer!

Width Manipulators

  • setw

setw(n) - Set Field Width

setw(n) sets the minimum width for the next output item only.

int num = 42;

cout << setw(5) << num << endl;    //    42 (5 characters wide, right-aligned)
cout << setw(10) << num << endl;   //        42 (10 characters wide)
cout << num << endl;               // 42 (back to normal)

Important: setw() only affects the NEXT item, then resets!

Creating Columns

setw() is perfect for aligning columns:

cout << setw(10) << "Name" << setw(8) << "Age" << setw(10) << "Grade" << endl;
cout << setw(10) << "Alice" << setw(8) << 20 << setw(10) << 95 << endl;
cout << setw(10) << "Bob" << setw(8) << 22 << setw(10) << 88 << endl;
cout << setw(10) << "Charlie" << setw(8) << 19 << setw(10) << 92 << endl;

Output:

      Name     Age     Grade
     Alice      20        95
       Bob      22        88
   Charlie      19        92

Alignment Manipulators

left and right

By default, output is right-aligned within the field width.

Right-aligned (default):

cout << setw(10) << "Hello" << endl;    //      Hello
cout << setw(10) << 42 << endl;         //         42

Left-aligned:

cout << left;  // Switch to left alignment

cout << setw(10) << "Hello" << endl;    // Hello     
cout << setw(10) << 42 << endl;         // 42

Switch back:

cout << right;  // Back to right alignment

Practical Example: Menu Display

cout << left;
cout << setw(20) << "Item" << setw(10) << "Price" << endl;
cout << setw(20) << "Coffee" << right << setw(10) << "$3.50" << endl;
cout << left << setw(20) << "Sandwich" << right << setw(10) << "$7.99" << endl;
cout << left << setw(20) << "Salad" << right << setw(10) << "$6.50" << endl;

Output:

Item                     Price
Coffee                   $3.50
Sandwich                 $7.99
Salad                    $6.50

Fill Character: setfill(c)

setfill(c) sets the character used to fill empty space in a field.

Default fill character is space:

cout << setw(10) << 42 << endl;    //         42 (filled with spaces)

Custom fill character:

cout << setfill('0');
cout << setw(5) << 42 << endl;     // 00042 (filled with zeros)

cout << setfill('*');
cout << setw(10) << "Hi" << endl;  // ********Hi (filled with asterisks)

Practical Uses

1. Zero-padded numbers (like invoice numbers):

int invoiceNum = 237;
cout << "Invoice #" << setfill('0') << setw(6) << invoiceNum << endl;
// Output: Invoice #000237

2. Decorative lines:

cout << setfill('-') << setw(40) << "" << endl;
// Output: ----------------------------------------

3. Credit card display:

cout << setfill('*') << setw(12) << "4532" << endl;
// Output: ********4532

Persistent vs. Temporary Manipulators

Persistent (Stay Until Changed)

These stay active until you explicitly change them:

  • fixed / scientific
  • setprecision(n)
  • boolalpha / noboolalpha
  • left / right
  • setfill(c)
  • showpoint / noshowpoint

Persistence Example

Common Mistakes and Solutions

Mistake 1: Forgetting #include <iomanip>

WRONG:

#include <iostream>  // Missing iomanip!
using namespace std;

int main()
{
    cout << setprecision(2) << 3.14159;  // Error! setprecision not defined
}

RIGHT:

#include <iostream>
#include <iomanip>  // Don't forget this!
using namespace std;

int main()
{
    cout << setprecision(2) << 3.14159;  // Works!
}

Mistake 2: Using setprecision() Without fixed

Problem:

double price = 19.5;
cout << setprecision(2) << price;  // Might show: 20 or 19 (not 19.50!)

Solution:

double price = 19.5;
cout << fixed << setprecision(2) << price;  // Shows: 19.50

Mistake 3: Forgetting setw() Resets

WRONG:

cout << setw(10);  // Set once
cout << 42 << endl;    //       42
cout << 17 << endl;    // 17 (NOT 10 wide! setw expired)

RIGHT:

cout << setw(10) << 42 << endl;    //       42
cout << setw(10) << 17 << endl;    //       17

Mistake 4: Wrong Order of Manipulators

WRONG:

cout << 3.14159 << fixed << setprecision(2);  // Too late! 3.14159 already displayed

RIGHT:

cout << fixed << setprecision(2) << 3.14159;  // Set format BEFORE displaying

Summary

Creating Formatted Tables in C++

~ 65

Introduction

You’ve learned how to format individual values with manipulators. Now let’s combine those skills to create something professional: formatted tables!

Tables are everywhere in programming:

  • Financial reports
  • Data summaries
  • System information displays
  • Log files
  • Database outputs

Key Question: How do we decide which table format to use?

Answer: The data inside the table drives the decision about which table to use.

Why Tables Matter

Well-formatted tables make data:

  • Readable - Easy to scan and understand
  • Professional - Look polished and organized
  • Comparable - Align values for easy comparison
  • Clear - Show relationships between data

Compare these two outputs:

Without formatting (bad):

Alice 30 50000.5
Bob 25 45000.8
Charlie 28 52000

With table formatting (good):

================================================
| Name         | Age        | Salary           |
================================================
| Alice        | 30         | 50000.5          |
| Bob          | 25         | 45000.8          |
| Charlie      | 28         | 52000            |
================================================

Isn’t this easier to read?

The Four Essential Table Types

We’ll cover four common table patterns, each suited for different types of data:

  1. Fixed Width Table with Borders - Professional reports with clear separation
  2. Hexadecimal and Octal Table - Number system conversions
  3. Cash Table with Negative Numbers - Financial data with parentheses for negatives
  4. Scientific Notation Table - Very large or very small numbers

Table Type 1: Fixed Width Table with Borders

When to Use

Use this table when you need:

  • Clear visual separation between columns
  • Professional appearance
  • Mixed data types (strings and numbers)
  • General-purpose data display

Example Output

Fixed Width Table with Borders
================================================
| Name         | Age        | Salary           |
================================================
| Alice        | 30         | 50000.5          |
| Bob          | 25         | 45000.8          |
| Charlie      | 28         | 52000            |
================================================
End of Report.

Key Techniques

  1. Define column widths as constants
  2. Use setfill() to create border lines
  3. Use setw() for each column
  4. Add spaces and | characters for visual borders

Complete Code Example

//FixWwBorders.cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    const int nameWidth = 15;
    const int ageWidth = 10;
    const int salaryWidth = 12;
    const int totalWidth = nameWidth + ageWidth + salaryWidth + 8;
    const string title = "Fixed Width Table with Borders";
    
    cout << title << endl;
    cout << setfill('=') << setw(totalWidth) << "=" << endl;
    cout << setfill(' ')
         << "| " << left << setw(nameWidth) << "Name"
         << "| " << setw(ageWidth) << "Age"
         << "| " << setw(salaryWidth) << "Salary" << " |" << endl;
    cout << setfill('=') << setw(totalWidth) << "=" << endl;
    
    cout << setfill(' ')
         << "| " << setw(nameWidth) << "Alice"
         << "| " << setw(ageWidth) << 30
         << "| " << setw(salaryWidth) << 50000.50 << " |" << endl;
    
    cout << "| " << setw(nameWidth) << "Bob"
         << "| " << setw(ageWidth) << 25
         << "| " << setw(salaryWidth) << 45000.75 << " |" << endl;
    
    cout << "| " << setw(nameWidth) << "Charlie"
         << "| " << setw(ageWidth) << 28
         << "| " << setw(salaryWidth) << 52000.00 << " |" << endl;
    
    cout << setfill('=') << setw(totalWidth) << "=" << endl;
    
    cout << "End of Report." << endl;
    
    return 0;
}

Important Concepts

1. Calculate total width:

const int totalWidth = nameWidth + ageWidth + salaryWidth + 8;
// +8 accounts for the "| | | |" characters and spaces

2. Create border lines with setfill:

cout << setfill('=') << setw(totalWidth) << "=" << endl;
// Fills the width with '=' characters

Important Concepts

3. Reset fill character:

cout << setfill(' ')  // Always reset to space after using setfill('=')

4. Each column needs setw():

cout << "| " << setw(nameWidth) << "Alice"  // setw() only affects next item!
     << "| " << setw(ageWidth) << 30         // Need setw() again

Table Type 3: Cash Table with Negative Numbers

When to Use

Use this table when you need:

  • Financial data (income/expenses)
  • Accounting reports
  • Budget tracking
  • Any data with positive and negative monetary values

Special Feature: Parentheses for Negative Numbers

In accounting, negative numbers are shown in parentheses:

  • Positive: 1500.75
  • Negative: ( $450.50) (not -450.50)

Example Output

========================================
| Description       | Cash Amount      |
----------------------------------------
| Salary            |        1500.75   |
| Groceries         |     ( $450.50)   |
| Freelance Work    |        2000.00   |
| Rent              |     ( $300.00)   |
| Investment        |         750.25   |
========================================

Key Techniques

  1. Use fixed and setprecision(2) for money
  2. Handle negative numbers with conditional formatting
  3. Use right alignment for numbers
  4. Adjust setw() for parentheses (subtract space for $ and ())

Complete Code Example

//CashTablewNegs.cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    const int colWidth = 15;
    const int totalWidth = colWidth * 2 + 6;
    
    double salary = 1500.75;
    double groceries = -450.50;
    double freelance = 2000.00;
    double rent = -300.00;
    double investment = 750.25;
    
    cout << setfill('=') << setw(totalWidth) << "=" << endl;
    cout << setfill(' ')
         << "| " << left << setw(colWidth) << "Description"
         << "| " << setw(colWidth) << "Cash Amount" << " |" << endl;
    cout << setfill('-') << setw(totalWidth) << "-" << endl;
    
    cout << setfill(' ');
    cout << fixed << setprecision(2);
    
    cout << "| " << setw(colWidth) << left << "Salary";
    cout << "| " << setw(colWidth) << right << salary << " |" << endl;
    
    cout << "| " << setw(colWidth) << left << "Groceries";
    cout << "|" << setw(colWidth - 6) << right << "( $" << -groceries << ") |" << endl;
    
    cout << "| " << setw(colWidth) << left << "Freelance Work";
    cout << "| " << setw(colWidth) << right << freelance << " |" << endl;
    
    cout << "| " << setw(colWidth) << left << "Rent";
    cout << "|" << setw(colWidth - 6) << right << "( $" << -rent << ") |" << endl;
    
    cout << "| " << setw(colWidth) << left << "Investment";
    cout << "| " << setw(colWidth) << right << investment << " |" << endl;
    
    cout << setfill('=') << setw(totalWidth) << "=" << endl;
    
    return 0;
}

Negative Number Formatting Explained

For negative numbers in accounting format:

Regular output:

cout << -450.50;  // Displays: -450.5

Accounting format with parentheses:

// Store as negative, display with parentheses
double groceries = -450.50;
cout << "( $" << -groceries << ")";  // Displays: ( $450.50)

Why setw(colWidth - 6)? - colWidth is 15 - ( $ takes 3 characters - ) takes 1 character
- We need space for the number: 15 - 6 = 9 characters for the amount

When to Use

Use this table when you need:

  • Very large numbers (millions, billions, trillions)

  • Precise notation regardless of scale

Key Techniques

  1. Use scientific manipulator
  2. Use setprecision() to control significant figures
  3. Simpler table structure (fewer columns)

Complete Code Example

// SciNotWLeft.cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    double num1 = 1234.56789;
    double num2 = 9876.54321;
    string title = "Scientific Notation Output:";
    
    cout << title << endl;
    cout << string(title.size(), '-') << endl;  // auto figure length
    cout << left << setw(15) << "Value 1:"
         << scientific << setprecision(3) << num1 << endl;
    cout << setw(15) << "Value 2:"
         << scientific << setprecision(5) << num2 << endl;
    
    return 0;
}

Scientific Notation Explained

Scientific notation format: 1.234e+03 - Coefficient: 1.234 - Exponent: +03 (means × 10³) - Value: 1.234 × 1000 = 1234

Controlling precision:

double num = 1234.56789;

cout << scientific << setprecision(3) << num;  // 1.235e+03
cout << scientific << setprecision(5) << num;  // 1.23457e+03

Uppercase E:

cout << scientific << uppercase << num;  // 1.235E+03 (capital E)
