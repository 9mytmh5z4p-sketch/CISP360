Abstract Data Types

When does one variable stop being enough?

  • You can store an employee’s ID in an int — but their name and pay rate need separate variables with no formal relationship.
  • An Abstract Data Type (ADT) defines a new type that groups related data together under one name.

Total slides: 3 This is a short motivating section — 5 minutes max. The goal is to create a felt need before showing struct syntax. Students have seen int, string, double, and arrays. The question is: how do you represent a real-world entity that has multiple attributes of different types? ADT is the answer. The next slide places structs in context alongside familiar types.

Where does struct fit?

Type Example Groups
Scalar variable int x; one value
Array int scores[5]; values of the same type
Struct Employee e; values of different types
Class string s; data + behavior

The “Table of Variables” gives students a mental map. Structs are the natural next step after arrays — same grouping idea, but with heterogeneous types. Don’t dwell here; the next topic builds structs from the ground up. The key line to point at: struct groups values of different types.

Summary

  • An ADT is any user-defined type — you decide what data it holds.
  • In C++, struct is the simplest way to create one.

Structures

What holds an employee together?

  • Three variables describe one employee: ID, name, and pay rate.
  • With 15 employees you’d need 45 separate variables with no formal relationship between them.
  • A struct solves this by creating one type that holds all three.

Total slides: 6 Problem-first framing from the original slides. Don’t skip the 45-variable problem — it makes the motivation concrete. Students have done parallel arrays (15.01/15.02); they know the pain. Struct is the fix.

Struct anatomy

struct Employee {
    int    employeeID;
    string name;
    double rate;
};
  • struct — keyword; creates a new type
  • Employee — the type name; use it like int or string
  • employeeID, name, rate — members; each has its own type
  • }; — closing brace must have a semicolon

This is the anatomy visualization. Walk through each labeled part. The semicolon after the closing brace is the most common syntax error students make — point at it explicitly. The struct declaration is a blueprint; it allocates no memory on its own. [Graphic suggestion: in-class pointer diagram showing struct declaration as a type blueprint (no box drawn yet), then a variable declaration drawing the actual memory box with three labeled slots.]

Declaring a struct variable

#include <iostream>
#include <string>
using namespace std;

struct Employee {
    int    employeeID;
    string name;
    double rate;
};

int main() {
    Employee e;          // e is a variable of type Employee
    return 0;
}

Once the struct type exists, declaring a variable is identical to declaring an int. The type name IS Employee. This slide exists to separate “defining the type” (previous slide) from “creating a variable of that type” (this slide).

Accessing members with dot notation

int main() {
    Employee e;

    e.employeeID = 14335;      // variable.member = value
    e.name       = "Pat Smith";
    e.rate       = 17.55;

    cout << e.employeeID << "\n";   // 14335
    cout << e.name       << "\n";   // Pat Smith
    cout << e.rate       << "\n";   // 17.55

    return 0;
}

The dot operator is the only new syntax here. Read it as “e’s employeeID.” The members behave like ordinary variables — you assign them, read them, pass them. The variable name comes first, then the dot, then the member name.

Two ways to initialize

// Initialization list — values in declaration order
Employee a = {14335, "Pat Smith", 17.55};

// Member-by-member — order doesn't matter
Employee b;
b.employeeID = 29401;
b.name       = "Jordan Lee";
b.rate       = 19.75;

Both forms are valid. Initialization list is concise but requires knowing the declaration order. Member-by-member is verbose but explicit. Students will mix these freely — that’s fine. The important constraint: initialization list values must appear in the same order as the struct’s member declarations.

Summary

  • Declare the struct type once; create as many variables as needed.
  • Use variable.member to read or write any individual field.

What’s the Output?

What does this program print?

  • Trace through carefully — one path uses an initialization list, the other uses member assignment.
  • Pay attention to how the struct is passed into addBonus.

Total slides: 3 Pair work recommended. Give students 3–4 minutes. Key traps: (1) addBonus takes by reference — it modifies a.bonus; (2) b.bonus is already set to 5 and addBonus is never called on b; (3) 80/10 = 8 (integer division).

Exercise

#include <iostream>
#include <string>
using namespace std;

struct Score {
    string player;
    int    points;
    int    bonus;
};

int addBonus(Score& s) {
    s.bonus = s.points / 10;
    return s.points + s.bonus;
}

int main() {
    Score a = {"Alice", 80, 0};
    Score b;
    b.player = "Bob";
    b.points = 95;
    b.bonus  = 5;

    int aTotal = addBonus(a);

    cout << a.player << ": " << aTotal     << "\n";
    cout << b.player << ": " << b.points + b.bonus << "\n";

    return 0;
}

Answer: Alice: 88 Bob: 100

Trace: - a is initialized with initialization list: player=“Alice”, points=80, bonus=0 - b is initialized member-by-member: player=“Bob”, points=95, bonus=5 - addBonus(a): pass by reference; a.bonus = 80/10 = 8; returns 80+8 = 88 - aTotal = 88 - addBonus is NOT called on b - b.points + b.bonus = 95 + 5 = 100

Common mistake: students may try to compute addBonus for b too, or miss that bonus is modified by reference.

Answer

cout << a.player << ": " << aTotal     << "\n";  // Alice: 88
cout << b.player << ": " << b.points + b.bonus << "\n";  // Bob: 100
  • addBonus takes Score& — it modifies a.bonus to 8, then returns 88
  • addBonus is never called on b — b.bonus stays at 5
  • b.points + b.bonus = 95 + 5 = 100

Arrays of Structures

An array where every element is a struct

  • An array of structs uses the same bracket syntax as any other array — the element type is just your struct.
  • Each element in the array gives you access to all of that struct’s members via dot notation.

Total slides: 4 Students know arrays. Students know structs. This topic combines them. The new syntax is minimal: Student roster[5] instead of int scores[5]. The tricky part is the two-step access: roster[i].name. Walk through the indexing + dot notation carefully.

Declaring and accessing

struct Student {
    string name;
    int    grade;
};

int main() {
    Student roster[3];           // array of 3 Student structs

    roster[0].name  = "Alice";   // [index].member
    roster[0].grade = 92;

    roster[1] = {"Bob", 85};     // initialization list on array element

    cout << roster[0].name << ": " << roster[0].grade << "\n";
    cout << roster[1].name << ": " << roster[1].grade << "\n";

    return 0;
}

Point out: roster[0] is a Student; roster[0].name is a string. The subscript operator selects the element; the dot operator selects the member. Both initialization forms (member-by-member and initialization list) work on array elements.

Looping through an array of structs

const int SIZE = 4;
Student   roster[SIZE] = {
    {"Alice", 92},
    {"Bob",   85},
    {"Carol", 97},
    {"Dan",   78}
};

int best = 0;
for (int i = 1; i < SIZE; i++)
    if (roster[i].grade > roster[best].grade) best = i;

cout << "Top student: " << roster[best].name
     << " with "        << roster[best].grade << "\n";

The max-tracker pattern is the same as with a plain int array — the only difference is roster[i].grade instead of scores[i]. This prepares students for the programming task, which asks them to build this from scratch.

Summary

  • StructType array[n] — same syntax as a primitive array, struct is the element type.
  • Access: array[i].member — subscript first, then dot.

Programming Task — Banking System

Build it: a simple banking system

You are building a banking system. The core data structure is a struct with three members: a string name, an int accountNumber, and a float balance. Present the customer with a menu that loops until they quit:

  • Add funds — deposit an amount; validate that the amount is positive
  • Withdraw funds — reduce the balance; validate sufficient funds and a positive amount
  • Transfer funds — reduce the balance and send money to another bank; collect the recipient bank name, account number, and transfer amount, then encode the recipient account number and transfer amount using SHA-256
  • Quit — exit the loop

After every transaction, display the balance before and after.

Total slides: 2 This is the capstone task for the structs unit. Students must combine struct declaration, a menu loop, input validation, and conditional logic. The SHA-256 encoding of the outbound transfer data is new territory — it introduces the idea that sensitive data should not travel in plaintext. Students do not need to implement SHA-256 from scratch; they can use OpenSSL or a header-only library. Flag this as a stretch goal if time is short.

Think before you code

  1. What does this program do — describe it in one sentence.
  2. What states or moments can the program be in?

Key Takeaways

  • A struct groups related members of different types under a single user-defined type name
  • Declare the struct type once; create as many variables as needed — just like int or string
  • Access members with dot notation: variable.member
  • Arrays of structs use the same indexing syntax as arrays of primitives: roster[i].name

Appendix

Abstract Data Types: Good Practices

  • Use a struct any time you manage multiple variables that describe the same entity — parallel variables with no formal relationship are a signal to create an ADT.
  • Name struct types with a capital letter and a noun (Employee, Student, Product) — the type name describes what the struct represents.
  • Keep struct declarations outside of functions so all functions in the file can use the type.

Abstract Data Types: Common Mistakes

  • Forgetting the semicolon after the closing brace: struct Employee { ... } — the compiler reports an error on the next line, not at the struct itself.
  • Treating the struct declaration as a variable — struct Employee { ... }; creates a type, not a storage location; you must separately declare Employee e;.
  • Reaching for an array when fields have different types — that’s a struct, not an array.

Abstract Data Types: More Examples

// A Point ADT for 2D geometry
struct Point {
    double x;
    double y;
};

// A Date ADT
struct Date {
    int month;
    int day;
    int year;
};

Any real-world entity with multiple attributes of potentially different types is a candidate for an ADT.

Structures: Good Practices

  • Declare the struct type once, outside any function, so it is available throughout the file.
  • Use an initialization list when all values are known at declaration time; use member-by-member assignment when values come from user input or computation.
  • Pass structs by const reference (const Employee& e) to functions that only read the data — avoids copying all members.

Structures: Common Mistakes

  • Omitting the semicolon after } — required after every struct definition; missing it causes confusing errors on the lines that follow.
  • Initializing list values in the wrong order — values must match the order of member declarations in the struct, not alphabetical or logical order.
  • Writing Employee.name instead of e.name — the dot operator belongs to the variable, not the type name.

Structures: More Examples

// Pass by const reference — no copy, members read-only
void printEmployee(const Employee& e) {
    cout << e.employeeID << ": " << e.name
         << " @ $" << e.rate << "\n";
}

// Return a struct from a function
Employee makeEmployee(int id, string name, double rate) {
    Employee temp;
    temp.employeeID = id;
    temp.name       = name;
    temp.rate       = rate;
    return temp;
}

Arrays of Structures: Good Practices

  • Use a named constant for array size: const int SIZE = 5; — change it in one place and everything updates.
  • Use the two-step access pattern consistently: subscript first, then dot — roster[i].name, not roster.name[i].
  • Declare the struct type before the array; the type must be defined before it can be used as an element type.

Arrays of Structures: Common Mistakes

  • Writing roster.name[i] instead of roster[i].name — roster is an array; you must index into it before accessing a member.
  • Initializing array elements with values in the wrong order — initialization list values map to struct members in declaration order.
  • Treating the array variable as a single struct — roster holds multiple structs; you must index to get one.

Arrays of Structures: More Examples

// Finding the minimum
int worst = 0;
for (int i = 1; i < SIZE; i++)
    if (roster[i].grade < roster[worst].grade) worst = i;
cout << "Lowest: " << roster[worst].name
     << " (" << roster[worst].grade << ")\n";

// Passing the array to a function
void printRoster(Student roster[], int size) {
    for (int i = 0; i < size; i++)
        cout << roster[i].name << ": " << roster[i].grade << "\n";
}

Programming Task — Banking System: Good Practices

  • Declare the struct outside main so helper functions for deposit, withdrawal, and transfer can all use it.
  • Validate every input before modifying the balance — check for positive amounts and sufficient funds before any operation changes state.
  • Display the balance before and after every transaction immediately after the operation, not at the end of the loop.

Programming Task — Banking System: Common Mistakes

  • Modifying the balance before validating the input — validate first, then update; a failed validation should leave the balance unchanged.
  • Using float for currency arithmetic without understanding rounding — float has limited precision; double is safer for money calculations.
  • Skipping the cin.ignore() before getline for the bank name — without it, the newline left in the buffer will be consumed as an empty string.

Programming Task — Banking System: More Examples

// Balance before/after pattern

account.balance += amount;
cout << "Before: $" << before << "  After: $" << account.balance << "\n";

// Transfer validation
if (amount <= 0 || amount > account.balance) {
    cout << "Invalid transfer amount.\n";
} else {
    account.balance -= amount;
    // encode and transmit recipient info
}
