Nested Loops

Repeating Repetition

  • Sometimes each pass through a loop contains another full loop.
  • That pattern is called a nested loop.

Slides 3-17

Outer Loop, Inner Loop

Loop Role
Outer loop Chooses the larger context
Inner loop Completes repeated work inside that context

Reading Nested Loops

  • Read the outer loop first to understand the big structure.
  • Then read the inner loop to understand what happens inside each outer pass.
  • During execution, the inner loop finishes before the next outer pass begins.

A Small Trace

Outer Pass Inner Passes
row = 1 col = 1, col = 2, col = 3
row = 2 col = 1, col = 2, col = 3

The inner loop runs three times for each row.

The Inner Loop Starts Over

  • Each new outer-loop pass creates a fresh run of the inner loop.
  • The inner loop’s control variable is initialized again.
  • This is why the column count restarts on each row.

Rows And Columns

Idea Loop Role
Row Outer loop
Column Inner loop
Cell One pass through the inner loop

Printing A Rectangle

#include <iostream>
using namespace std;

int main() {
    for (int row = 1; row <= 3; row++) {
        for (int col = 1; col <= 5; col++) {
            cout << '*';
        }
        cout << endl;
    }

    return 0;
}

Printing A Triangle

#include <iostream>
using namespace std;

int main() {
    for (int row = 1; row <= 5; row++) {
        for (int col = 1; col <= row; col++) {
            cout << '*';
        }
        cout << endl;
    }

    return 0;
}

Counting Total Passes

  • When both loops have fixed counts, multiply them.
  • 3 rows and 5 columns create 15 inner-loop passes.
  • If one loop depends on the other, trace the rows one at a time.

Loop Order Matters

for (int row = 1; row <= 3; row++) {
    for (int col = 1; col <= 5; col++) {
        cout << '*';
    }
    cout << endl;
}
for (int col = 1; col <= 5; col++) {
    for (int row = 1; row <= 3; row++) {
        cout << '*';
    }
    cout << endl;
}

The same two counts can produce a different traversal pattern.

Choosing Loop Variables

  • Use names that describe the structure: row, col, student, score.
  • Do not reuse the same loop variable name in both loops.
  • Clear names make the nesting easier to trace.

Nested Loops With Totals

int grandTotal = 0;

for (int student = 1; student <= 3; student++) {
    int studentTotal = 0;

    for (int test = 1; test <= 4; test++) {
        int score;
        cin >> score;
        studentTotal += score;
    }

    grandTotal += studentTotal;
}

Keeping Nesting Shallow

  • Two levels of nesting are common.
  • Three or more levels can become hard to read quickly.
  • When nesting gets deep, ask whether a function can name part of the work.

Summary

  • The outer loop chooses the context.
  • The inner loop completes the repeated work for that context.
  • The inner loop starts over for each outer-loop pass.

Multi-Dimensional Vectors

A Vector Of Vectors

  • A vector can store other vectors.
  • That lets us represent table-like data with rows and columns.

Slides 18-37

Rows First

Structure Meaning
Outer vector The collection of rows
Inner vector One row of values
Element One value inside a row

Include The Vector Library

#include <iostream>
#include <vector>
using namespace std;

The vector type comes from the vector library.

Declaring A 2D Vector

vector<vector<int>> scores;

Read this as a vector where each element is another vector<int>.

Creating A Sized Grid

int rows = 3;
int cols = 4;

vector<vector<int>> grid(rows, vector<int>(cols));

This creates 3 rows. Each row starts with 4 integer values.

Initializer Lists

vector<vector<int>> grid = {
    {10, 20, 30},
    {40, 50, 60},
    {70, 80, 90}
};

Nested braces show the rows directly.

Accessing One Element

int value = grid[1][2];

The first index selects the row. The second index selects the column inside that row.

Changing One Element

grid[1][2] = 99;

The same row-column indexing can assign a new value.

Rows, Columns, And .size()

int rows = grid.size();
int cols = grid[0].size();

grid.size() counts rows. grid[0].size() counts columns in the first row.

Traversing With Nested Loops

for (int row = 0; row < grid.size(); row++) {
    for (int col = 0; col < grid[row].size(); col++) {
        cout << grid[row][col] << " ";
    }
    cout << endl;
}

The outer loop moves row to row. The inner loop moves across one row.

Displaying A Grid

#include <iomanip>

for (int row = 0; row < grid.size(); row++) {
    for (int col = 0; col < grid[row].size(); col++) {
        cout << setw(5) << grid[row][col];
    }
    cout << endl;
}

setw keeps columns visually aligned.

Filling A Grid From Input

for (int row = 0; row < grid.size(); row++) {
    for (int col = 0; col < grid[row].size(); col++) {
        cout << "Row " << row << ", col " << col << ": ";
        cin >> grid[row][col];
    }
}

Row Totals

for (int row = 0; row < grid.size(); row++) {
    int rowTotal = 0;

    for (int col = 0; col < grid[row].size(); col++) {
        rowTotal += grid[row][col];
    }

    cout << "Row total: " << rowTotal << endl;
}

Column Totals

for (int col = 0; col < grid[0].size(); col++) {
    int colTotal = 0;

    for (int row = 0; row < grid.size(); row++) {
        colTotal += grid[row][col];
    }

    cout << "Column total: " << colTotal << endl;
}

Column totals assume each row has the same number of columns.

Range-Based Traversal

for (vector<int> row : grid) {
    for (int value : row) {
        cout << value << " ";
    }
    cout << endl;
}

Use this when you need each value but do not need row or column numbers.

Jagged Vectors

  • Each row is its own vector.
  • Rows can have different lengths.
  • Use grid[row].size() when the shape may not be rectangular.

Bounds And Shape Assumptions

  • grid[0].size() assumes the grid has at least one row.
  • Column-total code assumes every row has enough columns.
  • Check the shape before relying on a rectangular grid.

When To Use 2D Vectors

  • Gradebooks
  • Game boards
  • Seating charts
  • Tables of row-and-column data

Summary

  • The outer vector selects a row.
  • The inner vector stores the values in that row.
  • Nested loops are the natural way to traverse row-column data.

Key Takeaways

  • Nested loops represent repeated work inside repeated work.
  • Row-column thinking gives nested loops a concrete structure.
  • Multi-dimensional vectors store table-like data as a vector of rows.

Appendix

Nested Loops: Good Practices

  • Name loop variables for their roles.
  • Keep the inner loop focused on one repeated task.
  • Trace one outer pass before tracing the whole structure.

Nested Loops: Common Mistakes

  • Forgetting that the inner loop starts over for each outer pass.
  • Reusing the same loop variable name in both loops.
  • Assuming total passes without checking whether the inner loop count changes.

Nested Loops: More Examples

for (int student = 1; student <= 3; student++) {
    for (int score = 1; score <= 4; score++) {
        cout << "Student " << student << ", score " << score << endl;
    }
}

Multi-Dimensional Vectors: Good Practices

  • Treat the outer vector as rows.
  • Use grid[row].size() when rows may have different lengths.
  • Check shape assumptions before using grid[0].size().

Multi-Dimensional Vectors: Common Mistakes

  • Forgetting #include <vector>.
  • Reversing row and column indexes.
  • Assuming every row has the same number of columns.

Multi-Dimensional Vectors: More Examples

vector<vector<int>> seats = {
    {1, 1, 0},
    {0, 1, 1}
};

for (int row = 0; row < seats.size(); row++) {
    for (int col = 0; col < seats[row].size(); col++) {
        cout << seats[row][col] << " ";
    }
    cout << endl;
}
