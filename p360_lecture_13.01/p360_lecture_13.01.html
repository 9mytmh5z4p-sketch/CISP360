Multidimensional Arrays

Remember Vector Grids?

  • Rows held vectors
  • Columns held values inside each row

Slides 3–23

A Familiar Shape

vector<vector<int>> vectorGrid = {
    {8, 6, 7},
    {5, 3, 0}
};

int arrayGrid[2][3] = {
    {8, 6, 7},
    {5, 3, 0}
};

Both structures use grid[row][column]. The important change is the storage contract, not the mental model.

The Same Shape, a Fixed Contract

Property Multidimensional vector Multidimensional array
Shape Can change at runtime Fixed at compile time
Rows May have different lengths Always rectangular
Access grid[row][col] grid[row][col]
Storage Separate vector objects One contiguous block

Rows and Columns

Column 0 Column 1 Column 2
Row 0 grid[0][0] grid[0][1] grid[0][2]
Row 1 grid[1][0] grid[1][1] grid[1][2]

[Graphic suggestion: highlight one cell and trace its row index first, then its column index.]

Declaring a Two-Dimensional Array

const int ROWS = 2;
const int COLS = 3;

int grid[ROWS][COLS];
  • ROWS is the number of arrays
  • COLS is the number of elements in each row

Named Dimensions

const int DAYS = 7;
const int HOURS = 24;

double temperatures[DAYS][HOURS];
  • Dimensions must be compile-time constants
  • Names keep declarations and loops synchronized

Valid Index Ranges

  • Valid rows: 0 through ROWS - 1
    • grid[0] is the first row
    • grid[ROWS - 1] is the last row
  • Valid columns: 0 through COLS - 1
    • Each row has the same column range

Initializing a Grid

int grid[2][3] = {
    {8, 6, 7},
    {5, 3, 0}
};
  • Each inner brace initializes one row
  • Values inside that brace fill columns from left to right

Partial Initialization

int grid[2][3] = {
    {8, 6},
    {5}
};
Column 0 Column 1 Column 2
Row 0 8 6 0
Row 1 5 0 0

Reading and Writing Elements

int score = grid[1][2];

grid[0][1] = 42;
grid[1][2]++;
  • Both subscripts identify one element
  • That element behaves like an ordinary variable

Traversing Every Element

for (int row = 0; row < ROWS; row++) {
    for (int col = 0; col < COLS; col++) {
        cout << grid[row][col] << " ";
    }
    cout << "\n";
}

Displaying a Grid

void displayGrid(const int grid[][COLS], int rows) {
    for (int row = 0; row < rows; row++) {
        for (int col = 0; col < COLS; col++) {
            cout << setw(4) << grid[row][col];
        }
        cout << "\n";
    }
}

The complete program declares the function prototype above main and defines the function below main.

Processing Rows

for (int row = 0; row < ROWS; row++) {
    int total = 0;

    for (int col = 0; col < COLS; col++) {
        total += grid[row][col];
    }

    cout << "Row " << row << ": " << total << "\n";
}

Processing Columns

for (int col = 0; col < COLS; col++) {
    int total = 0;

    for (int row = 0; row < ROWS; row++) {
        total += grid[row][col];
    }

    cout << "Column " << col << ": " << total << "\n";
}

Rectangular and Jagged Shapes

int fixedGrid[3][4];

vector<vector<int>> jaggedGrid = {
    {1, 2},
    {3, 4, 5, 6},
    {7}
};
  • A built-in multidimensional array is rectangular
  • A vector of vectors can be jagged

Memory Is Still Linear

grid[0][0]  grid[0][1]  grid[0][2]  grid[1][0]  grid[1][1]  grid[1][2]
  • C++ stores one complete row, then the next row
  • Row-first traversal follows that storage order

[Graphic suggestion: flatten a 2-by-3 table into a six-cell memory strip with matching colors.]

Passing a Grid to a Function

const int COLS = 3;

void displayGrid(const int grid[][COLS], int rows);
  • The first dimension may be passed separately
  • The compiler needs every remaining dimension to locate an element

Beyond Two Dimensions

const int FLOORS = 4;
const int ROWS = 3;
const int COLS = 5;

int seats[FLOORS][ROWS][COLS] = {};
seats[2][1][4] = 1;
  • Add one subscript and one loop for each dimension
  • The same indexing pattern continues

Choosing the Right Shape

  • Use a multidimensional array when every dimension is fixed and rectangular
  • Use a vector grid when dimensions must change at runtime
    • A vector of vectors may also represent jagged rows
  • Choose dimensions that match the problem: row and column, floor and seat, or layer and pixel

Summary

  • Multidimensional arrays extend familiar indexing with one subscript per dimension
  • Named dimensions, nested loops, and row-major traversal keep the shape explicit

C-Style Arrays and C-Strings

The Arrays We Already Know?

  • int scores[10] is a C-style array
  • The name distinguishes it from modern C++ containers

Slides 24–45

Three Array Families

Structure Size Interface
C-style array Fixed at compile time Built-in syntax
std::array Fixed at compile time Container interface
vector Runtime and resizable Container interface

Why Keep the Name?

  • C++ inherited type name[size] arrays from C
  • They remain useful for C-compatible and low-level interfaces
  • In modern C++, the name signals a more limited contract than std::array

Character Arrays

char initials[4] = {'P', '3', '6', '0'};
  • Each element stores one character
  • An ordinary character array is not automatically a string

A String Needs a Terminator

char course[5] = {'P', '3', '6', '0', '\0'};
  • \0 marks the end of a C-string
  • \0, '0', and nullptr mean different things

Initializing a C-String

char course[] = "P360";

char sameCourse[5] = {
    'P', '3', '6', '0', '\0'
};
  • A string literal adds the terminator
  • The first array therefore contains five elements

Capacity and Length

char name[20] = "Ada";
  • Capacity: 20 character positions
  • Length: 3 characters before \0
  • Unused capacity allows the text to grow

sizeof Versus strlen

char name[20] = "Ada";

cout << sizeof(name) << "\n";  // Array capacity in bytes
cout << strlen(name) << "\n";  // Number of characters
  • sizeof reports the array storage in this scope
  • strlen counts characters before the terminator

Processing Until \0

for (int index = 0; text[index] != '\0'; index++) {
    cout << text[index] << "\n";
}
  • The terminator is the loop sentinel
  • The loop does not need to visit unused capacity

Counting Vowels

int countVowels(const char text[]) {
    int count = 0;

    for (int i = 0; text[i] != '\0'; i++) {
        char letter = tolower(text[i]);
        if (letter == 'a' || letter == 'e' || letter == 'i' ||
            letter == 'o' || letter == 'u') {
            count++;
        }
    }

    return count;
}

Converting to Uppercase

void convertToUpper(char text[]) {
    for (int i = 0; text[i] != '\0'; i++) {
        text[i] = toupper(text[i]);
    }
}
  • The function changes the caller’s character array
  • The terminator remains in place

Reversing a C-String

void reverseText(char text[]) {
    int left = 0;
    int right = strlen(text) - 1;

    while (left < right) {
        char temp = text[left];
        text[left] = text[right];
        text[right] = temp;
        left++;
        right--;
    }
}

Reading C-Strings

const int CAPACITY = 80;
char title[CAPACITY] = {};

cin.getline(title, CAPACITY);
  • cin >> title stops at whitespace
  • cin.getline can read spaces and receives the destination capacity

Measuring C-Strings

#include <cstring>

char message[] = "Hello";
cout << strlen(message) << "\n";
  • strlen returns the number of characters before \0
  • The terminator is not included in the length

Comparing C-Strings

int result = strcmp(first, second);

if (result == 0) {
    cout << "The text matches\n";
}
  • Negative means first comes before second
  • Zero means equal; positive means first comes after second

Copying and Combining

char destination[30] = {};

strcpy(destination, "Programming");
strcat(destination, " 360");
  • strcpy replaces the destination text
  • strcat appends to it
  • The destination must have enough capacity for the result and \0

Crossing a Function Boundary

void readTitle(char title[], int capacity) {
    cin.getline(title, capacity);
}
  • A C-style array parameter does not retain its capacity
  • Pass capacity when the function may write into the array

C-Strings and string

C-string string
char[] plus \0 Class that manages its storage
Capacity managed manually Grows when needed
<cstring> functions Member functions and operators
Needed by some C interfaces Preferred for new C++ text

Where C-Style Arrays Still Appear

  • Existing C and C++ libraries
  • Embedded and low-level systems with strict storage requirements
  • Fixed multidimensional arrays
  • Interfaces that exchange raw buffers

The Tradeoff

  • C-style arrays provide fixed, contiguous storage with minimal machinery
  • They do not know their own size or provide a container interface
  • Prefer a modern C++ container unless compatibility requires the built-in form

Summary

  • Previously studied built-in arrays are C-style arrays
  • A C-string adds a \0 convention that requires explicit capacity and length management

std::array

A Modern Fixed-Size Array?

  • Keep contiguous, compile-time storage
  • Add the interface expected from a C++ container

Slides 46–67

Declaring a std::array

#include <array>

using namespace std;

array<int, 5> scores;
  • First template argument: element type
  • Second template argument: fixed number of elements

Initializing the Elements

array<int, 5> scores = {91, 85, 88, 94, 90};

array<int, 5> zeros = {};
  • Brace initialization fills elements in order
  • Empty braces value-initialize every element to zero

The Size Is Part of the Type

array<int, 5> quizScores;
array<int, 10> examScores;
  • These are different types
  • The size cannot change while the program runs

Familiar Indexing

scores[0] = 91;
scores[4] = 90;

cout << scores[2] << "\n";
  • std::array preserves ordinary subscript syntax
  • Valid indexes remain 0 through size - 1

Knowing Its Own Size

for (int index = 0; index < scores.size(); index++) {
    cout << scores[index] << "\n";
}
  • .size() travels with the container
  • A separate size variable is unnecessary

Range-Based Traversal

for (int score : scores) {
    cout << score << "\n";
}
  • The container supplies its beginning and end
  • Use a reference when the loop should modify elements

Checked Access

cout << scores[2] << "\n";
cout << scores.at(2) << "\n";
  • [] performs unchecked access
  • .at() checks the index and throws out_of_range when invalid

First and Last Elements

cout << scores.front() << "\n";
cout << scores.back() << "\n";
  • .front() names the first element
  • .back() names the last element

Passing by Reference

void addBonus(array<int, 5>& scores) {
    for (int& score : scores) {
        score += 2;
    }
}
  • std::array does not decay into a pointer
  • The parameter retains its fixed size and container interface

Read-Only Parameters

void display(const array<int, 5>& scores) {
    for (int score : scores) {
        cout << score << " ";
    }
}
  • const prevents element changes
  • & avoids copying the whole container

Whole-Array Value Semantics

array<int, 3> first = {1, 2, 3};
array<int, 3> second = first;

second = first;
cout << (first == second) << "\n";
  • Copy, assignment, return, and comparison operate on the entire container
  • C-style arrays do not provide these operations directly

Standard Algorithms

#include <algorithm>

sort(scores.begin(), scores.end());

auto found = find(scores.begin(), scores.end(), 94);
  • Iterators connect std::array to the standard algorithms library
  • The same algorithm style works with other containers

Same Storage, Better Interface

Property C-style array std::array
Extent Compile-time Compile-time
Storage Contiguous Contiguous
Dynamic allocation No No
Member functions No Yes
Whole-container operations No Yes

Interoperability with .data()

array<int, 5> scores = {91, 85, 88, 94, 90};

legacyFunction(scores.data(), scores.size());
  • .data() returns a pointer to the first element
  • Use it only when an interface requires a raw buffer

Still Not Resizable

array<int, 5> fixedScores;
vector<int> growingScores;
  • std::array modernizes a fixed-size array
  • vector remains the modern choice for a runtime or changing size

Multidimensional std::array

array<array<int, 3>, 2> grid = {{
    {8, 6, 7},
    {5, 3, 0}
}};

cout << grid[1][2] << "\n";
  • The outer array stores rows
  • Each inner array stores one fixed-length row

Complete std::array Program

#include <algorithm>
#include <array>
#include <iostream>
using namespace std;

void display(const array<int, 5>& values);

int main() {
    array<int, 5> values = {42, 7, 19, 3, 11};
    sort(values.begin(), values.end());
    cout << "Count: " << values.size() << "\n";
    display(values);
    return 0;
}

void display(const array<int, 5>& values) {
    for (int value : values) {
        cout << value << " ";
    }
    cout << "\n";
}

Three Choices

Need Default choice
Fixed size, modern C++ std::array
Runtime or changing size vector
C-compatible raw buffer C-style array or .data()

Which One Should You Use?

  • Default to std::array when the size is fixed at compile time
  • Default to vector when the size is determined or changed at runtime
  • Use a C-style array when compatibility or a specific low-level interface requires it

Summary

  • std::array is the modern C++ fixed-size alternative to a C-style array
  • It preserves contiguous storage while adding size awareness, value semantics, and standard-library support

Key Takeaways

  • Multidimensional arrays apply one index per dimension and store rectangular data in row-major order
  • C-strings are character arrays whose meaningful text ends at \0
  • std::array is the modern C++ default for fixed-size arrays
  • Use vector when the size must be determined or changed at runtime
