Recall that a pointer stores a typed address, dereferencing accesses its target, and array expressions often produce a pointer to the first element. This lecture concentrates on what decay discards and how programs represent an absent target safely.

Today’s Agenda

  • Array Decay
  • Null Pointers

The lecture moves from the array-to-pointer conversion and its lost extent to deliberate no-object states and nullable interfaces.

Array Decay

What Survives the Conversion?

  • An array stores elements and knows its fixed extent
  • Most expressions convert that array to a pointer to its first element

Slides 3–25

An Array Is More Than an Address

int values[5] = {10, 20, 30, 40, 50};
  • The type is “array of five int values”
  • The object contains all five elements in contiguous storage
  • Its extent is part of the array type

Decay Produces the First Element’s Address

int values[5] = {10, 20, 30, 40, 50};

int* first = values;

cout << values << "\n";
cout << &values[0] << "\n";
cout << first << "\n";

Decay in an Assignment

#include <iostream>
using namespace std;

int main() {
    int scores[4] = {82, 91, 76, 88};
    int* ptr = scores;

    cout << *ptr << "\n";
    return 0;
}

Indexing Still Works Through the Pointer

int values[4] = {5, 10, 15, 20};
int* ptr = values;

for (int i = 0; i < 4; i++) {
    cout << values[i] << " " << ptr[i] << "\n";
}
  • Both expressions access the same elements

Pointer Arithmetic After Decay

int values[4] = {5, 10, 15, 20};
int* ptr = values;

for (int i = 0; i < 4; i++) {
    cout << *(ptr + i) << "\n";
}

Decay Does Not Copy the Array

int values[3] = {2, 4, 6};
int* ptr = values;

ptr[1] = 40;
*(ptr + 2) = 60;

cout << values[1] << " " << values[2] << "\n";

Function Parameters Trigger Decay

void printValues(const int values[], int size);
void printValues(const int* values, int size);
  • These declarations describe the same parameter type
  • The function receives a pointer, not a complete array object

The Brackets Do Not Preserve the Size

void change(int values[100]);

int main() {
    int small[3] = {1, 2, 3};
    change(small);
    return 0;
}

void change(int values[100]) {
    values[0] = 99;
}

sizeof Before the Function Call

int values[6] = {4, 8, 12, 16, 20, 24};

cout << sizeof(values) << "\n";
cout << sizeof(values[0]) << "\n";

int count = sizeof(values) / sizeof(values[0]);
cout << count << "\n";

sizeof Inside the Function

void inspect(int values[]) {
    cout << sizeof(values) << "\n";
}
  • values is a pointer parameter inside the function
  • sizeof(values) reports the pointer size, not the caller’s array size

The Missing Size Must Be Restored

void printValues(const int* values, int size) {
    for (int i = 0; i < size; i++) {
        cout << values[i] << "\n";
    }
}
  • The pointer identifies the first element
  • The separate size states how many elements belong to the range

Pointer and Size Form One Contract

int values[3] = {10, 20, 30};

printValues(values, 3); // correct range
printValues(values, 5); // function trusts an incorrect size
  • A raw pointer carries no bounds information
  • The caller and function must agree on the range

Begin and End Form a Range

void printRange(const int* begin, const int* end) {
    for (const int* current = begin; current != end; current++) {
        cout << *current << "\n";
    }
}

int values[5] = {3, 6, 9, 12, 15};
printRange(values, values + 5);
printRange(values + 1, values + 4);

The second call describes the subrange containing 6, 9, and 12. The end pointer is a boundary and is never dereferenced.

Read-Only Array Parameters

int sum(const int* values, int size) {
    int total = 0;
    for (int i = 0; i < size; i++) {
        total += values[i];
    }
    return total;
}
  • const prevents the function from changing caller elements

Returning a Position

int* findValue(int* values, int size, int target) {
    for (int i = 0; i < size; i++) {
        if (values[i] == target) {
            return &values[i];
        }
    }
    return nullptr;
}

Decay Is Context-Dependent

int values[5] = {};

int* elementPtr = values;
int (*arrayPtr)[5] = &values;

cout << sizeof(values) << "\n";
cout << sizeof(elementPtr) << "\n";
  • sizeof(values) examines the array without decay
  • &values produces a pointer to the whole array

C-Style Arrays Versus array

#include <array>
using namespace std;

void inspect(const array<int, 5>& values) {
    cout << values.size() << "\n";
    cout << values.at(2) << "\n";
}
  • Passing by reference preserves the complete container type

A Two-Dimensional Array Decays by Rows

int grid[3][4] = {};

int (*rowPtr)[4] = grid;
cout << rowPtr[1][2] << "\n";
  • grid decays to a pointer to its first four-element row
  • The column extent must remain known for row arithmetic

An Array Reference Preserves Its Extent

void inspect(int (&values)[5]) {
    int count = sizeof(values) / sizeof(values[0]);
    cout << count << "\n";
}
  • Only an array of exactly five int values can bind to this parameter
  • No array-to-pointer conversion occurs

Choose an Interface That Preserves the Contract

Interface Size preserved Typical use
int* and size Separately Legacy or low-level interface
begin and end pointers As a range Subranges and algorithms
array reference In the type Exact C-style array extent
array& or vector& In the object Modern container code

Array Decay Summary

  • Decay preserves access to the first element but discards the array extent
  • Raw-pointer interfaces must restore bounds through a size, an end pointer, or a stronger type

Null Pointers

What If There Is No Object?

  • A search can finish without finding a matching element
  • A pointer needs a deliberate state for “no result”

Slides 26–47

A Pointer Needs a Defined State

int value = 42;

int* valid = &value;
int* empty = nullptr;
int* unknown;
Pointer State
valid selects a live int
empty deliberately selects nothing
unknown indeterminate and unusable

nullptr Means No Target

int* selected = nullptr;

cout << selected << "\n";
  • nullptr is the C++ null pointer literal
  • It represents the absence of an object address
  • It is a state to test, not an object to access

Initialize Optional Pointers

int score = 88;
bool scoreIsReady = false;

int* selected = nullptr;
if (scoreIsReady) {
    selected = &score;
}

Test Before Dereferencing

if (selected != nullptr) {
    cout << *selected << "\n";
} else {
    cout << "No score selected\n";
}
  • The test separates the object-present and object-absent paths

Short-Circuit Evaluation Protects Access

int* ptr = nullptr;

if (ptr != nullptr && *ptr > 0) {
    cout << "positive\n";
}
  • && stops when its left operand is false
  • *ptr is evaluated only when an address is present

Null Pointers Cannot Be Dereferenced

int* ptr = nullptr;

// No int object exists at the selected location.
cout << *ptr << "\n";
  • Dereferencing a null pointer is undefined behavior
  • A null check must occur before the access

Returning nullptr From Search

int* findValue(int* values, int size, int target) {
    for (int i = 0; i < size; i++) {
        if (values[i] == target) {
            return &values[i];
        }
    }
    return nullptr;
}

The Caller Handles Both Outcomes

int values[5] = {12, 18, 25, 31, 44};
int* found = findValue(values, 5, 25);

if (found != nullptr) {
    cout << "value: " << *found << "\n";
    cout << "index: " << found - values << "\n";
} else {
    cout << "not found\n";
}

Nullable Function Parameters

void recordScore(int score, int* latest) {
    cout << "score: " << score << "\n";
    if (latest != nullptr) {
        *latest = score;
    }
}
  • A pointer parameter can state that an object is optional

Required Parameters Should Not Be Nullable

void addBonus(int& score) {
    score += 5;
}

void addBonusIfPresent(int* score) {
    if (score != nullptr) {
        *score += 5;
    }
}
  • Use a reference when the function requires an object
  • Use a pointer when absence is part of the contract

Optional Output Parameters

void analyze(const int* values, int size,
             int* minimum, int* maximum) {
    if (minimum != nullptr) *minimum = values[0];
    if (maximum != nullptr) *maximum = values[0];

    for (int i = 1; i < size; i++) {
        if (minimum != nullptr && values[i] < *minimum)
            *minimum = values[i];
        if (maximum != nullptr && values[i] > *maximum)
            *maximum = values[i];
    }
}

nullptr Is Type-Safe

int* first = nullptr;
double* second = nullptr;

int* legacyA = 0;
int* legacyB = NULL;
  • Prefer nullptr in modern C++
  • It represents a null pointer without also acting as an integer literal

Overloads Reveal the Difference

void inspect(int value);
void inspect(int* ptr);

inspect(0);       // calls inspect(int)
inspect(nullptr); // calls inspect(int*)
  • nullptr communicates pointer intent to the compiler

Resetting Records No Target

int score = 88;
int* selected = &score;

cout << *selected << "\n";
selected = nullptr;
  • Resetting the pointer records that it no longer selects an object
  • It does not destroy score

One Null Assignment Does Not Fix Aliases

int value = 42;
int* first = &value;
int* second = first;

first = nullptr;

cout << *second << "\n";
  • Each pointer stores its own copy of the address
  • Changing one pointer does not change another

Null Is Only One Safety Check

Question What it establishes
ptr != nullptr an address is present
Is the target alive? its lifetime has not ended
Is the position in range? dereferencing selects an element
  • A non-null pointer is not automatically valid

Pointers Convert to Boolean Conditions

if (ptr) {
    cout << *ptr << "\n";
}

if (!ptr) {
    cout << "No object\n";
}
  • if (ptr) means if (ptr != nullptr)
  • Use the style consistently within a program

Guard the Arrow Operator Too

struct Student {
    string name;
    int score;
};

Student* selected = nullptr;
if (selected != nullptr) {
    cout << selected->name << " " << selected->score;
}
  • selected->score accesses a member through the pointer

Deleting nullptr Does Nothing

int* ptr = nullptr;

delete ptr;
  • C++ permits deleting a null pointer
  • Dynamic allocation and ownership arrive next lecture

Null Pointer Summary

  • Initialize optional pointers with nullptr and test before every indirect access
  • Use nullable pointers only when “no object” is a meaningful part of the interface

Key Takeaways

  • Array decay produces a pointer to the first element and discards the array extent
  • Raw array interfaces must carry bounds through a size, an end pointer, or a stronger type
  • nullptr represents a deliberate no-object state and must be tested before indirect access
  • A non-null pointer may still be invalid when its target is out of range or no longer alive

Appendix

Array Decay: Good Practices

  • Pair every raw array pointer with an explicit size or end pointer
  • Use const for read-only access and prefer containers when their interfaces fit
  • Keep pointer and bounds arguments adjacent in function declarations

Array Decay: Common Mistakes

  • Expecting brackets in a function parameter to preserve the array size
  • Using sizeof on an array parameter to count caller elements
  • Passing a pointer with a size that exceeds the actual range

Array Decay: More Examples

double average(const int* begin, const int* end) {
    int total = 0;
    int count = 0;
    for (const int* ptr = begin; ptr != end; ptr++) {
        total += *ptr;
        count++;
    }
    return count == 0 ? 0.0 : static_cast<double>(total) / count;
}

Null Pointers: Good Practices

  • Initialize every optional pointer with a valid address or nullptr
  • Test nullable pointers before *ptr and ptr->member
  • Prefer references when a function requires an object

Null Pointers: Common Mistakes

  • Treating an uninitialized pointer as though it were null
  • Assuming non-null automatically means live and in bounds
  • Resetting one alias and assuming every copied pointer also changed

Null Pointers: More Examples

const int* chooseLarger(const int* first, const int* second) {
    if (first == nullptr) return second;
    if (second == nullptr) return first;
    return *first >= *second ? first : second;
}
