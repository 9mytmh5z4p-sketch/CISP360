Recall that a pointer stores a typed address, dereferencing accesses its target, and array expressions often produce a pointer to the first element. This lecture concentrates on what decay discards, how programs represent absence, and how pointer parameters express function-boundary contracts.

Today’s Agenda

  • Array Decay
  • Null Pointers
  • Passing Pointers

The lecture moves from array-to-pointer conversion, to deliberate no-object states, to pointer parameters as interface design.

Array Decay

When an Array Stops Looking Like an Array

  • Why does an array seem to change when we pass it to a function?
  • The short answer: most array expressions automatically become pointers.

Slides 3–24

The Conversion

  • Array decay is the automatic conversion from an array expression to a pointer to the first element.
  • You do not call decay. C++ applies it in many expression contexts.
int values[4] = {10, 20, 30, 40};
int* first = values;

cout << *first << "\n";

Why This Matters

  • Decay is why C-style array functions need an explicit boundary.
  • The function can reach the elements, but it cannot know the size by looking at the pointer alone.

What Survives Decay

Before decay After decay
Array element type Element type
First element address First element address
Number of elements Lost
  • The pointer is not the array.

Before Decay

int scores[5] = {82, 91, 76, 88, 94};
  • scores is an array object
  • Its type is “array of five int values”
  • Its extent is part of the array type

[Graphic suggestion: show a five-cell array labeled scores, with indexes 0 through 4 and a bracket showing the whole array object.]

The Pointer Is Not the Array

int scores[5] = {82, 91, 76, 88, 94};
int* ptr = scores;

cout << scores << "\n";
cout << &scores[0] << "\n";
cout << ptr << "\n";
  • The printed addresses may match
  • The types and the information carried do not match

Predict the Output

void inspect(int values[]);

int main() {
    int values[6] = {4, 8, 12, 16, 20, 24};

    cout << sizeof(values) << "\n";
    inspect(values);
    return 0;
}
  • Will both lines report the same size?

sizeof in main

int values[6] = {4, 8, 12, 16, 20, 24};

cout << sizeof(values) << "\n";
cout << sizeof(values[0]) << "\n";

int count = sizeof(values) / sizeof(values[0]);
cout << count << "\n";
  • In the caller, values is still the whole array object.

sizeof in a Function

void inspect(int values[]) {
    cout << sizeof(values) << "\n";
}
  • Inside the function, values is a pointer parameter.
  • sizeof(values) reports the pointer size, not the caller’s array size.

Parameter Brackets Do Not Save the Size

void printValues(const int values[], int size);
void printValues(const int* values, int size);
  • These declarations describe the same parameter type.
  • The brackets may look array-like, but the function receives a pointer.

The Boundary Must Travel Separately

void printValues(const int* values, int size) {
    for (int i = 0; i < size; i++) {
        cout << values[i] << "\n";
    }
}
  • The pointer identifies where the range starts.
  • The size states how many elements belong to the range.

Passing Array and Size

#include <iostream>
using namespace std;

void printValues(const int values[], int size);

int main() {
    int values[4] = {7, 14, 21, 28};
    printValues(values, 4);
    return 0;
}

void printValues(const int values[], int size) {
    for (int i = 0; i < size; i++) {
        cout << values[i] << "\n";
    }
}

Mutation Through a Decayed Array

void curveScores(int scores[], int size) {
    for (int i = 0; i < size; i++) {
        scores[i] += 5;
    }
}
  • The function receives a pointer.
  • Writing through that pointer changes the caller’s array elements.

Read-Only Array Parameters

int sum(const int* values, int size) {
    int total = 0;
    for (int i = 0; i < size; i++) {
        total += values[i];
    }
    return total;
}
  • Use const when the function should read elements without changing them.

Begin and End Pointers

int values[5] = {3, 6, 9, 12, 15};

const int* begin = values;
const int* end = values + 5;
  • begin points to the first element.
  • end points one past the last element and is a boundary, not a value to read.

[Graphic suggestion: show begin pointing to index 0 and end pointing just beyond index 4.]

Walking a Pointer Range

void printRange(const int* begin, const int* end) {
    for (const int* current = begin; current != end; current++) {
        cout << *current << "\n";
    }
}
  • The loop stops at the boundary.
  • It never dereferences end.

Subarrays Without Copying

int values[6] = {10, 20, 30, 40, 50, 60};

printValues(values, 6);
printValues(values + 2, 3);
printRange(values + 1, values + 5);
  • A pointer can describe a smaller range inside a larger array.

Two-Dimensional Array Decay

void printGrid(const int grid[][3], int rows) {
    for (int r = 0; r < rows; r++) {
        for (int c = 0; c < 3; c++) {
            cout << grid[r][c] << " ";
        }
        cout << "\n";
    }
}
  • The row count can travel separately.
  • The column count must still be known by the function type.

Preserving the Full Array Type

void printFour(const int (&values)[4]) {
    for (int value : values) {
        cout << value << "\n";
    }
}
  • A reference-to-array parameter preserves the array extent.
  • This is useful to recognize, but pointer-plus-size is the main pattern today.

Choosing an Array Interface

Interface Best fit
const int* values, int size Simple read-only raw array
int* values, int size Caller-visible element changes
begin, end Subranges and pointer traversal
vector<int>& Resizable collection with size attached

What to Remember About Decay

  • Array decay gives a function access to elements, not the array’s extent.
  • The pointer is not the array; the boundary must travel separately.

Null Pointers

When a Pointer Has No Target

  • What should a pointer store when there is no object to point at?
  • C++ gives us a deliberate no-object value: nullptr.

Slides 25–47

Null Is a Value, Not a Mistake

  • nullptr is the modern C++ value meaning “points to no object.”
  • A null pointer is safe when its meaning is intentional and checked before access.
int* selected = nullptr;

Why This Matters

  • Pointers sometimes need to represent absence: not found, no current selection, or optional information.
  • nullptr lets us represent absence without using an unknown address.

nullptr Is Not an Address You Can Use

int value = 42;

int* valid = &value;
int* empty = nullptr;
int* unknown;
Pointer Meaning
valid points to a live int
empty deliberately points to nothing
unknown uninitialized and unusable

Initialize to Something Known

int score = 88;
bool ready = false;

int* selected = nullptr;
if (ready) {
    selected = &score;
}
  • A pointer should begin as either a valid address or nullptr.

Predict the Branch

int* current = nullptr;

if (current == nullptr) {
    cout << "nothing selected\n";
} else {
    cout << *current << "\n";
}
  • Which branch runs?

Testing for nullptr

if (current == nullptr) {
    cout << "nothing selected\n";
}

if (current != nullptr) {
    cout << *current << "\n";
}
  • The test separates the object-absent path from the object-present path.

Pointer Truth Tests

if (current) {
    cout << *current << "\n";
}

if (!current) {
    cout << "nothing selected\n";
}
  • This shorthand is common.
  • Explicit nullptr comparisons are clearer while the habit is new.

Guard Before Dereference

int* current = nullptr;

if (current != nullptr) {
    cout << *current << "\n";
}
  • The null check is the gate before *current.
  • Do not dereference a pointer that points to no object.

Safe Dereference Example

void printIfPresent(const int* value) {
    if (value != nullptr) {
        cout << "value: " << *value << "\n";
    } else {
        cout << "no value\n";
    }
}

Null Is Safer Than Wild

int* safeEmpty = nullptr;
int* wild;

if (safeEmpty == nullptr) {
    cout << "known empty state\n";
}
  • nullptr can be tested.
  • An uninitialized pointer contains an indeterminate value.

Null Does Not Mean Valid Later

  • A pointer can be non-null and still be invalid.
  • Null checks answer “does this pointer equal nullptr?” not “is this target alive and in range?”

Keep this as a signpost. Heap lifetime, leaks, and use-after-free are intentionally deferred until after dynamic allocation.

Nullable Return Values

int* findValue(int* values, int size, int target) {
    for (int i = 0; i < size; i++) {
        if (values[i] == target) {
            return &values[i];
        }
    }
    return nullptr;
}

Calling a Nullable Return Function

int values[5] = {12, 18, 25, 31, 44};
int* found = findValue(values, 5, 25);

if (found != nullptr) {
    cout << "value: " << *found << "\n";
    cout << "index: " << found - values << "\n";
} else {
    cout << "not found\n";
}

What Does nullptr Mean Here?

  • Every nullable pointer needs a documented meaning.
  • nullptr might mean “not found,” “optional input omitted,” or “no current selection.”

Nullable Parameters

void printScore(const int* score) {
    if (score == nullptr) {
        cout << "score not available\n";
    } else {
        cout << "score: " << *score << "\n";
    }
}
  • The function contract must say whether nullptr is allowed.

Required Object? Use a Reference

void addBonus(int& score) {
    score += 5;
}

void addBonusIfPresent(int* score) {
    if (score != nullptr) {
        *score += 5;
    }
}
  • Use a reference when the object is required.
  • Use a pointer when absence is meaningful.

The Arrow Operator With Nullable Pointers

struct Player {
    string name;
    int score;
};

void printPlayer(const Player* player) {
    if (player != nullptr) {
        cout << player->name << ": " << player->score << "\n";
    }
}
  • player->name means (*player).name.

Deleting nullptr Is Safe, But Not the Point

int* ptr = nullptr;
delete ptr;
  • C++ allows deleting nullptr.
  • Dynamic allocation and ownership belong in the next lecture.

A Null Check Is Not a Lifetime Check

if (ptr != nullptr) {
    cout << *ptr << "\n";
}
  • This check protects the immediate dereference from a null pointer.
  • It does not prove the pointer is in bounds or still points to a live object.

Choosing a Null Contract

Situation Prefer
Object is required Reference
Object is optional Nullable pointer
Search may fail Pointer return or index sentinel
Array range may be empty Pointer plus size or begin/end

Null Pointer Summary

  • nullptr represents intentional absence.
  • Dereferencing requires more than “not null”: the target must also be valid for the operation.

Passing Pointers

What Crosses the Function Boundary?

  • When a pointer is passed to a function, the address crosses the boundary.
  • The function gets its own pointer variable, but it may share access to the caller’s object.

Slides 48–72

Passing an Address

  • Passing a pointer means passing an address value into a pointer parameter.
  • The parameter can use that address to read or write the pointed-to object.
void printValue(const int* value);

int score = 91;
printValue(&score);

The Two Things to Track

Thing Where it lives
Pointer parameter Local to the called function
Pointed-to object Wherever the address came from
  • The pointer is passed by value.
  • The target may still be shared.

Anatomy of a Pointer Parameter

void addBonus(int* score);

int main() {
    int exam = 85;
    addBonus(&exam);
    return 0;
}

void addBonus(int* score) {
    *score += 5;
}
  • &exam supplies the address.
  • *score accesses the caller’s exam.

Pointer Parameter Trace

int exam = 85;
addBonus(&exam);
cout << exam << "\n";
Moment exam score inside function
before call 85 does not exist
during call 85 address of exam
after *score += 5 90 address of exam

Read Through a Pointer

void printValue(const int* value) {
    if (value != nullptr) {
        cout << "value: " << *value << "\n";
    }
}
  • const int* means the function may read the target but not change the int.

Write Through a Pointer

void resetScore(int* score) {
    if (score != nullptr) {
        *score = 0;
    }
}
  • Writing through *score changes the caller-owned object.

Predict the Output

void doubleValue(int* value) {
    *value *= 2;
}

int main() {
    int number = 12;
    doubleValue(&number);
    cout << number << "\n";
    return 0;
}
  • What prints after the function returns?

Changing ptr Versus Changing *ptr

void changeBoth(int* ptr) {
    *ptr = 99;
    ptr = nullptr;
}

int value = 10;
int* original = &value;
changeBoth(original);
  • *ptr = 99 changes value.
  • ptr = nullptr changes only the local pointer copy.

Pointer Parameter Versus Reference Parameter

void addOnePointer(int* value) {
    if (value != nullptr) {
        *value += 1;
    }
}

void addOneReference(int& value) {
    value += 1;
}
  • Both can modify the caller’s object.
  • The pointer version can also represent absence.

Required Target? Prefer a Reference

void curveRequired(int& score) {
    score += 5;
}
  • A reference says the function requires a real object.
  • There is no null branch to explain or defend.

Optional Target? Use a Nullable Pointer

void curveIfPresent(int* score) {
    if (score != nullptr) {
        *score += 5;
    }
}
  • A pointer is appropriate when nullptr is part of the contract.

Guard Nullable Parameters

void printLabel(const string* label) {
    if (label == nullptr) {
        cout << "(unlabeled)\n";
    } else {
        cout << *label << "\n";
    }
}
  • If the parameter can be null, the function must check before dereferencing.

What Does nullptr Mean Here?

void showScore(const int* score) {
    if (score == nullptr) {
        cout << "score omitted\n";
        return;
    }
    cout << *score << "\n";
}
  • The function’s contract gives nullptr a specific meaning.

Pointer Output Parameters

bool divide(int numerator, int denominator, double* result) {
    if (denominator == 0) {
        return false;
    }
    if (result != nullptr) {
        *result = static_cast<double>(numerator) / denominator;
    }
    return true;
}

Multiple Outputs With Pointers

void findMinMax(const int* values, int size,
                int* minimum, int* maximum) {
    if (size <= 0) return;

    if (minimum != nullptr) *minimum = values[0];
    if (maximum != nullptr) *maximum = values[0];

    for (int i = 1; i < size; i++) {
        if (minimum != nullptr && values[i] < *minimum)
            *minimum = values[i];
        if (maximum != nullptr && values[i] > *maximum)
            *maximum = values[i];
    }
}

Pointer Parameters and Arrays

void printValues(const int* values, int size) {
    for (int i = 0; i < size; i++) {
        cout << values[i] << "\n";
    }
}

int scores[3] = {88, 91, 76};
printValues(scores, 3);
  • An array argument decays to a pointer at the function boundary.

Pointer Plus Size Contract

int scores[3] = {88, 91, 76};

printValues(scores, 3); // correct
printValues(scores, 5); // broken contract
  • The pointer says where the range begins.
  • The size says where the valid range ends.

Passing a Subrange

int values[6] = {10, 20, 30, 40, 50, 60};

printValues(values, 6);
printValues(values + 2, 3);
  • The second call starts at 30 and processes three elements.

Do Not Return Pointers to Locals

int* badPointer() {
    int local = 42;
    return &local;
}
  • local disappears when the function returns.
  • The returned pointer would refer to an object that no longer exists.

Pointer Parameters in a Prototype

void printValues(const int* values, int size);
void resetScore(int* score);
bool divide(int numerator, int denominator, double* result);
void printLabel(const string* label);
  • const suggests read-only access.
  • A pointer suggests the target may be shared, optional, or range-like.

Caller Responsibilities

  • Pass a valid address when the function requires one.
  • Pass nullptr only when the function contract allows it.
  • Keep pointer ranges paired with the correct boundary.

Use the Least Powerful Parameter

Need Prefer
Small input copied safely Value
Required read-only large object const reference
Required caller-visible change Reference
Optional object Pointer
Raw array range Pointer plus size or begin/end

Passing Pointers Summary

  • Passing a pointer copies an address into a local parameter.
  • Changing *ptr can change the caller’s object; changing ptr changes only the local pointer copy.

Key Takeaways

  • Array decay gives functions access to elements but not the array’s extent
  • nullptr represents intentional absence and must have a documented meaning
  • Passing a pointer copies an address, while dereferencing it may share access to caller-owned data
  • Use the least powerful parameter style that expresses the function’s contract

Appendix

Array Decay: Good Practices

  • Pair every raw array pointer with an explicit size or end pointer
  • Use const for read-only access
  • Prefer containers when the size should travel with the data

Array Decay: Common Mistakes

  • Expecting brackets in a function parameter to preserve the array size
  • Using sizeof on an array parameter to count caller elements
  • Passing a pointer with a size that exceeds the actual range

Array Decay: More Examples

double average(const int* begin, const int* end) {
    int total = 0;
    int count = 0;
    for (const int* ptr = begin; ptr != end; ptr++) {
        total += *ptr;
        count++;
    }
    return count == 0 ? 0.0 : static_cast<double>(total) / count;
}

Null Pointers: More Examples

const int* chooseLarger(const int* first, const int* second) {
    if (first == nullptr) return second;
    if (second == nullptr) return first;
    return *first >= *second ? first : second;
}

bool getFirstPositive(const int* values, int size, int* result) {

            if (result != nullptr) {
                *result = values[i];

            return true;
        }
    }
    return false;
}
