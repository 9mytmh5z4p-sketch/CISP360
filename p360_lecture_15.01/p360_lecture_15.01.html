Pseudodynamic Memory

Fixed Storage, Variable Use?

  • Can a fixed array use a length chosen when the program starts?
  • What remains fixed, and what is allowed to change?

Slides 3–19 Pseudodynamic memory is course terminology, not a standard C++ term.

Capacity Versus Logical Length

  • Capacity is the number of elements physically reserved.
  • Logical length is the number of elements currently treated as valid.
  • The logical length may change, but it must stay between 0 and CAPACITY.

[Graphic suggestion: ten array cells with six marked active and four marked unused.]

Three Meanings of Size

Term Meaning Example
Capacity Physical slots in the array 100
Requested length Amount the user asks to use 8
Logical length Valid elements after validation 8

The requested length becomes the logical length only when it fits the capacity.

The Pseudodynamic Pattern

const int CAPACITY = 100;
double sales[CAPACITY] = {};
int length;

cout << "Number of sales values: ";
cin >> length;

The allocation is fixed at compile time. The program chooses how much of it to use at runtime.

Validate the Requested Length

while (length < 0 || length > CAPACITY)
{
    cout << "Enter a length from 0 to " << CAPACITY << ": ";
    cin >> length;
}

Boundary: valid indexes will be 0 through length - 1.

Initialize Only Active Elements

for (int index = 0; index < length; index++)
{
    cout << "Sale " << index + 1 << ": ";
    cin >> sales[index];
}

The loop follows logical length, not physical capacity.

Display Only Active Elements

for (int index = 0; index < length; index++)
{
    cout << sales[index] << '\n';
}

Predict: if length is 4, how many values appear when capacity is 100?

Complete Fixed-Capacity Program

#include <iostream>
using namespace std;

int main()
{
    const int CAPACITY = 10;
    int scores[CAPACITY] = {};
    int length;

    cout << "How many scores? ";
    cin >> length;
    while (length < 0 || length > CAPACITY)
    {
        cout << "Enter 0 through " << CAPACITY << ": ";
        cin >> length;
    }

    for (int i = 0; i < length; i++)
    {
        cout << "Score " << i + 1 << ": ";
        cin >> scores[i];
    }

    for (int i = 0; i < length; i++)
        cout << scores[i] << ' ';
    cout << '\n';
    return 0;
}

Make the Contract Visible

void display(const int values[], int length);
double average(const int values[], int length);
  • The array parameter provides access to the storage.
  • length identifies the valid range.
  • Capacity is needed only by operations that may add elements.

Compute with Active Elements

#include <iostream>
using namespace std;

double average(const int values[], int length);

int main()
{
    const int CAPACITY = 8;
    int values[CAPACITY] = {12, 18, 24, 30};
    int length = 4;

    cout << average(values, length) << '\n';
    return 0;
}

double average(const int values[], int length)
{
    if (length == 0)
        return 0.0;

    int total = 0;
    for (int i = 0; i < length; i++)
        total += values[i];
    return static_cast<double>(total) / length;
}

Change Use, Not Allocation

length = 3;  // Three elements are logically active
length = 6;  // Six elements are logically active

Changing length does not create or destroy array cells. Newly activated cells must receive valid values before use.

Add at the Logical End

bool append(int values[], int& length, int capacity, int value)
{
    if (length == capacity)
        return false;

    values[length] = value;
    length++;
    return true;
}

Invariant: 0 <= length && length <= capacity.

Remove the Last Element

bool removeLast(int& length)
{
    if (length == 0)
        return false;

    length--;
    return true;
}

The stored bits may remain, but that cell is no longer logically valid.

Trace the Boundary

const int CAPACITY = 5;
int values[CAPACITY] = {4, 7, 9};
int length = 3;

append(values, length, CAPACITY, 12);
removeLast(length);

Predict: after both calls, what is length, and which indexes are valid?

[Graphic suggestion: before/after array cells with a bold boundary at length.]

The Capacity Wall

  • Appending works only while unused physical slots remain.
  • A request beyond capacity cannot be satisfied by changing logical length.
  • Runtime allocation removes the predetermined capacity, but adds ownership duties.

Summary

  • Pseudodynamic memory uses fixed capacity with a runtime logical length.
  • Every operation preserves 0 <= length <= capacity and accesses only valid indexes.

Dynamic Memory and Dynamic Arrays

When Fixed Capacity Is Not Enough

  • What if the program cannot predict a safe capacity?
  • Runtime allocation trades a fixed limit for explicit ownership responsibility.

Slides 20–61

Scalar Ownership Review

  1. Allocate with new.
  2. Use the object through its pointer.
  3. Release it with delete.
  4. Set the pointer to nullptr when it remains in scope.

Invariant: one owner, one matching release.

One Dynamically Allocated Value

#include <iostream>
using namespace std;

int main()
{
    int* score = new int;
    *score = 92;
    cout << *score << '\n';

    delete score;
    score = nullptr;
    return 0;
}

From One Value to Many

int* oneScore = new int;
int* manyScores = new int[count];

The second allocation reserves count contiguous int elements on the heap.

[Graphic suggestion: two stack pointer variables, one pointing to a scalar heap cell and one to an array block.]

Scalar Versus Array Allocation

Allocation Access Matching release
new int *pointer delete pointer
new int[count] pointer[index] delete[] pointer

The bracket form is part of the allocation contract.

Why the Brackets Matter

int* value = new int;
int* values = new int[10];

delete value;
delete[] values;

Mixing scalar and array release forms causes undefined behavior.

Pointer on the Stack, Array on the Heap

  • The pointer variable has its own storage and scope.
  • The heap array remains allocated until its owner releases it.
  • Losing the pointer does not release the array.

[Graphic suggestion: stack frame containing values and length, with an arrow to a heap block. Label the pointer and allocation lifetimes separately.]

Validate Before Allocating

cout << "How many scores? ";
cin >> length;

while (length < 0)
{
    cout << "Enter zero or more: ";
    cin >> length;
}

No allocation should occur until the requested length satisfies the program contract.

Zero Elements Means No Allocation

int* scores = nullptr;

if (length > 0)
    scores = new int[length];

nullptr clearly represents “no array.” Both delete[] nullptr and guarded traversal are safe.

Allocate the Requested Length

int* scores = new int[length];

for (int index = 0; index < length; index++)
    scores[index] = 0;

The allocated length is fixed for the lifetime of this allocation.

Read and Display Values

for (int i = 0; i < length; i++)
{
    cout << "Score " << i + 1 << ": ";
    cin >> scores[i];
}

for (int i = 0; i < length; i++)
    cout << scores[i] << ' ';
cout << '\n';

Boundary: every subscript must remain below length.

Complete Dynamic Array Program

#include <iostream>
using namespace std;

int main()
{
    int length;
    cout << "How many temperatures? ";
    cin >> length;

    if (length <= 0)
        return 0;

    double* temperatures = new double[length];
    for (int i = 0; i < length; i++)
    {
        cout << "Temperature " << i + 1 << ": ";
        cin >> temperatures[i];
    }

    for (int i = 0; i < length; i++)
        cout << temperatures[i] << '\n';

    delete[] temperatures;
    temperatures = nullptr;
    return 0;
}

Three Meanings of Size

  • Requested length: input proposed by the user.
  • Allocated length: elements reserved by the successful new[] expression.
  • Logical length: elements the program currently treats as valid.

These values may match, but the program must track them explicitly.

Pass Pointer and Length Together

void display(const int values[], int length);
double average(const int values[], int length);

The pointer carries an address. The length carries the boundary contract.

Compute Through a Pointer

#include <iostream>
using namespace std;

double average(const int values[], int length);

int main()
{
    int length = 4;
    int* values = new int[length]{10, 20, 30, 40};
    cout << average(values, length) << '\n';
    delete[] values;
    values = nullptr;
    return 0;
}

double average(const int values[], int length)
{
    if (length == 0)
        return 0.0;

    int total = 0;
    for (int i = 0; i < length; i++)
        total += values[i];
    return static_cast<double>(total) / length;
}

Where Are the Bounds?

  • A raw pointer does not know the array length.
  • values[length] is already one element beyond the allocation.
  • An out-of-bounds read or write causes undefined behavior.

One Element Too Far

int* values = new int[3]{4, 8, 12};

cout << values[2] << '\n';  // Last valid element
cout << values[3] << '\n';  // Outside the allocation

delete[] values;

Predict: is any particular output guaranteed for the second statement?

Preserve the Base Address

int* values = new int[4]{3, 6, 9, 12};
int* cursor = values;

cursor++;
cout << *cursor << '\n';

delete[] values;
values = nullptr;

The owning pointer retains the address returned by new[].

Traverse Without Losing Ownership

int* cursor = values;
int* end = values + length;

while (cursor != end)
{
    cout << *cursor << '\n';
    cursor++;
}

cursor moves; values continues to own the base address.

Losing the Only Pointer

int* values = new int[10];
values = new int[20];

The first allocation still exists, but its address is gone. It can no longer be released.

[Graphic suggestion: first heap block with its ownership arrow removed, then a second block receiving the pointer.]

Replace an Allocation Safely

delete[] values;
values = nullptr;

values = new int[newLength];
length = newLength;

Release the old allocation before replacing its only owning pointer when no data must be preserved.

Cleanup Before Every Return

int* values = new int[length];

if (!readValues(values, length))
{
    delete[] values;
    values = nullptr;
    return 1;
}

delete[] values;
values = nullptr;
return 0;

Each exit path must honor the ownership invariant.

One Cleanup Path

int result = 0;
int* values = new int[length];

if (!readValues(values, length))
    result = 1;
else
    display(values, length);

delete[] values;
values = nullptr;
return result;

One release point makes manual ownership easier to audit.

Automatic Array Ownership

#include <memory>
using namespace std;

unique_ptr<int[]> values = make_unique<int[]>(length);
values[0] = 42;

The smart pointer releases its array automatically when its lifetime ends.

Managed Dynamic Array

#include <iostream>
#include <memory>
using namespace std;

int main()
{
    int length;
    cout << "How many values? ";
    cin >> length;

    if (length <= 0)
        return 0;

    unique_ptr<int[]> values = make_unique<int[]>(length);
    for (int i = 0; i < length; i++)
        cin >> values[i];

    for (int i = 0; i < length; i++)
        cout << values[i] << '\n';
    return 0;
}

Lifetime Versus Bounds Safety

unique_ptr<Type[]> provides:

  • Automatic release with the correct array form
  • Exclusive ownership
  • Subscript access

It does not store length or check whether an index is valid.

Why Resizing Needs a New Array

  • An existing allocation cannot grow or shrink in place through C++ syntax.
  • Resizing means allocating a different block and preserving the desired values.
  • Ownership transfers only after the copy succeeds.

The Resize Algorithm

  1. Validate the new length.
  2. Allocate a temporary array.
  3. Copy the elements that fit.
  4. Release the old array.
  5. Point the owner to the new array.
  6. Update the tracked length.

How Many Elements Should Be Copied?

int copyCount = oldLength;
if (newLength < oldLength)
    copyCount = newLength;

The general rule is min(oldLength, newLength).

Grow an Array

int newLength = length + 3;
int* temporary = new int[newLength]{};

for (int i = 0; i < length; i++)
    temporary[i] = values[i];

delete[] values;
values = temporary;
temporary = nullptr;
length = newLength;

Trace a Growth Operation

  1. values owns the old block.
  2. temporary owns the new block.
  3. Both blocks exist during the copy.
  4. The old block is released.
  5. values becomes the owner of the new block.

[Graphic suggestion: five before/after frames with ownership arrows and copied cells highlighted.]

Add One Element

void append(int*& values, int& length, int value)
{
    int* temporary = new int[length + 1];

    for (int i = 0; i < length; i++)
        temporary[i] = values[i];

    temporary[length] = value;
    delete[] values;
    values = temporary;
    temporary = nullptr;
    length++;
}

Both pointer and length use reference parameters because the function changes the caller’s state.

Use the Append Function

#include <iostream>
using namespace std;

void append(int*& values, int& length, int value);

int main()
{
    int length = 3;
    int* values = new int[length]{2, 4, 6};

    append(values, length, 8);
    for (int i = 0; i < length; i++)
        cout << values[i] << ' ';

    delete[] values;
    values = nullptr;
    return 0;
}

void append(int*& values, int& length, int value)
{
    int* temporary = new int[length + 1];
    for (int i = 0; i < length; i++)
        temporary[i] = values[i];
    temporary[length] = value;
    delete[] values;
    values = temporary;
    length++;
}

Shrink an Array

int newLength = length - 1;
int* temporary = new int[newLength];

for (int i = 0; i < newLength; i++)
    temporary[i] = values[i];

delete[] values;
values = temporary;
temporary = nullptr;
length = newLength;

Only retained elements are copied.

Remove the Last Element

bool removeLast(int*& values, int& length)
{
    if (length == 0)
        return false;

    int* temporary = nullptr;
    if (length > 1)
        temporary = new int[length - 1];

    for (int i = 0; i < length - 1; i++)
        temporary[i] = values[i];

    delete[] values;
    values = temporary;
    length--;
    return true;
}

Resize to Any Length

void resize(int*& values, int& length, int newLength)
{
    if (newLength < 0 || newLength == length)
        return;

    int* temporary = nullptr;
    if (newLength > 0)
        temporary = new int[newLength]{};

    int copyCount = length;
    if (newLength < length)
        copyCount = newLength;

    for (int i = 0; i < copyCount; i++)
        temporary[i] = values[i];

    delete[] values;
    values = temporary;
    length = newLength;
}

Resize with unique_ptr<Type[]>

unique_ptr<int[]> next = make_unique<int[]>(newLength);

for (int i = 0; i < copyCount; i++)
    next[i] = values[i];

values = move(next);
length = newLength;

Assignment transfers ownership and automatically releases the old array.

The Cost of Resizing

  • Allocating a new block requires work proportional to the requested storage.
  • Copying retained elements takes O(N) time.
  • Growing by one element repeatedly can perform many unnecessary copies.

Why We Usually Choose vector

vector<int> values = {2, 4, 6};
values.push_back(8);
values.pop_back();

Manual dynamic arrays expose allocation mechanics. vector normally provides the safer application-level abstraction.

Choose the Storage Model

Need Suitable model
Small known maximum Fixed array plus logical length
Study manual ownership Raw dynamic array
Exclusive automatic ownership unique_ptr<Type[]>
General resizable sequence vector<Type>

Summary

  • Dynamic arrays pair new[] with delete[], preserve the base address, and track bounds separately.
  • Resizing allocates, copies, releases, transfers ownership, and updates length; higher-level code usually uses vector.

Key Takeaways

  • Pseudodynamic storage separates fixed capacity from runtime logical length.
  • Dynamic arrays use new[] and the matching delete[] form.
  • Raw arrays do not track their length or check their bounds.
  • Resizing means allocate, copy, release, transfer ownership, and update length.
  • unique_ptr<Type[]> manages lifetime; vector is normally the safer resizable sequence.

Appendix

Pseudodynamic Memory: Good Practices

  • Give capacity and logical length different names.
  • Validate requested length before using it.
  • Pass logical length with the array and capacity to operations that may append.

Pseudodynamic Memory: Common Mistakes

  • Treating every capacity slot as valid data
  • Allowing logical length to exceed capacity
  • Growing logical length without initializing newly active elements

Pseudodynamic Memory: More Examples

bool contains(const int values[], int length, int target)
{
    for (int i = 0; i < length; i++)
        if (values[i] == target)
            return true;
    return false;
}

Dynamic Memory and Dynamic Arrays: Good Practices

  • Match new[] with delete[].
  • Preserve an owning pointer to the allocation’s base address.
  • Keep length beside the pointer and release ownership on every exit path.
  • Prefer automatic ownership and standard containers in application code.

Dynamic Memory and Dynamic Arrays: Common Mistakes

  • Mixing scalar and array release forms
  • Losing the only owning pointer
  • Accessing outside 0 through length - 1
  • Updating length before ownership transfer is complete

Dynamic Memory and Dynamic Arrays: More Examples

int* clone(const int source[], int length)
{
    if (length == 0)
        return nullptr;

    int* copy = new int[length];
    for (int i = 0; i < length; i++)
        copy[i] = source[i];
    return copy;
}

The caller receives ownership of the returned allocation.
