Stuff To Do With A Loop

What Can A Loop Do?

  • A loop repeats the same kind of work.
  • The useful question is what changes each time.

Slides 3-17

Loop Pattern Thinking

Design Question Why It Matters
What repeats? Names the work inside the loop
What changes? Identifies the control variable or state
What stops it? Defines the loop condition

Counters

  • A counter records how many times something has happened.
  • It usually changes by 1 each time the loop body runs.
  • Counters answer “how many?”

Counting Loop Passes

#include <iostream>
using namespace std;

int main() {
    int count = 0;

    while (count < 5) {
        cout << "Loop pass " << count + 1 << endl;
        count++;
    }

    return 0;
}

Running Totals

  • A running total accumulates values across loop passes.
  • Start the total at 0.
  • Add the new amount each time the loop processes another value.

Adding One Value At A Time

#include <iostream>
using namespace std;

int main() {
    int total = 0;

    for (int day = 1; day <= 3; day++) {
        int sales;
        cout << "Sales for day " << day << ": ";
        cin >> sales;
        total += sales;
    }

    cout << "Total sales: " << total << endl;
    return 0;
}

Sentinels

  • A sentinel is a special value that means “stop.”
  • It works when the program does not know ahead of time how much data will arrive.
  • The sentinel is not processed as regular data.

Stopping On A Signal

#include <iostream>
using namespace std;

int main() {
    int score;
    int total = 0;

    cout << "Enter a score or -1 to stop: ";
    cin >> score;

    while (score != -1) {
        total += score;
        cout << "Enter a score or -1 to stop: ";
        cin >> score;
    }

    cout << "Total: " << total << endl;
    return 0;
}

Flag-Controlled Loops

  • A flag is a bool variable that records whether a condition has happened.
  • The loop continues while the flag says more work is needed.
  • Flags are useful when the stopping reason is a state change.

Finding Something

#include <iostream>
using namespace std;

int main() {
    int target = 17;
    int value;
    bool found = false;

    while (!found && cin >> value) {
        if (value == target) {
            found = true;
        }
    }

    cout << boolalpha << found << endl;
    return 0;
}

Tracking The Best Value

  • A max or min tracker remembers the best value seen so far.
  • Initialize it with the first real value when possible.
  • Compare each new value to the current best.

Max Tracker Shape

#include <iostream>
using namespace std;

int main() {
    int highest;
    int value;

    cin >> highest;

    for (int count = 2; count <= 5; count++) {
        cin >> value;
        if (value > highest) {
            highest = value;
        }
    }

    cout << "Highest: " << highest << endl;
    return 0;
}

Building Output Repeatedly

  • A loop can create visible structure by printing one piece at a time.
  • Repeated symbols can become a line, a bar, or a table.
  • This is the bridge from loop mechanics to terminal graphs.

Summary

  • Counters count passes.
  • Running totals accumulate values.
  • Sentinels, flags, searches, trackers, and output builders solve different loop problems.

Drawing Graphs Using Loops

Can A Loop Draw?

  • A loop can draw by repeating output.
  • Each pass prints one symbol, one row, or one labeled value.

Slides 18-31

What Should The Graph Show?

  • A graph should make a comparison easier to see.
  • Decide what the viewer should compare before choosing labels or scale.
  • The loop prints the pattern; the design decides the meaning.

One Symbol At A Time

#include <iostream>
using namespace std;

int main() {
    for (int count = 1; count <= 10; count++) {
        cout << '*';
    }

    cout << endl;
    return 0;
}

Drawing One Bar

int value = 7;

for (int mark = 1; mark <= value; mark++) {
    cout << '#';
}

cout << endl;

[Graphic suggestion: Show the number 7 turning into seven repeated marks.]

Adding A Label

cout << "Monday    ";

for (int mark = 1; mark <= 7; mark++) {
    cout << '#';
}

cout << endl;

The label tells the viewer what the repeated symbols represent.

Graph From Data

int sales;

cout << "Enter sales for Monday: ";
cin >> sales;

cout << "Monday    ";
for (int mark = 1; mark <= sales; mark++) {
    cout << '#';
}
cout << endl;

Graph From A Formula

int speed = 40;
int totalHours = 3;

for (int hour = 1; hour <= totalHours; hour++) {
    int distance = speed * hour;
    cout << hour << " hr ";

    for (int mark = 1; mark <= distance / 10; mark++) {
        cout << '#';
    }

    cout << " " << distance << " miles" << endl;
}

Collect, Then Display

vector<int> sales(3);

for (int day = 0; day < sales.size(); day++) {
    cout << "Sales for day " << day + 1 << ": ";
    cin >> sales[day];
}

cout << "\nSales\n";
for (int day = 0; day < sales.size(); day++) {
    cout << "Day " << day + 1 << " ";
    for (int mark = 1; mark <= sales[day]; mark++) {
        cout << '#';
    }
    cout << endl;
}

The first loop collects the complete data set. The second loop displays the results as one graph.

Scaling The Bar

  • Raw values may be too large to print one symbol per unit.
  • A scale factor lets one symbol represent several units.
  • Example: one # can represent 10 miles, dollars, or votes.

Graph Shape Comes From Loop Structure

Loop Role Graph Role
Outer loop Chooses the next row or category
Inner loop Prints repeated symbols for that row
Counter Controls length, position, or label

Nested Loops For Graphs

for (int row = 1; row <= 4; row++) {
    for (int col = 1; col <= row; col++) {
        cout << '*';
    }
    cout << endl;
}

The outer loop starts a new row. The inner loop prints the symbols in that row.

Readable Terminal Graphs

  • Keep labels aligned.
  • Use one symbol consistently.
  • Print the numeric value when the bar length is approximate.

Summary

  • Terminal graphs are repeated output with a purpose.
  • Choose the comparison first, then use loop structure to print it clearly.

Menu-Driven Programming

Why Menus Need Loops

  • A menu gives the client a set of choices.
  • A useful menu usually comes back after each action.

Slides 32-49

The Menu Cycle

Step Responsibility
Display Show choices clearly
Input Read the client’s choice
Dispatch Run the selected action
Repeat Return to the menu unless quitting

A Menu Needs A Quit Choice

  • One option should mean “stop the menu.”
  • That quit option works like a sentinel.
  • The program repeats while the choice is not the quit value.

The Basic Menu Shell

int choice;

do {
    cout << "1. Display graph" << endl;
    cout << "2. Clear data" << endl;
    cout << "0. Quit" << endl;
    cout << "Choice: ";
    cin >> choice;

    // Dispatch happens here.
} while (choice != 0);

Dispatching The Choice

  • Dispatch means connecting a menu choice to the code that handles it.
  • The menu reads the choice.
  • The dispatch logic decides what action runs.

Dispatch With if / else if

if (choice == 1) {
    cout << "Display graph" << endl;
}
else if (choice == 2) {
    cout << "Clear data" << endl;
}
else if (choice == 0) {
    cout << "Goodbye" << endl;
}
else {
    cout << "Unknown choice" << endl;
}

Dispatch With switch

switch (choice) {
case 1:
    cout << "Display graph" << endl;
    break;
case 2:
    cout << "Clear data" << endl;
    break;
case 0:
    cout << "Goodbye" << endl;
    break;
default:
    cout << "Unknown choice" << endl;
}

Numeric Menus

  • Numeric menus use choices such as 1, 2, 3, and 0.
  • They are easy to order and easy to use with switch.
  • They can become harder to remember as the menu grows.

Alphanumeric Menus

  • Alphanumeric menus use choices such as A, D, Q, or 1A.
  • Letters can make the action easier to remember.
  • Mixed choices can be useful, but they require clear prompts.

Choosing A Menu Style

Style Strong Fit
Numeric Short ordered menus
Letter Action names such as add, delete, quit
Alphanumeric Grouped commands or larger menus

Keeping Actions Separate

  • The menu should coordinate the program.
  • Action code belongs in functions when it grows beyond a few lines.
  • This keeps main readable.

Functions As Menu Actions

#include <iostream>
using namespace std;

void displayGraph();
void clearData();
void showSummary();

int main() {
    int choice;

    do {
        cout << "1. Display graph\n2. Clear data\n3. Summary\n0. Quit\n";
        cin >> choice;

        if (choice == 1) {
            displayGraph();
        }
        else if (choice == 2) {
            clearData();
        }
        else if (choice == 3) {
            showSummary();
        }
    } while (choice != 0);

    return 0;
}

Menu Action Definitions

void displayGraph() {
    cout << "Displaying graph..." << endl;
}

void clearData() {
    cout << "Clearing data..." << endl;
}

void showSummary() {
    cout << "Showing summary..." << endl;
}

Function names should read like the action the menu performs.

Menu State

  • Useful menus usually operate on data that lasts across choices.
  • The data may be totals, graph values, account balances, or settings.
  • The menu loop keeps the program alive while that state changes.

Menu-Driven Graph Program

Choice Action
1 Enter graph data
2 Display graph
3 Clear data
0 Quit

The menu controls when each part of the graph program runs.

Common Practices

  • Use clear labels for each option.
  • Use one quit choice.
  • Separate display, input, dispatch, and action code.

Summary

  • A menu is repeated program control.
  • The loop keeps interaction going; dispatch chooses the action.
  • Functions keep the menu from becoming the whole program.

Key Takeaways

  • Loop patterns name the problem before the syntax.
  • Terminal graphs are repeated output with labels, scale, and structure.
  • Menu-driven programs use loops to keep interaction going.

Appendix

Stuff To Do With A Loop: Good Practices

  • Name the purpose of the loop before writing it.
  • Initialize counters, totals, flags, and trackers before the loop.
  • Make the loop move toward stopping.

Stuff To Do With A Loop: Common Mistakes

  • Updating a counter in the wrong place can change how many times the loop runs.
  • Processing a sentinel value as real data changes the result.
  • Initializing a max or min tracker to an arbitrary value can hide real data.

Menu-Driven Programming: Good Practices

  • Use a clear quit option.
  • Keep the menu cycle easy to read.
  • Move larger actions into functions.

Menu-Driven Programming: Common Mistakes

  • Forgetting break in a switch can run the next case.
  • Mixing menu display and action code makes the program hard to change.
  • A menu with no quit path traps the client.

Menu-Driven Programming: More Examples

char choice;

do {
    cout << "A. Add\nD. Display\nQ. Quit\n";
    cin >> choice;

    switch (choice) {
    case 'A':
        cout << "Add" << endl;
        break;
    case 'D':
        cout << "Display" << endl;
        break;
    }
} while (choice != 'Q');
