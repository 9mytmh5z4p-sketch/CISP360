Modular Programming

Programs as Pieces

  • How can we make a program easier to understand before we read every line?
  • Modular programming breaks a program into named pieces.
  • Each piece handles one part of the larger job.

Slides 3-14

What Changes When Code Grows?

  • A short main() is easy to scan.
  • A long main() can hide the program’s structure inside many details.
  • Bugs become harder to isolate when all processing happens in one place.

One Job Per Piece

  • A module should have a clear job.
  • In beginner C++, our modules are usually functions.
  • A function name should describe the work the function performs.

Main as Coordinator

int main() {
    displayTitle();
    readScores();
    calculateAverage();
    displayResults();

    return 0;
}

main() should show the program’s outline before the implementation details.

Function-Centered Architecture

  • Structured C++ programs are often organized around functions.
  • Data is stored in variables.
  • Processing is grouped into functions that can call other functions.

Functions and the IPO Model

Input Processing Output
scores typed by user calculate total and average printed report

Functions do not change what the program can do. They reorganize where the processing is written.

Functions and the Execution Trace

Step Location What Happens
1 main() Program reaches a function call
2 function Processing happens inside the function
3 main() Execution returns to the next line

An execution trace now follows control into and out of functions.

Common Practices

  • Use verb-based function names, such as displayMenu() or calculateTotal().
  • Keep each function focused on one clear job.
  • Put function prototypes above main() and function definitions below main().

Summary

  • Modular programming breaks a program into named pieces.
  • Functions reorganize processing without changing the IPO model.
  • A clear main() helps the reader see the program’s structure first.

Coding Functions

Where Does a Function Live?

  • A function has a prototype, a call, and a definition.
  • The prototype appears above main().
  • The definition usually appears below main().

Slides 15-26 [Graphic suggestion: show a vertical code map with prototype near the top, main in the middle, and function definition below.]

Smallest Useful Function

#include <iostream>
using namespace std;

void displayMessage();      // Function Prototype

int main() {
    displayMessage();       // Function Call
    return 0;
}

void displayMessage() {     // Function Definition
    cout << "Hello from a function!" << endl;
}

Function Prototype

#include <iostream>
using namespace std;

// Function Prototypes
void displayMessage();
  • The prototype tells the compiler the function exists.
  • It gives the return type, function name, and parameter list.
  • The semicolon marks this as a declaration, not the function body.

Function Definition

return 0;
}       // main()

void displayMessage() {
    cout << "Hello from a function!" << endl;
}
  • The definition contains the function body.
  • This is the code that runs when the function is called.

Function Call

displayMessage();
  • The function call transfers control to the function.
  • When the function finishes, execution returns to the next line.

Function Call Overview

Function call overview

Call Order

int main() {
    cout << "Before" << endl;
    displayMessage();
    cout << "After" << endl;
    return 0;
}
  • Before prints first.
  • The function runs next.
  • After prints when control returns to main().

Functions Calling Functions

void deeper();
void deep();

void deep() {
    cout << "Inside deep()" << endl;
    deeper();
}

void deeper() {
    cout << "Inside deeper()" << endl;
}

A function can call another function.

Multiple Function Calls on the Stack

Call stack
main()
deep()
deeper()

main() starts the call chain.

[Graphic suggestion: emphasize last-in, first-out order. The most recent function call returns first.]

No Nested Function Definitions

void outer() {
    void inner() {          // not allowed in C++
        cout << "Nope";
    }
}
  • C++ functions can call other functions.
  • C++ functions cannot be defined inside other functions.

Summary

  • A prototype declares the function before it is called.
  • A call runs the function.
  • A definition contains the function body.

Passing Data to a Function

Functions Need Inputs

  • A fixed function works only for one situation.
  • A flexible function receives data from the caller.
  • Parameters let one function work with different values.

Slides 27-38

Parameters Receive Data

void greet(string person) {
    cout << "Hello, " << person << "!" << endl;
}
  • person is a parameter.
  • It is a variable that belongs to the function.
  • It receives data when the function is called.

Arguments Provide Data

greet("Maya");
  • "Maya" is an argument.
  • An argument is the actual value supplied by the caller.
  • The argument is copied into the parameter.

Arguments vs Parameters

Arguments vs parameters

Argument to Parameter

void greet(string person) {
    cout << "Hello, " << person << "!" << endl;
}

int main() {
    greet("Maya");
    return 0;
}

The call sends "Maya" into the function, where person stores the copy.

One Parameter

#include <iostream>
#include <string>
using namespace std;

void greet(string person);

int main() {
    greet("Maya");
    greet("Ari");
    greet("Chen");
    return 0;
}

void greet(string person) {
    cout << "Welcome, " << person << "!" << endl;
}

One function can handle many different values.

Multiple Parameters

#include <iostream>
#include <string>
using namespace std;

void displayProduct(string name, double price);

int main() {
    displayProduct("Notebook", 4.99);
    displayProduct("Backpack", 39.95);
    return 0;
}

void displayProduct(string name, double price) {
    cout << name << " costs $" << price << endl;
}

Separate parameters with commas.

Order Matters

void displayStudent(string name, int age) {
    cout << name << " is " << age << endl;
}

displayStudent("Maya", 20);  // correct
displayStudent(20, "Maya");  // wrong order

Arguments match parameters by position.

Types Matter

void displayStudent(string name, int age, double gpa);
  • Each parameter needs its own type.
  • The argument type must be compatible with the parameter type.
  • The compiler checks function calls before the program runs.

Parameters Are Local

void printDouble(int number) {
    cout << number * 2 << endl;
}

int main() {
    printDouble(7);
    cout << number << endl;  // error
    return 0;
}

number exists only inside printDouble().

Summary

  • Parameters receive data inside a function.
  • Arguments provide data at the function call.
  • Order and type determine how values match up.

Returning Data from a Function

Functions Can Send Data Back

  • Parameters move data into a function.
  • A return value sends one result back to the caller.
  • This completes the basic data-flow pattern for functions.

Slides 39-48

Return Type

int add(int first, int second) {
    return first + second;
}
  • int is the return type.
  • It tells C++ what kind of value comes back.

The Return Statement

return first + second;
  • return sends a value back to the caller.
  • The function ends when the return statement runs.
  • The returned value replaces the function call.

Store the Returned Value

#include <iostream>
using namespace std;

int add(int first, int second);

int main() {
    int total = add(3, 4);
    cout << total << endl;
    return 0;
}

int add(int first, int second) {
    return first + second;
}

The variable total stores the value returned by add().

Use the Returned Value Directly

cout << add(3, 4) << endl;

int result = add(10, add(2, 3));
  • A returned value can be printed directly.
  • A returned value can be used inside a larger expression.

Return Type Must Match

int getWholeNumber() {
    return 42;
}

string getName() {
    return "Maya";
}

The value after return must be compatible with the function’s return type.

One Value Comes Back

  • A normal return statement sends one value back.
  • That value may be stored, printed, or used in another expression.
  • Later tools give us other ways to communicate more data.

Returned Data on the Stack

Call stack
main()
result = ?
add(3, 4)
first = 3
second = 4

main() calls add(3, 4).

[Graphic suggestion: show the return value leaving the function frame before the frame disappears.]

Summary

  • A return value lets a function compute an answer.
  • The return type tells C++ what kind of answer comes back.
  • The caller decides what to do with the returned value.

Void Functions

What If Nothing Comes Back?

  • Some functions calculate an answer.
  • Other functions perform an action.
  • void functions are for action-only work.

Slides 49-57

Void Means No Return Value

void displayLine() {
    cout << "--------------------" << endl;
}
  • void means the function does not send a value back.
  • The function can still do useful work.

A Void Function

#include <iostream>
using namespace std;

void displayWelcome();

int main() {
    displayWelcome();
    return 0;
}

void displayWelcome() {
    cout << "Grade Calculator" << endl;
    cout << "Ready to begin." << endl;
}

The visible result is the action the function performs.

Call It Like Any Other Function

displayWelcome();
  • A void function is called by name with parentheses.
  • Do not store a void function call in a variable.
  • There is no returned value to store.

When Void Makes Sense

  • Display a header.
  • Show a menu.
  • Print a divider line.
  • Pause or clear part of the interface.

Use void when the caller needs an action, not an answer.

Return in a Void Function

void checkAge(int age) {
    if (age < 18) {
        cout << "Access denied." << endl;
        return;
    }

    cout << "Access granted." << endl;
}

return; exits a void function early, but it does not return a value.

Void or Return Value?

Use This When
return value the caller needs an answer
void the function performs an action

Choose based on what the caller needs after the function runs.

Summary

  • void functions do not return values.
  • They still help modularize a program.
  • Their result is the action they perform.

Key Takeaways

  • Modular programming organizes a program into named pieces.
  • Functions reorganize processing without changing the IPO model.
  • A function prototype tells the compiler a function exists before it is called.
  • Parameters receive data from arguments.
  • Return values send one answer back to the caller.
  • void functions perform actions without returning a value.
