Introduction to Pointers and Their Value in Modern Programming

When Does a Name Stop Being Enough?

  • A variable name identifies one object
  • What if the program must choose the object while it is running?

Slides 3–13 Ask students to imagine one operation that might update either of two counters. The operation is fixed, but its target is not.

From Symbol Table to Runtime Data

Source code Compiler knowledge Runtime storage
score name, type, scope, address 88
  • A name lets the compiler locate an object
  • A pointer makes an object’s address available to the running program

What Is a Pointer?

  • A pointer is a typed variable whose value is a memory address
  • The object at that address is the pointer’s target or pointee
  • Using the address to reach the target is indirection

The Pointer Model

Pointer variable Stored value Target variable
selected address of score score = 88

selected → address → score

  • The pointer has its own storage, separate from its target

[Graphic suggestion: draw two memory cells. The selected cell contains an arrow-shaped address that leads to the score cell.]

Why Store an Address?

  • Select an existing object while the program runs
  • Read or modify that object indirectly
  • Describe positions and ranges in contiguous data
  • Work with libraries and resources whose locations are provided at runtime

One Operation, Different Targets

Before selection Runtime choice After operation
regular = 40 target regular regular = 35
sale = 28 target sale sale = 23
  • The operation stays the same while the selected target changes

Keep this example conceptual. Topic 2 supplies the C++ syntax and execution trace.

Where Do Pointers Fit in Modern C++?

  • Raw pointers commonly express non-owning access or library interoperability
  • Smart pointers express ownership of dynamically allocated resources
  • Containers such as vector usually replace manually managed arrays
  • Pointers remain foundational to arrays, iterators, dynamic memory, and polymorphism

Pointers Are Not the Default

Need Prefer
Store a value ordinary variable
Require another name for an existing object reference
Represent address semantics or no target pointer
  • Use the highest-level tool that clearly expresses the design

Capability Comes With Responsibility

A pointer may be used only when it identifies:

  • An object of the expected type
  • An object whose lifetime has not ended
  • A location within the operation’s permitted boundary

Introduction to Pointers: Summary

  • Pointers store typed addresses and provide indirect access to objects
  • Address semantics enable runtime flexibility but add validity and lifetime responsibilities
  • Modern C++ uses raw pointers selectively and prefers higher-level ownership and container tools

Initializing and Using Pointers

How Does a Pointer Reach a Variable?

  • The target already has an address
  • How do we obtain that address and store it safely?

Slides 14–33

The Address Operator

int score = 88;

cout << &score << "\n";
  • Unary & produces the address of an object
  • Printed addresses are implementation-controlled identifiers, not meaningful numbers to interpret

Declaring a Pointer Variable

int* scorePtr;
Part Meaning
int target type
* pointer declarator
scorePtr pointer variable name

Match the Pointer to the Target

int score = 88;
double average = 91.5;

int* scorePtr = &score;
// int* wrongPtr = &average; // incompatible target type

Declare One Pointer at a Time

int* first, second;
  • first is an int pointer
  • second is an ordinary int
int* first;
int* second;

Initialize With an Address

int score = 88;
int* scorePtr = &score;
Expression Meaning
&score address of score
scorePtr address stored in the pointer

[Graphic suggestion: show separate cells for score and scorePtr, then draw the stored address as an arrow from scorePtr to score.]

Initialize With nullptr

int* selected = nullptr;
  • nullptr deliberately represents no target
  • Use it when a valid target is not yet available
  • In modern C++, prefer nullptr over NULL or 0

Valid, Null, or Indeterminate?

Pointer state Meaning Dereference?
valid address identifies a live target yes
nullptr deliberately identifies no target no
uninitialized indeterminate value no
  • Reading or dereferencing an uninitialized pointer has undefined behavior

What Does Dereferencing Mean?

int score = 88;
int* scorePtr = &score;

cout << *scorePtr << "\n";
  • Unary * follows the stored address to the target
  • Here, * is dereference—not multiplication or declaration syntax

Four Related Expressions

int value = 88;
int* ptr = &value;
Expression Result
value target value
&value target address
ptr stored address
*ptr value reached through ptr

Reading Through a Pointer

int score = 88;
int* scorePtr = &score;

cout << *scorePtr << "\n";

scorePtr → address of score → 88

Trace three distinct facts: the pointer’s stored address, the target’s value, and whether dereferencing is valid.

Writing Through a Pointer

int score = 88;
int* scorePtr = &score;

*scorePtr = 95;
cout << score << "\n";
  • score changes from 88 to 95
  • scorePtr continues to store the same address

Predict the State

int points = 10;
int* ptr = &points;

*ptr = 14;
points += 3;
  • What is points?
  • What is *ptr?
  • Has the address stored in ptr changed?

Answer: points and *ptr are both 17. ptr still stores the address of points.

Changing the Target

int first = 10;
int second = 20;
int* ptr = &first;

ptr = &second;
  • ptr = &second; changes the target
  • *ptr = 25; would change the current target’s value

One Operation, Runtime Choice

int regularPrice = 40;
int salePrice = 28;
bool saleIsActive = true;

int* selected = &regularPrice;
if (saleIsActive) {
    selected = &salePrice;
}

*selected -= 5;

Check Before Dereferencing

int* selected = nullptr;

if (selected != nullptr) {
    cout << *selected << "\n";
}
  • A null check prevents dereferencing the no-target state
  • Non-null is necessary, but the target must also still be alive

When a Target Disappears

int* ptr = nullptr;

{
    int temporary = 42;
    ptr = &temporary;
} // temporary's lifetime ends
  • ptr is now dangling: it contains an address but has no live target
  • A null pointer and a dangling pointer are different invalid states

A Complete Pointer Lifecycle

int first = 10;
int second = 20;
int* selected = &first;

*selected += 5;
selected = &second;

if (selected != nullptr) {
    *selected += 5;
}

selected = nullptr;

Trace: selected starts at first; first becomes 15; selected is reseated to second; second becomes 25; selected is reset to the deliberate no-target state.

Initializing and Using Pointers: Summary

  • Initialize every pointer with a valid address or nullptr
  • Dereference only when the pointer identifies a compatible, live target
  • Distinguish changing a pointer’s target from changing the target’s value

Key Takeaways

  • A pointer is a typed variable that stores an address
  • Dereferencing follows that address to read or modify a live target
  • Initialize pointers with a valid address or nullptr
  • Modern C++ uses raw pointers selectively and prefers higher-level tools when possible
