Why Pointers?

What Can an Address Do?

  • A variable gives a value a name
  • A pointer makes the value’s location usable data

Addresses Can Cross Boundaries

  • An address can be stored for later access
  • The same address can describe an array position or cross a function boundary

Why Add Indirection?

  • Select an object while the program runs
  • Modify an existing object through another access path
  • Traverse contiguous data without copying it
  • Prepare for interfaces and memory whose location is known only at runtime

Pointers Extend the Memory Model

name → address → pointer → object

  • A pointer does not replace the object
  • It makes the object’s location programmable

Have We Already Used Pointers?

Every Element Has an Address

int values[4] = {12, 18, 25, 31};

for (int index = 0; index < 4; index++) {

An Array Name Identifies Its First Element

int values[4] = {12, 18, 25, 31};

Store the First Element’s Address

int values[4] = {12, 18, 25, 31};
int* ptr = values;

for (int index = 0; index < 4; index++) {

Pointers Can Use Subscripts

int* ptr = values;

Two Ways to Read an Array

    cout << value << " ";

Start in the Middle

int values[5] = {10, 20, 30, 40, 50};
int* ptr = &values[2];

    cout << ptr[offset] << "\n";
}

The Pointer Can Walk; the Array Name Cannot

int values[3] = {7, 8, 9};

ptr++;
cout << *ptr << "\n"; // 8

// values++; // compiler error

Arrays Become Pointers at Function Boundaries

void display(const int values[], int size);
void display(const int* values, int size);
  • For a parameter, these declarations describe the same interface
  • The function receives the first-element address, not the whole array object

The Function Still Needs the Size

#include <iostream>
using namespace std;

int sum(const int values[], int size);

int main() {
    int values[4] = {2, 4, 6, 8};
    cout << sum(values, 4) << "\n";
    return 0;
}

int sum(const int values[], int size) {
    int total = 0;
    for (int i = 0; i < size; i++) total += values[i];
    return total;
}

No Copy Required

int values[6] = {5, 10, 15, 20, 25, 30};
int* beginning = &values[2];
int length = 3;
int total = 0;

for (int i = 0; i < length; i++) {
    total += beginning[i];
}

cout << total << "\n"; // 60

An Array and a Pointer Are Related, Not Identical

int values[5] = {};

cout << sizeof(values) << "\n";
cout << sizeof(ptr) << "\n";
  • The array owns storage for five integers
  • The pointer stores one address and can be reseated

Pointers Explain Familiar Array Behavior

  • Array names usually convert to first-element pointers
  • Subscripting performs indirect access with an offset
  • An address plus a length can describe any contiguous subrange

Pointer Arithmetic and Comparison

How Does a Pointer Walk Through Data?

  • Pointer arithmetic moves between elements of one array
  • The pointed-to type determines each step

Slides 61–78

One Step Means One Element

int values[3] = {10, 20, 30};
int* ptr = values;

cout << ptr << "\n";
cout << ptr + 1 << "\n";
cout << sizeof(*ptr) << "\n";
  • ptr + 1 advances by one int, not one byte

Different Types Take Different Steps

char letters[2] = {'A', 'B'};
int counts[2] = {10, 20};
double rates[2] = {1.5, 2.5};

cout << static_cast<void*>(letters) << " "
     << static_cast<void*>(letters + 1) << "\n";
cout << counts << " " << counts + 1 << "\n";
cout << rates << " " << rates + 1 << "\n";

Adding an Offset

int values[5] = {10, 20, 30, 40, 50};
int* ptr = values;

cout << *(ptr + 0) << "\n";
cout << *(ptr + 2) << "\n";
cout << *(ptr + 4) << "\n";

Moving Several Elements

int values[5] = {10, 20, 30, 40, 50};
int* ptr = values;

ptr += 3;
cout << *ptr << "\n"; // 40

ptr -= 2;
cout << *ptr << "\n"; // 20

Incrementing a Pointer

int values[4] = {12, 18, 25, 31};

for (int* ptr = values; ptr != values + 4; ptr++) {
    cout << *ptr << "\n";
}

Moving Backward

int values[4] = {12, 18, 25, 31};
int* ptr = &values[3];

for (int count = 0; count < 4; count++) {
    cout << *ptr << "\n";
    ptr--;
}

The One-Past-the-End Address

int values[4] = {12, 18, 25, 31};
int* begin = values;
int* end = values + 4;

cout << begin << "\n";
cout << end << "\n";
  • end is valid for comparison
  • No array element exists at end

Begin and End Define a Range

[begin, end)

  • begin identifies the first included element
  • end identifies the first excluded position
  • An empty range has begin == end

Traverse With Pointer Sentinels

int values[4] = {12, 18, 25, 31};
int* begin = values;
int* end = values + 4;

for (int* current = begin; current != end; current++) {
    cout << *current << "\n";
}

Compare Positions Within One Array

int values[5] = {};
int* first = &values[1];
int* second = &values[3];

cout << (first == second) << "\n";
cout << (first < second) << "\n";
cout << (second > first) << "\n";

Subtract Pointers to Measure Distance

int values[8] = {};
int* first = &values[2];
int* last = &values[7];

cout << last - first << "\n"; // 5 elements
  • The result counts elements, not bytes

Search With Pointers

int values[5] = {12, 18, 25, 31, 44};
int target = 25;
int* found = nullptr;

for (int* ptr = values; ptr != values + 5; ptr++) {
    if (*ptr == target) {
        found = ptr;
        break;
    }
}

Search Results Preserve Position

if (found != nullptr) {
    cout << "value: " << *found << "\n";
    cout << "index: " << found - values << "\n";
} else {
    cout << "not found\n";
}

Process a Subrange

int values[6] = {5, 10, 15, 20, 25, 30};
int* begin = &values[1];
int* end = &values[5];
int total = 0;

for (int* ptr = begin; ptr != end; ptr++) {
    total += *ptr;
}
cout << total << "\n"; // 70

Index Loops and Pointer Loops Express the Same Walk

for (int i = 0; i < 5; i++) {
    cout << values[i] << "\n";
}
for (int* p = values; p != values + 5; p++) {
    cout << *p << "\n";
}

Pointer Arithmetic Builds Ranges

  • Arithmetic scales by the pointed-to type
  • Begin and end pointers describe a half-open range
  • Arithmetic, ordering, and subtraction belong within one array

Pointer Safety

When Is a Pointer Safe to Use?

  • The pointer is initialized
  • Its target is alive and its position is in range

Slides 79–97

A Pointer Has Three Useful States

int value = 42;
int values[3] = {1, 2, 3};

int* valid = &value;
int* empty = nullptr;
int* end = values + 3;
  • Valid addresses may be dereferenced
  • Null and end-sentinel states must not be dereferenced

Initialize Every Pointer

int score = 88;

int* selected = &score;
int* waiting = nullptr;
  • Initialization gives the pointer a state that the program can reason about

Uninitialized Is Not Empty

int* unknown;           // indeterminate value
int* empty = nullptr;   // deliberate no-object state
  • Do not read or dereference an uninitialized pointer
  • Use nullptr when no valid target exists yet

Check Before Dereferencing

int score = 88;
int* ptr = &score;

if (ptr != nullptr) {
    cout << *ptr << "\n";
} else {
    cout << "No score selected\n";
}

nullptr Means No Object

  • nullptr is an intentional absence of an address
  • It can be assigned, compared, and returned by later interfaces
  • It cannot be dereferenced

Validity Depends on Lifetime

int* ptr = nullptr;

{
    int temporary = 42;
    ptr = &temporary;
    cout << *ptr << "\n";
}

// temporary no longer exists here

Addresses Can Outlive Their Objects

int* ptr = nullptr;

{
    int temporary = 42;
    ptr = &temporary;
    cout << ptr << "\n";
}

cout << ptr << "\n"; // old address remains
  • The address value may remain even though the object is gone

Dangling Pointers Do Not Reset Themselves

int* ptr = nullptr;
{
    int temporary = 42;
    ptr = &temporary;
}

ptr = nullptr; // explicitly record no valid target

Non-Null Does Not Mean Valid

Check What it establishes
ptr != nullptr an address is present
target is alive the object still exists
begin <= ptr && ptr < end the position is dereferenceable
  • Pointer validity is a contract, not one Boolean test

A Dangling Pointer Still Looks Like an Address

cout << ptr << "\n";
  • Printing an address cannot prove that its target exists
  • Only the program’s ownership, scope, and lifetime logic can establish validity

Array Pointers Need Bounds

int values[4] = {2, 4, 6, 8};
int* begin = values;
int* end = values + 4;

for (int* ptr = begin; ptr != end; ptr++) {
    cout << *ptr << "\n";
}

One Past the End Is for Comparison Only

int values[4] = {2, 4, 6, 8};
int* end = values + 4;

if (end == values + 4) {
    cout << "range boundary\n";
}

// cout << *end; // no element exists here

Container Changes Can Invalidate Addresses

vector<int> values = {10, 20, 30};
int* before = values.data();

values.push_back(40);
int* after = values.data();

cout << before << "\n";
cout << after << "\n";
  • Growth may relocate a vector’s elements

Reacquire Addresses After Vector Growth

vector<int> values = {10, 20, 30};

values.push_back(40);
int* begin = values.data();
int* end = begin + values.size();

for (int* ptr = begin; ptr != end; ptr++) {
    cout << *ptr << "\n";
}

Keep the Owner Alive

  • A pointer provides access; it does not extend an object’s lifetime
  • Local variables live until their scope ends
  • Array and container elements live only while their owner and storage remain valid

Use a Pointer Only When Its Contract Holds

  1. The pointer was initialized
  2. The pointer is not nullptr
  3. The target still exists
  4. The address identifies an element inside the permitted range

Pointer Safety Is a Reasoning Discipline

  • Track state, lifetime, and bounds together
  • A visible address is not evidence of a valid target

Choosing Direct Access, References, or Pointers

Do We Need a Pointer Here?

  • Use the simplest access model that expresses the program’s contract
  • Choose a pointer only when address-level capability adds value

Slides 98–120

Direct Access Is the Simplest Choice

double price = 40.0;
price *= 0.80;
cout << price << "\n";
  • Use the variable name when the code already knows the object

References Express a Required Alias

double price = 40.0;
double& selected = price;

selected *= 0.80;
cout << price << "\n";
  • A reference must bind to an object and cannot be reseated

Pointers Express Optional or Changeable Access

double price = 40.0;
double* selected = nullptr;

selected = &price;
if (selected != nullptr) {
    *selected *= 0.80;
}

The Same Problem With Direct Access

double regularPrice = 40.0;
double salePrice = 28.0;
bool discountSalePrice = true;

if (discountSalePrice) {
    salePrice *= 0.80;
} else {
    regularPrice *= 0.80;
}

The Same Problem With a Reference

double regularPrice = 40.0;
double salePrice = 28.0;
bool discountSalePrice = true;

double& selected = discountSalePrice
    ? salePrice
    : regularPrice;

selected *= 0.80;
  • The target is required and selected once

The Same Problem With a Pointer

double regularPrice = 40.0;
double salePrice = 28.0;
bool discountSalePrice = true;

double* selected = &regularPrice;
if (discountSalePrice) selected = &salePrice;

*selected *= 0.80;
  • The target can be changed later or represented as absent

Choose at Runtime

int morningTotal = 120;
int eveningTotal = 95;
char shift = 'E';

int* activeTotal = &morningTotal;
if (shift == 'E') activeTotal = &eveningTotal;

*activeTotal += 10;

Represent No Selection

int first = 10;
int second = 20;
int* selected = nullptr;

int choice = 2;
if (choice == 1) selected = &first;
if (choice == 2) selected = &second;

if (selected != nullptr) cout << *selected;

Walk Through Contiguous Data

int values[4] = {5, 10, 15, 20};

for (int* ptr = values; ptr != values + 4; ptr++) {
    *ptr += 1;
}
  • References do not support this reseatable traversal model

Describe a Subrange Without Copying

int values[6] = {5, 10, 15, 20, 25, 30};
int* begin = &values[2];
int* end = &values[5];

for (int* ptr = begin; ptr != end; ptr++) {
    cout << *ptr << "\n";
}

Choose an Index When Position Matters

int values[5] = {8, 3, 9, 4, 7};
int largestIndex = 0;

for (int i = 1; i < 5; i++) {
    if (values[i] > values[largestIndex]) {
        largestIndex = i;
    }
}
  • An index directly communicates a numeric position

Pointers Do Not Automatically Mean Ownership

int score = 88;
int* observer = &score;
  • score owns its storage
  • observer provides another access path
  • Dynamic-memory ownership arrives in a later lecture

Capability Has a Cost

Pointer capability Required reasoning
May be absent null state
May be reseated current target
May traverse valid bounds
May outlive a target lifetime

Pointers Are Not Automatically Faster

for (int i = 0; i < 5; i++) {
    total += values[i];
}
for (int* p = values; p != values + 5; p++) {
    total += *p;
}
  • Choose the form that communicates the algorithm clearly

Prefer the Highest-Level Tool That Fits

  • Use vector when the size changes at runtime
  • Use std::array for modern fixed-size storage
  • Use range-based loops when every element receives the same treatment
  • Use raw pointers when the address itself is part of the problem

Pointers Connect to Lower-Level Interfaces

array<int, 4> fixed = {1, 2, 3, 4};
vector<int> growing = {5, 6, 7, 8};
char word[] = "pointer";

int* fixedData = fixed.data();
int* vectorData = growing.data();
char* textData = word;

One Object, Three Access Forms

int value = 10;
int& alias = value;
int* pointer = &value;

value++;
alias++;
(*pointer)++;

cout << value << "\n"; // 13

The Interface Should Reveal the Contract

Access form Target required? Reseatable? Arithmetic?
Direct variable Yes No No
Reference Yes No No
Pointer No Yes For array positions

A Practical Decision Framework

  1. Already know the object? Use direct access.
  2. Need a required alias? Use a reference.
  3. Need absence, reseating, or array-position movement? Consider a pointer.
  4. Prefer a container or range-based loop when it states the intent more clearly.

What Pointers Add

  • Runtime selection of an object
  • Indirect reading and modification
  • Traversal and subranges over contiguous data
  • A bridge to lower-level interfaces and future dynamic memory

Pointers Make Addresses Programmable

  • The value is not the pointer; its location is
  • Use that extra level only when the program benefits from it

Key Takeaways

  • A pointer stores a typed address and dereferencing accesses the object there
  • Pointers add runtime selection, indirect modification, and traversal of contiguous data
  • C-style array indexing is built on pointer arithmetic
  • Valid use requires an initialized pointer, a live target, and a position inside its permitted range
  • Prefer direct access, references, and containers when address-level capability is unnecessary
