Common Elements

What every program uses

All programs use these elements

  • Identifiers
  • Reserved Words
  • Literals
  • Comments

Identifiers

These are words that you get to name. C++ doesn’t care what they’re called.

Example: age, myArray, MakeMoney(), GoHome(), yo

Reserved Words

These are words you do not get to name. C++ very much cares about these words because they mean something to the compiler.

Example: int, main(), #include, if, for

Why Should You Care

If you mix up and use a reserved word where you should have used an identifier

Or

You use a an identifier where you should’ve used a reserved word, your program will not compile or run.

Literals

// Examples of literals

cout << "Run, Spot, Run" << endl;

// Magic Number
myTax = salary * myRate + 8345.11;
  • this is data which is hardcoded into your program.
  • this is not under program control.
  • C++ allows you to get around that.

Comments


// This is a single-line comment

/*
   This is a multi-line comment
   See, another line
*/
  • Comments are ignored by the compiler
  • Used to explain intent
  • Used to temporarily disable code (debugging)

Write Comments

  • Use them to help your future self understand what you are doing.
  • We have plenty of memory these days, use it.
  • The best comments expose our intent.

Useful Comments


// Helpful
float age;   // store months after decimal, that's why not an int

// Unhelpful
float age;   // this is a float (no kidding)

Whole Line vs End-of-Line Comments


// Whole line comment because it uses an entire line

int age;  // End-of-line comment, use only with variables

Whole line comments are preferred because code frequently changes and it’s easy to move this kind of comment around.

Top Matter

The #include Directive

The #include directive tells the compiler to include the contents of another file before compilation begins.

It is handled by the preprocessor, not by the C++ compiler itself.

Why #include Exists

Programs are built from multiple pieces of code.

#include allows you to:

  • Reuse existing code
  • Access library functionality
  • Keep programs modular and readable

Angle Brackets vs Quotes

// Standard Library Header
#include <iostream>

// User Defined or old C libraries
#include "myheader.h"
  • Angle brackets < >
    • Used for standard library headers
  • Double quotes " "
    • Used for user-defined headers

Yes, you can create your own libraries. No, we won’t do it in here.

iostream

#include <iostream>
  • Provides input/output facilities
  • Enables:
    • cout
    • cin
    • endl
  • Required for basic terminal interaction

What Actually Gets Included?

Conceptually, #include works like a copy-and-paste.

When the preprocessor runs:

  • The contents of the header file are inserted
  • The compiler then compiles the expanded source

Preprocessing Happens First

Compilation occurs in stages:

  1. Preprocessing
  2. Compilation
  3. Linking

#include is resolved entirely during preprocessing.

Headers Do Not Contain Executable Code

Header files typically contain:

  • Function declarations
  • Class declarations
  • Constants

They usually do not contain:

  • main
  • Executable statements

But, the Preprocessor is its Own Language

  • You can program with it.
  • Preprocessor errors cannot be detected by the C++ compiler!
  • You can introduce hard to solve errors into your program.
  • Don’t include any other directives than #include.

I’ve heard about other libraries

Yes, historically, there were other libraries.

  • The C Standard Library
  • The C++ Standard Library
  • The Standard Template Library
  • The Boost Library
    We don’t need to worry about that today. Many of the features of the standard template library have been included within the C++ standard library. The C standard library is not for C++. The boost library is still used, but not in this class. Citation

Why Headers Matter

Using headers correctly:

  • Reduces duplication
  • Improves organization
  • Enables large programs
  • Supports team development

Common Beginner Mistakes

  • Forgetting to include required headers
  • Including headers unnecessarily
  • Confusing headers with source files

Key Takeaways

  • #include is a preprocessor directive
  • It brings in external declarations
  • Angle brackets and quotes have different meanings
  • Headers enable modular programming

  • 0 means successful execution
  • Non-zero values indicate errors
