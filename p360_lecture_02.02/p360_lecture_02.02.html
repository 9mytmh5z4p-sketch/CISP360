cout

Output as a Conversation

  • How does a program speak clearly to the person using it?
  • cout sends text, values, and line breaks to the terminal.

Slides 3-12

The Output Stream

cout is the standard output stream.

Use the insertion operator << to send each piece of output into the stream.

cout comes from the <iostream> header.

#include <iostream>
using namespace std;

cout << "Hello, world!" << endl;

Text and Values

  • Text literals go inside double quotes.
  • Variable names are written without quotes.
  • cout prints the current value stored in the variable.
string name = "Maya";
int score = 92;

cout << "Name: " << name << endl;
cout << "Score: " << score << endl;

Chaining Output

Several output pieces can be chained in one statement.

int cats = 2;
int dogs = 3;

cout << "Cats: " << cats << ", Dogs: " << dogs << endl;

Each << sends the next item to the terminal from left to right.

Several cout Statements

Multiple cout statements can build one visible output box.

cout << "+---------------+" << endl;
cout << "| Sands         |" << endl;
cout << "| Total: $12.50 |" << endl;
cout << "+---------------+" << endl;
+---------------+
| Sands         |
| Total: $12.50 |
+---------------+

Line Breaks

  • endl ends the line and flushes the output stream.
  • "\n" ends the line without forcing a flush.
  • For normal printed output, either one can make the next output start on a new line.
cout << "First line" << endl;
cout << "Second line\n";

Complete Example

#include <iostream>
#include <string>
using namespace std;

int main() {
    string name = "Maya";
    int score = 92;

    cout << "Student Report" << endl;
    cout << "Name: " << name << endl;
    cout << "Score: " << score << endl;

    return 0;
}

Summary

  • cout sends output to the terminal.
  • Chain text, variables, and line breaks with <<.
  • Clear output makes a program easier to read and use.

Complete Example Output

Student Report
Name: Maya
Score: 92

cin

Input as a Conversation

  • How does a program listen to the person using it?
  • cin reads typed input from the terminal and stores it in variables.

Slides 13-23

The Input Stream

cin is the standard input stream.

Use the extraction operator >> to pull typed data into a variable.

cin comes from the <iostream> header.

#include <iostream>
using namespace std;

int age;

cin >> age;

Prompt Then Read

Most input follows a two-step pattern.

int age;

cout << "Enter your age: ";
cin >> age;

The prompt explains what the program expects before cin waits.

Matching Types

cin reads according to the receiving variable’s type.

int count;
double price;

cout << "How many items? ";
cin >> count;

cout << "Price per item? ";
cin >> price;

If the input cannot be converted, the read fails.

Reading Multiple Values

One statement can read several values in order.

int feet;
int inches;

cout << "Enter feet and inches: ";
cin >> feet >> inches;

The user can type both values on one line separated by whitespace.

Whitespace Behavior

cin >> skips leading whitespace and stops at the next whitespace.

string fullName;

cout << "Full name: ";
cin >> fullName;

If the user types Maya Chen, only Maya is stored. The space stops the read.

When cin Fails

If the user types data that does not match the variable type, cin enters a failed state.

int age;

cout << "Age: ";
cin >> age;   // typing "ten" fails

After a failed read, the variable is not updated with useful input.

Complete Example

#include <iostream>
#include <string>
using namespace std;

int main() {
    string fullName;
    int age = 0;

    cout << "Full name: ";
    cin >> fullName;

    cout << "Age: ";
    cin >> age;

    cout << fullName << " is " << age << " years old." << endl;
    return 0;
}

If the user types Maya Chen, fullName stores Maya and Chen remains waiting for the next read.

Summary

  • cin reads terminal input into variables.
  • Prompts should appear before reads.
  • cin >> treats whitespace as a separator.

Complete Example Output

Full name: Maya Chen
Age: Maya is 0 years old.

Chen is left in the input stream, so the age read fails when it expects a number.

Output Manipulators

Controlling the Shape of Output

  • How do we make numeric output readable instead of merely correct?
  • Manipulators let cout control spacing, alignment, and decimal precision.

Slides 24-38

The iomanip Header

Many output manipulators live in <iomanip>.

#include <iostream>
#include <iomanip>
using namespace std;

Use this header when output needs columns, alignment, or fixed decimal places.

setw

setw(n) sets the width for the next item only.

cout << setw(10) << "Item"
     << setw(8) << "Price" << endl;

The next value is padded with spaces if it is shorter than the requested width.

Alignment

left and right control how values sit inside a field.

cout << left  << setw(12) << "Pencils"
     << right << setw(8)  << 1.25 << endl;

Labels often read best left-aligned; numbers often read best right-aligned.

Manipulators for Today

  • setw(n) sets the width for the next printed item.
  • left and right control alignment inside a field.
  • fixed switches decimal output to fixed-point notation.
  • setprecision(n) controls the number of digits after the decimal when used with fixed.

Alignment Output

Periods show the spaces that the terminal normally leaves blank.

Pencils.........1.25

left pads after the label; right pads before the number.

fixed and setprecision

fixed prints floating-point values with a fixed number of digits after the decimal.

double total = 18.5;

cout << fixed << setprecision(2);
cout << "Total: $" << total << endl;

This prints money-like values with two decimal places.

fixed and setprecision Output

Sticky and One-Time Settings

  • setw(n) affects only the next printed item.
  • fixed, setprecision(n), left, and right stay active until changed.
cout << fixed << setprecision(2);
cout << setw(8) << 3.5 << endl;
cout << setw(8) << 12.75 << endl;

Sticky Settings Output

The width is applied separately to each value.

....3.50
...12.75

Periods show the spaces added by each setw(8) call.

Complete Example

#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    cout << fixed << setprecision(2);

    cout << left << setw(12) << "Item"
         << right << setw(8) << "Price" << endl;

    cout << left << setw(12) << "Notebook"
         << right << setw(8) << 4.99 << endl;

    return 0;
}

Complete Example Output

Item..........Price
Notebook......4.99

Periods show padding spaces. In the actual terminal, those positions are blank.

Summary

  • Use <iomanip> when output needs formatting.
  • setw creates columns; fixed and setprecision control decimals.
  • Formatting is part of program communication.

Other Useful Manipulators

  • setfill(ch) changes the padding character used by setw.
  • boolalpha prints Boolean values as true or false instead of 1 or 0.
  • showpoint forces a decimal point to appear for floating-point values.
  • scientific prints floating-point values in scientific notation.
  • hex, dec, and oct switch integer output between base 16, base 10, and base 8.

Terminal Color Codes

Output With Emphasis

  • How can a terminal program make important messages stand out?
  • ANSI escape codes can change text color in many modern terminals.

Slides 39-49

ANSI Escape Codes

An ANSI escape code is a short string that tells the terminal to change how text is displayed.

"\033[031m"

This code switches the following terminal text to red.

Color Code Pattern

Most simple foreground color codes follow this shape.

\033[COLORm
  • \033 is the escape character.
  • [ starts the control sequence.
  • COLOR is the numeric color code.
  • m ends the sequence.

Common Foreground Colors

  • \033[030m black
  • \033[031m red
  • \033[032m green
  • \033[033m yellow
  • \033[034m blue
  • \033[035m magenta
  • \033[036m cyan
  • \033[037m white

Always Reset

The reset code returns the terminal to normal output.

const string NORM = "\033[0m";

If you do not reset, later output may keep the previous color.

Store Colors as Constants

Named constants make color output readable.

const string RED = "\033[031m";
const string GREEN = "\033[032m";
const string YELLOW = "\033[033m";
const string BLUE = "\033[034m";
const string NORM = "\033[0m";

The name explains the purpose better than the number alone.

Meaningful Colors

Use color to reinforce meaning, not to decorate every line.

  • Red: error or important warning.
  • Yellow: caution.
  • Green: success.
  • Blue: information.

Complete Example

#include <iostream>
#include <string>
using namespace std;

int main() {
    const string RED = "\033[031m";
    const string GREEN = "\033[032m";
    const string YELLOW = "\033[033m";
    const string NORM = "\033[0m";

    cout << GREEN << "Success: saved file." << NORM << endl;
    cout << YELLOW << "Warning: low space." << NORM << endl;
    cout << RED << "Error: file missing." << NORM << endl;

    return 0;
}

Summary

  • ANSI escape codes can color terminal text.
  • Store codes as named constants.
  • Always reset with NORM after colored output.

Slides 50-58

What Makes a Table

  • A title tells the reader what the data represents.
  • Headings name each column.
  • Rows repeat the same column order.
  • Width and alignment keep the structure visible.

Choose Column Widths

Pick widths based on the longest expected item.

cout << left << setw(14) << "Item"
     << right << setw(8) << "Qty"
     << right << setw(10) << "Price" << endl;

Start with slightly generous widths and adjust after seeing real output.

Align by Data Type

  • Text columns usually read best left-aligned.
  • Numeric columns usually read best right-aligned.
  • Decimal values should use consistent precision.
cout << fixed << setprecision(2);

Header and Rows

The header and each row should use the same widths.

cout << left << setw(14) << "Item"
     << right << setw(8) << "Qty"
     << setw(10) << "Price" << endl;

cout << left << setw(14) << "Notebook"
     << right << setw(8) << 3
     << setw(10) << 4.99 << endl;

Separator Lines

A separator gives the eye a boundary between headings and data.

cout << "--------------------------------" << endl;

Keep separator length close to the total table width.

Complete Example

#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    cout << fixed << setprecision(2);

    cout << left << setw(14) << "Item"
         << right << setw(8) << "Qty"
         << setw(10) << "Price" << endl;
    cout << "--------------------------------" << endl;
    cout << left << setw(14) << "Notebook"
         << right << setw(8) << 3
         << setw(10) << 4.99 << endl;

    return 0;
}

Complete Example - Output

Summary

  • Tables are built from repeated output patterns.
  • Use consistent widths, alignment, and decimal formatting.
  • The goal is output a person can scan quickly.

Appendix

cout: Good Practices

  • Put prompts before reads and labels before values.
  • Use "\n" or endl intentionally; both create a new line, but endl also flushes.
  • Keep chained output readable by breaking long statements across lines.

cout: Common Mistakes

  • Putting variable names inside quotes and expecting their values to print.
  • Forgetting spaces inside string literals, causing output to run together.
  • Ending every prompt with endl when the input should appear on the same line.

cout: More Examples

cout << "Tax: $" << tax << "\n";
cout << "Grand total: $" << grandTotal << "\n";

  • Expecting cin >> name to read a full name with spaces.
  • Reading values in a different order than the prompt asks for them.
