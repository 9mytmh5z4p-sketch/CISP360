Introduction to std::vector in C++

What is a vector?

You’ve learned about individual variables that store single values:

int age = 25;
double price = 19.99;
string name = "Alice";

But what if you need to store multiple related values? Like: - Test scores for 10 students - Temperatures for each day of the week - Prices of 5 different products - Names of all your friends

You could do this:

int score1 = 85;
int score2 = 92;
int score3 = 78;
int score4 = 95;
int score5 = 88;

But this is tedious and doesn’t scale! What if you had 100 scores?

Enter the vector!

A vector is a container that holds multiple values of the same type in a single variable.

Think of it as a row of boxes, all labeled with the same variable name, where each box holds one value.

scores: [ 85 ] [ 92 ] [ 78 ] [ 95 ] [ 88 ]
         [0]    [1]    [2]    [3]    [4]

Why use vectors?

Vectors solve several problems:

  • Store multiple related values under one name
  • Organized - Keep related data together
  • Flexible - Can hold any number of items
  • Easy to process - Can access any element by position
  • Part of the STL (Standard Template Library) - Built into C++

Including vector in your program

To use vectors, you must include the <vector> header:

#include <iostream>
#include <vector>  // Required for std::vector
using namespace std;

Just like you include <iostream> for cout and <string> for strings, you need <vector> for vectors.

Basic vector syntax

The basic syntax for declaring a vector:

vector<type> name;
  • vector - The container type
  • <type> - The type of data it will hold (int, double, string, etc.)
  • name - Your variable name

Examples:

vector<int> scores;           // Vector of integers
vector<double> prices;        // Vector of doubles
vector<string> names;         // Vector of strings
vector<char> grades;          // Vector of characters

Important: All elements in a vector must be the same type.

Initializing a vector with values

The most common way to create a vector is to initialize it with values:

vector<int> scores = {85, 92, 78, 95, 88};

This creates a vector named scores containing 5 integers.

More examples:

vector<double> temperatures = {72.5, 68.3, 75.1, 71.8, 69.5};
vector<string> days = {"Monday", "Tuesday", "Wednesday", "Thursday", "Friday"};
vector<char> grades = {'A', 'B', 'A', 'C', 'B'};
vector<bool> answers = {true, false, true, true, false};

Note: The curly braces { } contain the values, separated by commas.

Accessing vector elements

Each element in a vector has an index (position number).

Important: Indexes start at 0, not 1!

vector<int> scores = {85, 92, 78, 95, 88};
                      
Index:                 0   1   2   3   4
Value:                85  92  78  95  88

To access an element, use square brackets [ ]:

vector<int> scores = {85, 92, 78, 95, 88};

cout << scores[0];  // Displays: 85 (first element)
cout << scores[1];  // Displays: 92 (second element)
cout << scores[2];  // Displays: 78 (third element)
cout << scores[4];  // Displays: 88 (last element)

Complete example - Accessing elements

#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<double> prices = {19.99, 24.50, 15.75, 32.00, 8.25};
    
    cout << "First price: $" << prices[0] << endl;
    cout << "Second price: $" << prices[1] << endl;
    cout << "Third price: $" << prices[2] << endl;
    cout << "Fourth price: $" << prices[3] << endl;
    cout << "Fifth price: $" << prices[4] << endl;
    
    return 0;
}

Output:

First price: $19.99
Second price: $24.50
Third price: $15.75
Fourth price: $32.00
Fifth price: $8.25

Getting the size of a vector

Every vector knows how many elements it contains. Use the .size() method:

vector<int> scores = {85, 92, 78, 95, 88};

cout << "Number of scores: " << scores.size();  // Displays: 5

More examples:

vector<string> names = {"Alice", "Bob", "Charlie"};
int count = names.size();  // count = 3

vector<double> temperatures = {72.5, 68.3, 75.1, 71.8, 69.5, 70.2, 73.8};
cout << temperatures.size();  // Displays: 7

Important: .size() returns the count of elements, not the last index! - Size = 5 means indexes 0, 1, 2, 3, 4 - Last index = size - 1

Using size() to access the last element

Since the last index is always size() - 1, you can access the last element like this:

vector<int> scores = {85, 92, 78, 95, 88};

cout << "First score: " << scores[0] << endl;
cout << "Last score: " << scores[scores.size() - 1] << endl;

Output:

First score: 85
Last score: 88

Common mistakes with vectors

Mistake 1: Forgetting to include <vector>

// WRONG - Missing include
#include <iostream>
using namespace std;

int main() {
    vector<int> scores = {85, 92};  // Error! vector not defined
}

Fix: Always include the header

#include <iostream>
#include <vector>  // Don't forget this!
using namespace std;

Mistake 2: Accessing non-existent indexes

vector<int> scores = {85, 92, 78};

cout << scores[5];  // ERROR! Only indexes 0, 1, 2 exist

Fix: Only use valid indexes (0 to size-1)

cout << scores[0];  // Valid
cout << scores[1];  // Valid
cout << scores[2];  // Valid
// cout << scores[3];  // Would be invalid!

Mistake 3: Forgetting indexes start at 0

vector<int> scores = {85, 92, 78};

cout << scores[1];  // This is 92, not 85!
cout << scores[0];  // This is the first element: 85

Mistake 4: Mixing types in a vector

// WRONG - Can't mix types
vector<int> values = {10, 3.14, 25};  // 3.14 will be converted to 3!

Fix: All elements must be the same type

vector<double> values = {10.0, 3.14, 25.0};  // All doubles

Mistake 5: Confusing size with last index

vector<int> scores = {85, 92, 78, 95, 88};

cout << scores[scores.size()];  // WRONG! Index 5 doesn't exist

Fix: Last index is size() - 1

cout << scores[scores.size() - 1];  // Correct! Accesses index 4

Mistake 6: Modifying non-existent indexes

vector<int> scores = {85, 92, 78};

scores[5] = 100;  // DANGER! Index 5 doesn't exist

Fix: Use push_back() to add beyond current size

scores.push_back(100);  // Correct! Adds 100 at the end

When to use vectors

Use vectors when you need to:

  • Store multiple values of the same type
  • Keep related data together
  • Work with data sets (temperatures, scores, prices)
  • Store collections of information
  • Organize data by position

Examples of good vector uses: - Student test scores - Daily temperatures - Product prices - Employee names - Monthly sales figures - Game high scores

Adding elements to a vector

You can add new elements to the end of a vector using .push_back():

vector<int> scores = {85, 92, 78};

cout << "Original size: " << scores.size() << endl;

scores.push_back(95);  // Add 95 to the end

cout << "New size: " << scores.size() << endl;
cout << "Last element: " << scores[3] << endl;

Output:

Original size: 3
New size: 4
Last element: 95

What happened: - The vector originally had 3 elements (indexes 0, 1, 2) - push_back(95) added 95 at the end (index 3) - The size increased from 3 to 4

Removing the last element

You can remove the last element from a vector using .pop_back():

vector<int> scores = {85, 92, 78, 95, 88};

cout << "Original size: " << scores.size() << endl;
cout << "Last element: " << scores[4] << endl;

scores.pop_back();  // Remove the last element (88)

cout << "New size: " << scores.size() << endl;
cout << "New last element: " << scores[3] << endl;

Output:

Original size: 5
Last element: 88
New size: 4
New last element: 95

Important: pop_back() does not return the removed value, it just removes it!

pop_back() example - Undo last entry

vector<string> tasks = {"Email", "Meeting", "Report", "Call"};

cout << "Tasks: " << tasks.size() << endl;
cout << "Last task: " << tasks[3] << endl;

cout << "\nOops! Removing last task..." << endl;
tasks.pop_back();

cout << "Tasks now: " << tasks.size() << endl;
cout << "Last task: " << tasks[2] << endl;

Output:

Tasks: 4
Last task: Call

Oops! Removing last task...
Tasks now: 3
Last task: Report

Removing all elements

You can remove all elements from a vector using .clear():

vector<int> scores = {85, 92, 78, 95, 88};

cout << "Size before clear: " << scores.size() << endl;

if (scores.empty()) {

Output:

Size before clear: 5
Size after clear: 0
Vector is now empty!

Key Takeaways

1. Vectors store multiple values and can grow or shrink dynamically

Instead of creating separate variables like score1, score2, score3, use a vector to keep all related data together: vector<int> scores = {85, 92, 78, 95, 88};. You can add elements with push_back(), remove the last element with pop_back(), and clear all elements with clear(). Remember that all elements must be the same type and indexes start at 0.

2. Access and modify vector elements using square brackets and the index position

Use vectorName[index] to access any element, where the first element is at index 0: scores[0] gets the first score, scores[1] gets the second, and so on. You can also modify elements: scores[1] = 100; changes the second score. Always ensure the index exists (0 to size-1) before accessing or modifying it.

3. Use push_back() to add elements and .size() to track the count

Add new elements to the end of a vector with push_back(): scores.push_back(95); adds 95 to the end. Every vector knows its length via .size(), which returns the count of elements. If a vector has 5 elements, the valid indexes are 0, 1, 2, 3, and 4. The last element is always accessed with vectorName[vectorName.size() - 1].

Range-Based For Loop in C++

What is a range-based for loop?

You’ve learned how to access individual vector elements using indexes:

vector<int> scores = {85, 92, 78, 95, 88};

cout << scores[0] << endl;
cout << scores[1] << endl;
cout << scores[2] << endl;
cout << scores[3] << endl;
cout << scores[4] << endl;

But what if you have 100 scores? Typing each one would be tedious!

What to do, what to do?

Range-Based for Loop

The range-based for loop (also called a “for-each loop”) automatically goes through every element in a container like a vector.

Think of it as: “For each element in the vector, do something with it.”

Why use range-based for loops?

Range-based for loops make your code:

  • Cleaner - No index management needed
  • Easier to read - The intent is clear
  • Less error-prone - No risk of wrong indexes
  • More concise - Fewer lines of code
  • More modern - Introduced in C++11

Compare these two approaches:

Without range-based loop (manual indexing):

vector<int> scores = {85, 92, 78, 95, 88};

for (int i = 0; i < scores.size(); i++) {
    cout << scores[i] << endl;
}

I know you haven’t learned this loop yet, but this is what you would need to do if we didn’t have the eor-each loop.

With range-based loop:

vector<int> scores = {85, 92, 78, 95, 88};

for (int score : scores) {
    cout << score << endl;
}

Both produce the same output, but the range-based loop is simpler!

Basic syntax

The basic syntax of a range-based for loop:

for (type element : container) {
    // Use element here
}

Breaking it down: - for - The for loop keyword - type - The data type of each element - element - A variable name you choose (holds each value) - : - Read as “in” (for each element in container) - container - The vector (or other container) to loop through

Example:

vector<int> scores = {85, 92, 78, 95, 88};

for (int score : scores) {
    cout << score << endl;
}

Read as: “For each int score in scores, display the score”

How it works - Step by step

Let’s see what happens at each step:

vector<string> fruits = {"Apple", "Banana", "Orange"};

for (string fruit : fruits) {   // apple
    cout << fruit << endl;
}

Step 1: fruit gets the first element (“Apple”) - Executes: cout << "Apple" << endl;

vector<string> fruits = {"Apple", "Banana", "Orange"};

for (string fruit : fruits) {   // banana
    cout << fruit << endl;
}

Step 2: fruit gets the second element (“Banana”) - Executes: cout << "Banana" << endl;

Naming the loop variable

You can name the loop variable anything you want. Choose names that make sense!

Good names:

for (int score : scores) { }  // Clear and descriptive

vector<string> cities = {"Paris", "London", "Tokyo"};
for (string city : cities) { }  // Singular of container name

vector<double> temperatures = {72.5, 68.3, 75.1};
for (double temp : temperatures) { }  // Abbreviation is fine

Avoid confusing names:

vector<int> scores = {85, 92, 78};
for (int x : scores) { }  // Too generic
for (int data : scores) { }  // Vague
for (int scores : scores) { }  // Same as container name - confusing!

Convention: Use the singular form of your vector name for the loop variable.

Calculating with range-based loops

Range-based for loops are perfect for calculations involving all elements.

    total += score;
}

How it works: - Start with total = 0 - First iteration: total = 0 + 85 = 85 - Second iteration: total = 85 + 92 = 177 - Third iteration: total = 177 + 78 = 255 - Fourth iteration: total = 255 + 95 = 350 - Fifth iteration: total = 350 + 88 = 438

Copy behavior - Important concept

The range-based for loop copies each element into the loop variable.

vector<int> scores = {85, 92, 78};

for (int score : scores) {
    cout << score << endl;
}

What’s happening: - Each element is copied from the vector into score - score is a separate variable with its own copy - Changes to score inside the loop do NOT affect the vector

This is perfect for read-only operations!

vector<int> numbers = {10, 20, 30};

for (int num : numbers) {
    num = num * 2;  // Changes num, NOT the vector
}

// Vector is unchanged
for (int num : numbers) {
    cout << num << " ";  // Displays: 10 20 30
}

Using const with range-based loops

For read-only operations, you can use const to make your intent clear and prevent accidental modifications.

vector<int> scores = {85, 92, 78, 95, 88};

for (const int score : scores) {
    cout << score << endl;
    // score = 100;  // This would cause a compiler error!
}

Why use const: - Makes code safer - prevents accidental changes - Documents intent - clearly read-only - May allow compiler optimizations

Best practice: Use const when you’re only reading values, not modifying them.

Common patterns with range-based loops

Pattern 1: Display all elements

for (type element : container) {
    cout << element << endl;
}

Pattern 2: Sum all elements

type total = 0;
for (type element : container) {
    total += element;
}

Pattern 3: Count matching elements

int count = 0;
for (type element : container) {
    if (element meets condition) {
        count++;
    }
}

Common mistakes with range-based loops

Mistake 1: Wrong data type

vector<double> prices = {19.99, 24.50, 15.75};

for (int price : prices) {  // WRONG! Should be double
    cout << price;  // Truncates to integers: 19, 24, 15
}

Fix:

for (double price : prices) {  // Correct type
    cout << price;
}

Mistake 2: Same name for loop variable and container

vector<int> scores = {85, 92, 78};

for (int scores : scores) {  // CONFUSING! Same name
    cout << scores;
}

Fix:

for (int score : scores) {  // Use singular form
    cout << score;
}

Mistake 3: Trying to access index

vector<string> names = {"Alice", "Bob", "Charlie"};

for (string name : names) {
    cout << i << ": " << name;  // ERROR! i doesn't exist
}

Fix (if you need index, use traditional for loop):

for (int i = 0; i < names.size(); i++) {
    cout << i << ": " << names[i];
}

Mistake 4: Expecting modifications to affect vector

vector<int> numbers = {10, 20, 30};

for (int num : numbers) {
    num = num * 2;  // Changes num, NOT the vector!
}
// Vector is still {10, 20, 30}

Note: This isn’t an error, but a misunderstanding. The loop variable is a copy.

Summary

Range-based for loops provide a clean, simple way to process every element in a vector.

Key concepts covered: - Syntax: for (type element : container) - Automatic iteration through all elements - No index management needed - Works with vectors of any type - Perfect for read-only operations - Cleaner and less error-prone than traditional for loops - Loop variable contains a copy of each element - Using const for safety in read-only operations

When to use range-based for loops: - Processing all elements - Calculations (sum, average, product) - Counting elements that meet criteria - Finding min/max values - Searching for values - Displaying all elements - Any operation that doesn’t need indexes

Remember: - Choose descriptive loop variable names (singular of container name) - Match the data type to the vector’s element type - Range-based loops copy elements (safe for reading) - Use const when only reading values - For operations needing indexes, use traditional for loops

Practice using range-based for loops with different vector types and operations. They’ll become your go-to tool for processing collections!

Key Takeaways

1. Range-based for loops automatically iterate through every element in a vector

The syntax for (type element : container) eliminates the need for index management and automatically processes each element in order. Read it as “for each element in container” - the loop variable gets each value one at a time, making your code cleaner and less error-prone. This is perfect when you need to work with all elements and don’t care about their position numbers.

2. The loop variable is a copy of each element, making range-based loops ideal for read-only operations

Each iteration copies an element from the vector into the loop variable, so changes to the loop variable don’t affect the original vector. This copy behavior makes range-based loops perfect for displaying, calculating sums/averages, counting, searching, and finding min/max values. Use const in the declaration (e.g., for (const int score : scores)) to document read-only intent and prevent accidental modifications.

3. Use range-based loops when processing all elements without needing indexes

Choose range-based for loops over traditional for loops when you need to process every element and don’t need position information. They excel at summing values, counting elements that meet criteria, finding extremes, and displaying data. If you need the index number or must process multiple vectors in parallel, use a traditional for loop instead. The cleaner syntax of range-based loops makes them the modern C++ preferred approach for simple iteration.
