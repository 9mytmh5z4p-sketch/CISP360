What Can’t the For-Each Loop Do?

What Can’t the For-Each Loop Do?

The for-each loop is clean and readable — so why do we need three more loops? Today we’ll look at the specific tasks for-each cannot handle, which is exactly why while, do-while, and for exist.

Total slides: 6

For-Each Is Great — Until It Isn’t

  • Great when you need to visit every element of a collection and read it.
  • Anything beyond “visit each element in order,” for-each starts to break down.
vector<int> scores = {85, 72, 90, 63};

for (int s : scores) {
    cout << s << "\n";
}

No Index Access

For-each gives you the value, not the position. If the task depends on where an element is, you are stuck.

// CANNOT do this with for-each:
// "Print only odd-indexed elements"
// "Find the position of the maximum value"
// "Replace element at position 3"

// You need a traditional loop for index access:
for (int i = 0; i < scores.size(); i++) {
    cout << "Position " << i << ": " << scores[i] << "\n";
}

No Parallel Vectors

Parallel vectors store related data across two separate vectors at matching indices. For-each has no way to walk two vectors in sync.

vector<string> names  = {"Alice", "Bob", "Carol"};
vector<int>    grades = {88,      74,    95};

// For-each can only see one vector at a time.
// You CANNOT pair names with grades using for-each.

// A traditional loop keeps the index in sync:
for (int i = 0; i < names.size(); i++) {
    cout << names[i] << ": " << grades[i] << "\n";
}

This is one of the most common real-world reasons to leave for-each behind.

No Menu-Driven Programs

  • menu-driven program loops until the user chooses to quit.
// For-each has nothing to loop over in this scenario.
// The loop must run based on user input, not a collection.

int choice = 0;
do {
    displayMenu();
    cin >> choice;
    processChoice(choice);
} while (choice != 0);

For-each requires a collection to iterate over — there is no collection here, just a condition. Menu-driven programs are the natural home of the do-while loop. We will build one today.

Summary

For-each is the right tool when you read every element of a single collection in order. When you need an index, parallel data, or condition-based repetition, you need a different loop.

Appendix

Common Practices

  • Use for-each when you only need the value and do not need to modify the collection.
  • Switch to a traditional loop the moment you need the index, parallel traversal, or any form of early exit.

Common Mistakes

  • Using for-each with auto and expecting to modify the original vector — the loop variable is a copy.
  • Trying to keep a manual counter inside a for-each to simulate an index — this works but defeats the purpose; use a for loop instead.

More Examples

// Modifying elements requires a reference — but for-each still
// cannot give you the index even with a reference:
for (auto& s : scores) {
    s += 5;  // adds 5 to every element — OK
}

// Still cannot do: "add 5 only to elements at even positions"
// That requires a for loop.

  2. Given two parallel vectors names and scores, print each name with its corresponding score. Which loop type is most appropriate?
  3. List two real-world tasks from this course where for-each is the right choice and two where it is not.

The While Loop

The While Loop

An if statement checks a condition once. What if you need to keep checking until the condition becomes false? The while loop is an if statement that jumps back to the top.

Total slides: 10

The While Loop Is a Looping If Statement

An if statement checks a condition and runs a block — once. A while loop checks a condition and runs a block — then jumps back and checks again.

if (condition) {        while (condition) {
    // runs once            // runs repeatedly
}                       }

The only structural difference: at the end of the block, a while loop returns to the condition instead of falling through.

[Graphic suggestion: Side-by-side flowchart — if statement flows down after the block; while loop has an arrow looping back from the end of the block to the condition diamond.]

Syntax

while (condition) {
    // body — executes as long as condition is true
}

The condition is evaluated before each iteration. If it is false the first time, the body never runs. This makes the while loop an entry-condition loop.

The Three Parts of Every Loop

Every loop needs three things or it will run forever.

  1. Initialize the loop control variable
  2. Test the condition
  3. Update the loop control variable

The Three Parts of Every Loop

int count = 0;          // 1. Initialize the loop control variable

while (count < 5) {     // 2. Test the condition
    cout << count << "\n";
    count++;            // 3. Update the loop control variable
}

If you forget to update the loop control variable inside the body, the condition never changes and the loop runs forever.

[Graphic suggestion: Annotated code diagram with colored brackets highlighting each of the three parts: initialize (before loop), test (condition), update (inside body).]

Loop Control Variable Scope

  • The loop control variable can live in different places depending on how the rest of the program uses it.
  • With while loops, the loop control variable must be declared before the loop if you need to read its final value afterward.

Before Variable Scope

// Declared before the loop — visible outside it
int count = 0;
while (count < 5) {
    count++;
}
cout << count;   // 5 — still accessible here

After Variable Scope



// Declared inside the loop — this is a compile error:
while (int i = 0; i < 5) {  // NOT valid while syntax
    i = i + 1;
}

Counting Up

int i = 1;
while (i <= 10) {
    cout << i << "\n";
    i = i + 1;
}
// Prints 1 through 10

This is the most common while loop pattern. Start at a known value, test against a limit, increment each pass.

Counting Down

int countdown = 5;
while (countdown > 0) {
    cout << countdown << "\n";
    countdown = countdown - 1;
}
cout << "Launch!\n";
// Prints 5, 4, 3, 2, 1, then Launch!

The loop control variable can move in any direction as long as the condition will eventually become false.

Input Validation

  • One of the most practical uses of the while loop is forcing valid input. Keep asking until the user provides an acceptable value.

Input Validation

int age;
cout << "Enter your age (1-120): ";
cin >> age;             // called a priming read

while (age < 1 || age > 120) {
    cout << "Invalid. Enter your age (1-120): ";
    cin >> age;
}

// age is guaranteed valid here

The input is read once before the loop, then again inside the body so the condition can re-evaluate.

Instructor note: Point out the “priming read” pattern — cin before the loop, then cin again at the bottom of the body. This is a classic pattern students will use repeatedly.

Unknown Iteration Count

  • Use a while loop when you do not know in advance how many times the loop will run.

Unknown Iteration Count

int value;
int sum = 0;
cout << "Enter values (0 to stop): ";
cin >> value;

while (value != 0) {
    sum = sum + value;
    cin >> value;
}

cout << "Sum: " << sum << "\n";

The loop runs until the user decides to stop. There is no way to know in advance how many values they will enter.

Summary

The while loop tests its condition before every iteration — if the condition starts false, the body never runs. Initialize your loop control variable before the loop, test it in the condition, and update it inside the body.

Appendix

Common Practices

  • Read input once before a validation loop, then again at the bottom of the body. This pattern — called a priming read — keeps the condition clean.
  • Prefer while over for when the number of iterations is not known at the start of the loop.

Common Mistakes

  • Forgetting to update the loop control variable inside the body, causing an infinite loop.
  • Using = instead of == in the condition — this assigns a value rather than comparing, which is almost always a bug.
  • Initializing the loop control variable inside the loop body instead of before it, which resets it every iteration.

More Examples

// Sum all positive numbers entered by the user
int n, total = 0;
cout << "Enter a number (negative to stop): ";
cin >> n;

while (n >= 0) {
    total += n;
    cout << "Enter a number (negative to stop): ";
    cin >> n;
}

cout << "Total: " << total << "\n";

Practice Problems

  1. Write a while loop that prints all even numbers from 2 to 20.
  2. Write a while loop that asks the user for a password and keeps asking until they enter “open sesame”.
  3. Write a while loop that counts down from 100 to 1 in steps of 10.
  4. What is the output of this loop? int x = 10; while (x > 0) { cout << x << " "; x -= 3; }

The Do-While Loop

The Do-While Loop

What if you need the loop body to run at least once, regardless of the condition? The do-while loop puts the condition at the bottom — so the body always executes first.

Total slides: 8

Exit-Condition Loop

  • The while loop checks its condition before running.
  • The do-while loop checks its condition after running. This one difference changes everything about when to use it.
while (condition) {
    // may never run
}
do {
    // always runs at least once
} while (condition)

Exit-Condition Loop

The do-while is an exit-condition loop. The body executes first; only then is the condition evaluated.

[Graphic suggestion: Two flowcharts side by side — while with the condition diamond above the block, do-while with the condition diamond below the block. Arrow looping back from condition to the top of the block in both.]

Syntax

do {
    // body — always executes at least once
} while (condition);
//               ^ note the semicolon — required

Do not forget the semicolon after the closing parenthesis. It is the one place in C++ where a control structure ends with a semicolon.

When to Use Do-While

Use a do-while loop when the program must execute the body before it has any information to test.

Two classic cases:

  • Menus — display the menu, get a choice, then decide whether to repeat.
  • Input validation — prompt the user, get input, then check if it is acceptable.

In both cases, you cannot evaluate the condition until after you have run the body at least once.

Menu-Driven Programs

  • Menus display and run at least once.
  • A menu must be shown before the user can choose.
  • The choice is what the loop condition tests — so the display and input must come first.
  • If the user enters 0, the loop exits immediately after that pass.
int choice;

do {
    cout << "1. New Game\n";
    cout << "2. Load Game\n";
    cout << "0. Quit\n";
    cout << "Choice: ";
    cin >> choice;

    if (choice == 1) startNewGame();
    if (choice == 2) loadGame();

} while (choice != 0);

Instructor note: This is the canonical do-while example. Walk students through how the condition at the bottom serves as the “should we go again?” test after every menu interaction.

Input Validation (Alternative to While)

Do-while is also a clean option for input validation when you want to prompt first and check afterward.

int score;

do {
    cout << "Enter a score (0-100): ";
    cin >> score;
} while (score < 0 || score > 100);

// score is guaranteed valid here

Compare this to the while version, which requires a priming read before the loop. The do-while eliminates that duplication.

Comparing While and Do-While

// While — may skip the body entirely
int x = 10;
while (x < 5) {
    cout << x;   // never prints
}
// Do-while — always runs the body once
int y = 10;
do {
    cout << y;   // prints 10
} while (y < 5);

If there is any chance the body should be skipped on the first pass, use while. If the body must run at least once, use do-while.

Summary

The do-while loop runs its body first and tests its condition last. Use it for menus and input validation where you need at least one execution before you can evaluate whether to continue.

Appendix

Common Practices

  • Use do-while for menus and input validation. These are the two situations it was designed for.
  • Prefer while over do-while when there is any possibility the loop should not run at all — the entry-condition is safer by default.

Common Mistakes

  • Forgetting the semicolon after } while (condition); — this is a syntax error unique to do-while.
  • Using do-while when the loop might legitimately need to run zero times — the body will execute once regardless.
  • Putting the update to the loop control variable after the while condition instead of inside the body, causing the condition to never change.

More Examples

// Keep asking until a positive number is entered
int n;
do {
    cout << "Enter a positive number: ";
    cin >> n;
} while (n <= 0);

cout << "You entered: " << n << "\n";

  3. Rewrite this while loop as a do-while: int x = 0; while (x != 5) { cin >> x; } — is the behavior identical? Explain.

The For Loop

The For Loop

When you know exactly how many times a loop should run, the for loop puts initialization, condition, and update in one place — making the loop’s intent immediately clear.

Total slides: 9

One Line, Three Jobs

The for loop consolidates the three parts of every loop into its header, so nothing is scattered across the code.

//    initialize    condition    update
for ( int i = 0;   i < 10;      i++    ) {
    // body
}

Compare this to while, where initialize lives before the loop and update lives somewhere inside the body. The for loop makes the loop’s mechanics visible at a glance.

[Graphic suggestion: Annotated diagram of a for loop header with colored brackets over each of the three parts: initialize (green), condition (yellow), update (red).]

Execution Order

The three parts of a for loop do not execute in the order they are written.

1. Initialize  — runs once, before the loop starts
2. Condition   — evaluated before every iteration
3. Body        — runs if condition is true
4. Update      — runs after the body, before the next condition check
5. Repeat from step 2
for (int i = 0; i < 3; i= 1 + 1) { // Initialize & Condition here
    cout << i << "\n";        // Body
}                             // Update implied here
// Output: 0, 1, 2

The initialize step is never repeated. Everything else cycles until the condition is false.

Loop Control Variable Scope

The loop control variable declared in a for header exists only inside the loop.

for (int i = 0; i < 5; i++) {
    cout << i;  // valid
}
// cout << i;   // compile error — i is out of scope

// If you need the final value after the loop, declare it before:
int j;
for (j = 0; j < 5; j++) { }
cout << j;  // 5 — valid

Keeping the loop variable inside the header is preferred when you do not need it afterward — it limits its scope intentionally.

Counting Up

for (int i = 1; i <= 5; i++) {
    cout << i << "\n";
}
// Prints 1, 2, 3, 4, 5

This is the most common for loop pattern: start at a lower bound, stop at an upper bound, step by one.

Counting Down

for (int i = 10; i >= 1; i--) {
    cout << i << "\n";
}
// Prints 10, 9, 8, ... 1

The update expression can be any valid operation. Counting down requires decrementing (i--) and reversing the condition (>=).

Step Size

The update expression is not limited to i++. Any arithmetic works.

// Count by 5s
for (int i = 0; i <= 50; i += 5) {
    cout << i << "\n";
}
// Prints 0, 5, 10, 15, ... 50

// Count by 2s — even numbers only
for (int i = 2; i <= 20; i += 2) {
    cout << i << " ";
}

Choose the step size that matches the problem. Every step size is valid as long as the condition will eventually become false.

Iterating Over a Vector by Index

For loops are the natural choice for index-based vector iteration.

vector<string> names = {"Alice", "Bob", "Carol"};

for (int i = 0; i < names.size(); i++) {
    cout << i << ": " << names[i] << "\n";
}
// Output:
// 0: Alice
// 1: Bob
// 2: Carol

This gives you both the index and the value — something for-each cannot do.

Instructor note: This is a good moment to contrast with for-each and tie back to topic 1. Ask: “When would you use for-each here instead?”

Summary

Use the for loop when you know the number of iterations in advance. Its header puts initialize, condition, and update in one place, making counter-controlled loops readable and compact.

Appendix

Common Practices

  • Declare the loop control variable in the for header when you do not need it outside the loop — this keeps its scope intentionally small.
  • Use i < size rather than i <= size - 1 for clarity when iterating over a vector or array.

Common Mistakes

  • Using i <= size instead of i < size when iterating a vector — this causes an off-by-one error and accesses an out-of-bounds element.
  • Modifying the loop control variable inside the body as well as in the update expression — this double-increments and is almost always a bug.
  • Forgetting that the loop control variable declared in the header does not exist outside the loop.

More Examples

// Print a multiplication table row for 7
for (int i = 1; i <= 10; i++) {
    cout << "7 x " << i << " = " << 7 * i << "\n";
}

// Sum all elements in a vector
vector<int> scores = {85, 72, 90, 63, 78};
int total = 0;
for (int i = 0; i < scores.size(); i++) {
    total += scores[i];
}
cout << "Total: " << total << "\n";

  2. Write a for loop that counts backward from 20 to 1.
  3. Given a vector of integers, use a for loop to find and print the largest value.
  4. What is the output? for (int i = 0; i < 5; i += 2) { cout << i << " "; }

Which Loop to Use?

Which Loop to Use?

You now have three loops. The question is not which one works — any of them can technically solve most problems. The question is which one communicates your intent most clearly.

Total slides: 8

The Decision Framework

Three questions narrow the choice quickly.

1. Do you know the number of iterations in advance?
       YES → for loop

2. Must the body run at least once before you can test?
       YES → do-while loop

3. Neither of the above?
       → while loop

When in doubt, start with while. It is the most general-purpose loop and the easiest to reason about.

[Graphic suggestion: Flowchart with three decision diamonds leading to each loop type.]

While — Unknown Count, May Skip

Use while when you do not know how many times the loop will run, and it is acceptable — or required — that the body never runs at all.

// Read values until the user stops
int val;
cin >> val;
while (val != -1) {
    process(val);
    cin >> val;
}

If the user enters -1 immediately, the body runs zero times. That is correct behavior here.

Do-While —
Must Run At Least Once

  • Use do-while when the body must execute before you have anything to test.
  • Menus and input prompts are the defining cases.
int choice;
do {
    displayMenu();
    cin >> choice;
    handleChoice(choice);
} while (choice != 0);

You cannot show a menu and ask for a choice without running the body first. Do-while is the right tool.

For — Known Count

Use for when you know exactly how many iterations are needed before the loop starts.

int n;
cout << "How many students? ";
cin >> n;               // count is read before the loop begins

for (int i = 1; i <= n; i++) { // header makes the iteration plan explicit
    int score;
    cout << "Enter score for student " << i << ": ";
    cin >> score;
    process(score);
}

Scenario: Processing Test Scores

You ask the instructor how many students are in the class. You will read one test score per student. You do not run the program with zero students.


Which loop?

  • For-Each loop
  • While loop
  • Do While loop
  • for loop
  • If statement

[Graphic suggestion: Pause slide with the scenario — let students call it out before revealing the answer on the next slide.]

Scenario: Processing Test Scores — Answer

For loop.

The count is known before iteration begins (n students). The loop runs a fixed number of times. There is no need to test a condition mid-stream or guarantee at least one execution.

for (int i = 1; i <= n; i++) {
    cin >> score;
    total += score;
}

Summary

For: known count. Do-while: must run once. While: everything else. When the right answer is not obvious, while is the safe default — it is the most flexible and easiest to reason about.

Appendix

Common Practices

  • Choose the loop that communicates intent, not just the one that works. A for loop signals “I know how many times” to the next person reading the code.
  • When refactoring, if you find yourself adding a counter variable to a while loop just to track iterations, consider replacing it with a for loop.

Common Mistakes

  • Using a for loop when the iteration count depends on runtime input that might change mid-loop — this forces awkward workarounds; use while instead.
  • Using do-while when the body should potentially run zero times — use while to avoid an unintended first execution.
  • Choosing a loop based on what “looks right” syntactically rather than what the problem actually requires.

More Examples

// Scenario: Read records from a network. Process if age > 18.
// The record count is unknown and may be zero.
// → while loop

while (readRecord(record)) {
    if (record.age > 18) {
        process(record);
    }
}

// Scenario: Run a GUI event loop until the quit button is pressed.
// The loop must execute at least once to display the window.
// → do-while

do {
    event = getNextEvent();
    handleEvent(event);
} while (event.type != QUIT);

  2. You prompt the user for a team number (1–5, excluding 2). Which loop validates the input?
  3. A program monitors a server and checks its status every 10 seconds until a timeout is reached. You know the maximum check count. Which loop is most appropriate?
  4. Rewrite a for loop as a while loop. Does the behavior change?

Lab Exercise — Practicing Loops

Lab Exercise — Practicing Loops

You have seen all three loops. Now write one. This exercise asks you to solve the same problem using each loop type, then compare your solutions with for-each.

Total slides: 6

The Problem

Write a program that calculates how far a vehicle travels. The client enters the speed (mph) and total travel time (hours). The program prints the cumulative distance at each 1-hour mark.

Example: 3 hours at 40 mph

Hour    Distance
----    --------
   1          40
   2          80
   3         120

Your Assignment

Solve the problem three times — once with each loop type. Your solutions should produce identical output.

  • Row 1 — solve with a while loop
  • Row 2 — solve with a do-while loop
  • Row 3 — solve with a for loop

When your row is done, pick your best example and paste it into the class discussion thread. We will compare solutions and talk about the tradeoffs.

Instructor note: Give ~15 minutes. Then run the debrief slide before showing the for-each contrast. Watch for: students using do-while incorrectly if speed=0 is entered. Good teaching moment.

For-Each — Why It Doesn’t Fit Here

After writing your loop solutions, try to imagine writing this program with for-each. You will immediately run into a wall.

// For-each needs a collection. We don't have one.
// The iteration is driven by a count (travel time),
// not by elements in a container.

// The closest you could get would be to build a vector first:
vector<int> hours = {1, 2, 3};
for (int h : hours) {
    cout << h << "\t" << h * speed << "\n";
}
// But now you've done extra work just to use for-each.
// A for loop is the right tool here.

For-each iterates over existing data. For loops generate data through counting. These are different tasks.

Debrief

  • What worked in your solution?
  • What felt awkward — and why?
  • Which loop type felt most natural for this problem?

Connect your answer to the decision framework: known count, must run once, or neither.

Instructor note: Expected answer is for loop — the count is known (travel time entered by user before the loop starts). Students using while can explain why it also works but requires a separate counter. Do-while works but invites an off-by-one if not careful.

Summary

When iteration count is known in advance, the for loop expresses intent most clearly. For-each cannot replace counter-controlled loops — it requires a collection, not a count.

Appendix

Common Practices

  • Choose your loop type before writing a single line of code. Ask: do I know the count? Must the body run once? The answer determines the loop.
  • Print a header row before the loop and use setw() or tabs for alignment — clean output is part of the deliverable.

Common Mistakes

  • Off-by-one: starting the counter at 1 but running the condition as <= hours versus starting at 0 with < hours — both work, but be consistent.
  • Using do-while for this problem without checking whether the user could enter 0 hours — the body would run once and print distance for hour 1 before the loop exits.
  • Forgetting to read the user’s input before the loop starts.

More Examples

// Reference solution — for loop
int speed, hours;
cout << "Speed (mph): ";
cin >> speed;
cout << "Travel time (hours): ";
cin >> hours;

cout << "Hour\tDistance\n";
cout << "----\t--------\n";

for (int h = 1; h <= hours; h++) {
    cout << h << "\t" << h * speed << "\n";
}

Practice Problems

  1. Extend the program to also print the distance in kilometers (1 mile ≈ 1.609 km).
  2. Modify the program to accept a step size other than 1 hour (e.g., every 30 minutes).
  3. Rewrite the solution using a while loop. Is the result harder or easier to read?
  4. What would happen if the user enters 0 for travel time? Handle this case gracefully.

Key Takeaways

All three loops share the same three parts: initialize, test, update. The choice between them comes down to one question: do you know the count, must the body run at least once, or neither?
